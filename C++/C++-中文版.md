# 1. 基础

## 1.1 程序结构

### 1.1.1 头文件包含

头文件包含有两种方式：

```cpp
#include <iostream>      // 用于标准库头文件
#include "myheader.h"    // 用于自定义头文件
```

| 语法 | 搜索顺序 | 用途 |
| ------ | ------------------------------------ | ------------------------------------------- |
| `< >` | 优先搜索系统目录 | 标准库（如 `vector`、`string`） |
| `" "` | 优先搜索当前目录，其次系统目录 | 自定义头文件 |

### 1.1.2 使用 std 命名空间

使用标准库特性有两种方式：

#### 1.1.2.1 方式一：添加 `std::` 前缀

```cpp
#include <iostream>

int main() {
    std::cout << "Hello" << std::endl;
    return 0;
}
```

#### 1.1.2.2 方式二：使用 `using namespace std`

```cpp
#include <iostream>
using namespace std;

int main() {
    cout << "Hello" << endl;
    return 0;
}
```

### 1.1.3 main 函数

| 声明 | 标准 | 返回值 | 用法 |
| ------------- | -------------- | ------------------------------- | -------------------- |
| `int main()` | 标准 | `0` = 成功，非零 = 错误 | **始终使用此形式** |
| `void main()` | 非标准 | 无 | 避免使用（不可移植） |

```cpp
// 标准形式
int main() {
    return 0;  // 表示成功执行
}

// 带命令行参数
int main(int argc, char* argv[]) {
    return 0;
}
```

**注意**：`void main()` 在某些旧编译器（如 Turbo C++）上可以运行，但**不是有效的 C++**。为保证代码可移植性，请始终使用 `int main()`。

## 1.2 代码规范化

### 1.2.1 注释

| 类型 | 语法 | 用法 | 示例 |
|------|--------|-------|---------|
| **单行注释** | `//` | 从 `//` 到行尾的内容为注释 | `// 这是单行注释` |
| **多行注释** | `/* */` | `/*` 和 `*/` 之间的内容为注释 | `/* 这是多行注释 */` |

```cpp
// 这是单行注释，// 后的所有内容被忽略
int a = 10;  // 这是行尾注释

/*
 * 这是多行注释
 * 可以跨越多行
 * 常用于函数文档或详细说明
 */
int b = 20;

/* 多行注释也可以写在一行 */ int c = 30;
```

**注意**：
- 多行注释不能嵌套（`/* /* */ */` 会导致错误）
- 调试时可用 `//` 临时禁用一行代码
- 文档工具（如 Doxygen）可识别特殊注释格式

#### 1.2.1.1 程序结构注释

注释常用于标记代码段的开始和结束：

```cpp
/* 退出程序。 */
return 0;

/*---------------------------------------------*/
```

`return 0;` 语句结束程序执行并将控制权返回操作系统。返回值 **0** 表示成功执行。

`}`（右花括号）标记 `main` 函数体的结束，通常单独放在一行。为清晰起见，可用注释行标记函数结束。

### 1.2.2 空白字符

**空白字符**（空行和缩进）使程序更具**可读性**，更易于**修改**，并提供**一致的代码风格**。

| 类型 | 用途 | 示例 |
| --------------- | ----------------------------- | ------------------------- |
| **空行** | 分隔不同组件 | 函数段之间 |
| **缩进** | 显示程序结构 | `{ }` 块内部 |

#### 1.2.2.1 空行

空行分隔逻辑段以提高可读性：

```cpp
int main() {

    // 声明段
    int x = 5;
    double y;

    // 语句段
    y = x * 2;
    cout << y << endl;

    return 0;
}
```

#### 1.2.2.2 缩进

第二行的缩进表明它是**前一行的延续**。

```cpp
// 长语句跨多行
printf("The distance between the two points is "
       "%5.2f\n", distance);
// ^ 第二行缩进表示延续

// 另一个示例
cout << "This is a very long message that "
     << "needs to be split into multiple lines"
     << endl;
```

#### 1.2.2.3 表达式中的空格

| 风格 | 描述 | 示例 |
|------|------|------|
| **所有运算符周围加空格** | 有些人喜欢每个运算符周围都有空格 | `a * b + b / c * d` |
| **只在 + 和 - 周围加空格** | 推荐风格：只在二元加减法周围加空格（最后计算） | `a*b + b/c*d` |

**建议：** 只在二元 `+` 和 `-` 周围加空格，因为它们最后计算，使表达式结构更清晰。

```cpp
// 推荐风格
int result = a*b + b/c*d;  // 结构更清晰

// 替代风格（同样有效）
int result = a * b + b / c * d;  // 更多空格，但保持一致
```

> 另见：[1.4.5 表达式中的空格](#145-表达式中的空格) 了解与运算符优先级的关联。

### 1.2.3 标识符命名

选择有效标识符的规则：

| 规则 | 有效 | 无效 |
|------|---------|-----------|
| **首字符** | 字母或 `_` | 数字或特殊字符 |
| **其他字符** | 字母、数字、`_` | `-`、`@`、空格等 |
| **大小写敏感** | `abc` ≠ `ABC` | — |
| **长度** | 无限制 | — |

#### 1.2.3.1 有效标识符示例

| 标识符 | 说明 |
|------------|------|
| `distance` | 以字母开头 |
| `x_1` | 包含下划线和数字 |
| `X_sum` | 大小写敏感 — 与 `x_sum` 不同 |
| `average_measurement` | 带下划线的描述性名称 |
| `initial_time` | 清晰可读 |

#### 1.2.3.2 无效标识符示例

| 标识符 | 原因 |
|------------|--------|
| `1x` | 以数字开头 |
| `switch` | 保留关键字 |
| `$sum` | 包含无效字符 `$` |
| `rate%` | 包含无效字符 `%` |

#### 1.2.3.3 保留关键字

不能将 C++ 关键字用作标识符（如 `int`、`return`、`if`、`while`）。

#### 1.2.3.4 完整程序示例

```cpp
/*--------------------------------------*/
/* 程序 lecture1_1                      */
/*                                      */
/* 本程序计算两点间距离。               */
/*--------------------------------------*/

#include <stdio.h>
#include <math.h>

int main(void)
{
    // 声明并初始化变量。
    double x1=1, y1=5, x2=4, y2=7,
           side_1, side_2, distance;

    /* 计算直角三角形的边长。 */
    side_1 = x2 - x1;
    side_2 = y2 - y1;
    distance = sqrt(side_1*side_1 + side_2*side_2);

    /* 输出距离。 */
    printf("The distance between the two points is "
           "%5.2f \n", distance);

    /* 退出程序。 */
    return 0;
}
/*--------------------------------------*/
```

### 1.2.4 长表达式

将长表达式分解为多个语句以提高可读性。

**示例：** 计算 f = (x³ - 2x² + x - 6.3) / (x² + 0.05005x - 3.14)

```cpp
// 不好：难以阅读
f = (x*x*x - 2*x*x + x - 6.3) / (x*x + 0.05005*x - 3.14);

// 较好：使用续行符分割
f = (x*x*x - 2*x*x + x - 6.3) /
    (x*x + 0.05005*x - 3.14);

// 最佳：分别计算分子和分母
numerator = x*x*x - 2*x*x + x - 6.3;
denominator = x*x + 0.05005*x - 3.14;
f = numerator / denominator;
```

**关键要点：**
- 长单行表达式难以阅读和调试
- 对复杂子表达式使用中间变量
- 确保变量为浮点类型以获得正确的除法结果

## 1.3 声明与语句

`main` 函数包含两种类型的命令：

| 类型 | 用途 | 示例 |
|------|---------|---------|
| **声明** | 定义内存位置（变量） | `int x;` |
| **语句** | 执行动作和操作 | `x = 5; cout << x;` |

**关键规则**：声明必须**位于**语句之前。

```cpp
int main() {
    int x = 5;       // 声明（带初始化）
    double y;        // 声明（无初始化）

    y = x * 2;       // 语句
    cout << y;       // 语句

    return 0;
}
```

### 1.3.1 初始化

| 语法 | 说明 |
|--------|-------------|
| `int a;` | 仅声明（未初始化） |
| `int a = 5;` | 声明并初始化 |
| `int a(5);` | 直接初始化 |
| `int a{5};` | 花括号初始化（C++11） |

```cpp
double x1 = 1, y1 = 5;    // 已初始化
int a = 10, b;            // a 已初始化，b 未初始化

// 一行声明多个变量
double side_1, side_2, distance;  // 全部未初始化
```

**注意**：使用未初始化变量会导致未定义行为。使用前请始终初始化。

### 1.3.2 auto 类型推导（C++11）

`auto` 让编译器根据初始化表达式推导变量类型。

**基本用法：**
```cpp
auto i = 5;        // int
auto d = 5.0;      // double
auto f = 5.0f;     // float
auto c = 'a';      // char
auto s = "hello";  // const char*
```

**关键规则：**

| 语法 | 结果 | 说明 |
|--------|--------|-------|
| `auto x = expr` | 值类型（复制） | 去除引用和顶层 const |
| `auto& x = expr` | 引用类型 | 保留引用和 const |
| `auto* x = expr` | 指针类型 | 显式指针 |
| `const auto x = expr` | const 值 | 显式 const |

**常见模式：**
```cpp
// 迭代器（节省输入长类型名）
vector<int>::iterator it = v.begin();  // 冗长
auto it = v.begin();                    // 简洁

// 基于范围的 for 循环
for (auto& elem : container)  // 引用，避免复制
for (const auto& elem : container)  // const 引用

// Lambda 表达式
auto func = [](int x) { return x * 2; };
```

**重要：** `auto` 需要初始化。`auto x;` 是错误的。

## 1.4 运算符

C++ 中的运算符是对操作数执行操作的符号。

### 1.4.1 运算符分类

#### 1.4.1.1 按操作数数量分类

运算符可根据其需要的**操作数数量**进行分类：

| 分类 | 操作数数量 | 示例 |
| -------------------- | ------------------ | ------------------------------------------------------------------------------------------------------------- |
| **一元运算符** | 1 | `++`（自增）、`--`（自减）、`!`（逻辑非）、`~`（按位非）、`*`（解引用）、`&`（取地址） |
| **二元运算符** | 2 | `+`、`-`、`*`、`/`、`%`、`==`、`!=`、`<`、`>`、`&&`、`|\|`、`=` 等 |
| **三元运算符** | 3 | `?:`（条件运算符） |

**注意：** 某些符号根据上下文既可作为一元运算符也可作为二元运算符：
- `-`：一元负号（取反）vs. 二元减号（减法）
- `+`：一元正号 vs. 二元加号（加法）
- `*`：解引用 vs. 乘法
- `&`：取地址 vs. 按位与

#### 1.4.1.2 按用途分类

运算符也可按**功能或用途**分类：

| 类别 | 运算符 | 说明 |
|----------|-----------|-------------|
| **算术** | `+` `-` `*` `/` `%` | 数学计算 |
| **关系** | `==` `!=` `<` `>` `<=` `>=` | 比较值，返回布尔值 |
| **逻辑** | `&&` `\|\|` `!` | 布尔逻辑运算 |
| **位运算** | `&` `\|` `^` `~` `<<` `>>` | 对单个位进行操作 |
| **赋值** | `=` `+=` `-=` `*=` `/=` `%=` 等 | 为变量赋值 |
| **自增/自减** | `++` `--` | 加 1 或减 1 |
| **条件** | `?:` | 三元条件运算符 |
| **成员访问** | `.` `->` | 访问类/结构体成员 |
| **指针** | `*` `&` `->` `[]` | 解引用、取地址、索引 |
| **作用域** | `::` | 作用域解析 |
| **其他** | `,` `sizeof` `typeid` `new` `delete` | 逗号、大小查询、内存管理 |

### 1.4.2 二元运算符

#### 1.4.2.1 除法

```cpp
int a = 7 / 2;       // 结果：3（整数除法，截断小数部分）
double b = 7 / 2;    // 结果：3.0（仍是整数除法！）
double c = 7.0 / 2;  // 结果：3.5（浮点除法）
```

| 运算 | 结果 | 说明 |
|-----------|--------|------|
| `int / int` | `int` | 小数部分被舍弃 |
| `double / int` | `double` | 常规除法 |

#### 1.4.2.2 取模

返回除法的**余数**。

```cpp
int r = 7 % 3;   // 结果：1（7 = 3*2 + 1）
int s = 10 % 4;  // 结果：2（10 = 4*2 + 2）
int t = 6 % 2;   // 结果：0（整除，无余数）
```

**重要**：`%` 仅适用于**整数**。

```cpp
// double x = 7.5 % 2;  // 错误！% 要求整数
```

##### 1.4.2.2.1 负数取模

若任一整数为负，根据 C++ 标准不同，`a % b` 的结果**依赖于具体实现**：

##### 1.4.2.2.2 C++98/03：实现定义

标准**未规定**具体行为。不同编译器可能产生不同结果。

```cpp
// 结果可能因编译器而异！
-5 % 3;  // 可能是 -2 或 1，取决于编译器
```

##### 1.4.2.2.3 C++11 及以后：向零截断

自 C++11 起，标准统一规定整数除法**向零截断**，由此确定取模结果。

```cpp
// 所有符合 C++11 的编译器产生相同结果
-5 % 3;   // 结果：-2（保证）
5 % -3;   // 结果：2（保证）
-5 % -3;  // 结果：-2（保证）
```

##### 1.4.2.2.4 C++11 行为的关键特性
- 商 `a / b` 向零截断（舍弃小数部分）
- 余数符号与被除数（分子）相同
- 恒等式 `(a/b)*b + a%b == a` 始终成立

**注意：** C++ 只有这两个标准。现代编译器（C++11 及以后）采用"向零截断"规则。

#### 1.4.2.3 混合运算（隐式类型转换）

当操作数类型不同时，C++ 自动将"较窄"类型转换为"较宽"类型后再执行运算。

##### 1.4.2.3.1 类型提升层次
```
char → short → int → long → long long → float → double → long double
```

##### 1.4.2.3.2 常见混合运算场景

| 场景 | 结果类型 | 说明 |
|----------|-------------|-------------|
| `int` + `double` | `double` | `int` 提升为 `double` |
| `float` + `double` | `double` | `float` 提升为 `double` |
| `short` + `int` | `int` | `short` 提升为 `int` |
| `int` / `double` | `double` | 整数除法变为浮点除法 |

##### 1.4.2.3.3 常见陷阱 - 整数除法 vs 混合除法

```cpp
int sum = 7, count = 2;
double avg1 = sum / count;       // 结果：3.0（先执行整数除法！）
double avg2 = (double)sum / count; // 结果：3.5（正确）
double avg3 = sum / (double)count; // 结果：3.5（正确）
double avg4 = 1.0 * sum / count;   // 结果：3.5（正确）
```

**关键点：** 混合运算中，至少一个操作数必须是浮点类型才能得到浮点结果。赋值给 `double` 发生在除法运算**之后**。

> 另见：[幂运算](#1424-幂运算)

#### 1.4.2.4 幂运算

**重要：** C++ 中**没有**幂运算符。

| 表达式 | C++ | 说明 |
| ------ | --- | ---- |
| x⁴ | ❌ 没有 `^` 或 `**` 运算符 | 不像 Python (`**`) 或数学符号 |
| a² | `a * a` | 对于整数指数，使用重复乘法 |

**两种解决方案：**

##### 方案 1：重复乘法（推荐用于小整数指数）

```cpp
int square = a * a;           // a² - 快速、高效
int cube = a * a * a;         // a³ - 仍然高效
int fourth = a * a * a * a;   // a⁴ - 可接受
```

**何时使用：**
- 指数是小的固定整数（2、3、4）
- 性能关键代码
- 处理整数时

**为什么更快：** 直接 CPU 乘法，无函数调用开销，无浮点转换。

##### 方案 2：`pow()` 函数（用于一般幂运算）

```cpp
#include <cmath>  // 需要包含的头文件

// 语法：pow(底数, 指数)
double result1 = pow(x, 4);      // x⁴
double result2 = pow(2.0, 10);   // 2¹⁰ = 1024
double result3 = pow(x, 0.5);    // √x（平方根）
double result4 = pow(x, -1);     // 1/x
```

| 参数 | 类型 | 说明 |
|------|------|------|
| `base` | `double` | 底数 |
| `exponent` | `double` | 指数（可以是小数或负数） |
| **返回值** | `double` | 底数的指数次幂 |

**类型转换行为：**

| 你的输入 | 发生什么 | 结果类型 |
|----------|----------|----------|
| `pow(2, 3)` | `int` → 隐式转换为 `double` | `double` (8.0) |
| `pow(2.0f, 3)` | `float` → 转换为 `double` | `double` |
| `pow(2.0, 3.0)` | 已经是 `double` | `double` |

**关键点：**
- ✅ **参数不必是 double** - `int`、`float` 等会**隐式转换**为 `double`
- ✅ **返回值总是 double** - 即使你传入整数，结果也是 `double`

```cpp
int a = pow(2, 3);        // ⚠️ 警告：可能丢失数据（double 转 int）
                          // 实际结果 8.0 被截断为 8

double b = pow(2, 3);     // ✅ 正确：b = 8.0

// 常见陷阱
int c = pow(2, 3) / 2;    // 结果是 4.0 / 2 = 2.0，截断为 2
```

**何时使用：**
- 指数很大或在编译时未知
- 指数是小数（例如平方根、立方根）
- 指数是负数
- 处理浮点数时

**性能：** 由于函数调用开销和浮点计算，比重复乘法慢。

##### 对比总结

| 方法 | 速度 | 使用场景 | 示例 |
|------|------|----------|------|
| `a * a` | ⚡ 最快 | 小的固定整数指数 | `a²`、`a³` |
| `pow(a, 2)` | 🐢 较慢 | 变量或小数指数 | `a^b`、`√a` |

##### 重要警告

> **注意：** C++ 中的 `^` 符号是**按位异或运算符**，不是幂运算！
```cpp
int result = 2 ^ 3;  // 结果：1（二进制 XOR：10 XOR 11 = 01），不是 8！
```

##### double 值的 cout 格式化

> 另见：[1.7.2.7 double 值的 cout 格式化](#1727-double-值的-cout-格式化)

**问题：**
```cpp
cout << pow(5, 2);      // 输出：25（不是 25.0！）
cout << 25.0;           // 输出：25（不是 25.0！）
```

`pow()` **确实**返回 `double`（25.0），但 `cout` 对浮点数有**默认格式化规则**。

| 格式标志 | 效果 | 默认值 |
|----------|------|--------|
| `ios::fixed` | 固定小数位 | 未设置 |
| `ios::scientific` | 科学记数法 | 未设置 |
| `ios::showpoint` | **强制显示小数点** | **未设置** |

**默认行为：**
- 当 `showpoint` 未设置且小数部分为 `.0` 时，`cout` **省略小数点**
- 这是 C++ 中为了使输出更简洁的设计选择

**内存 vs 显示：**

```
内存 (IEEE 754):  [01000011 01100100 0000...] = 25.0 (double)
       ↓
   cout 格式化
       ↓
显示:            "25"（默认，省略 .0）
```

**如何强制显示小数：**

```cpp
#include <iomanip>  // 格式化需要

// 方法 1：强制显示小数点
cout << showpoint << 25.0;              // 输出：25.0000

// 方法 2：固定精度
cout << fixed << setprecision(2) << 25.0;  // 输出：25.00

// 方法 3：科学记数法
cout << scientific << 25.0;             // 输出：2.500000e+01
```

**验证：**

```cpp
auto result = pow(5, 2);  // auto 推导为：double
cout << typeid(result).name();  // 确认：它是 double！
```

> **关键要点：** 类型**确实是** `double`，但 `cout` 默认不显示 `.0`。这是格式化问题，不是类型错误！

### 1.4.3 类型转换运算符

类型转换运算符用于显式将值从一种数据类型转换为另一种。

#### 1.4.3.1 常见使用场景

| 表达式 | 结果 | 说明 |
|------------|--------|-------------|
| `sum / count`（均为 `int`） | `3`（若 sum=7, count=2） | 整数除法，舍弃小数 |
| `(float)sum / count` | `3.5` | `sum` 先转换为 float |

#### 1.4.3.2 C 风格类型转换

```cpp
float average = (float)sum / count;  // C 风格类型转换
int ascii = (int)'A';                // char 转 int
```

**语法：** `(目标类型)表达式`

**重要：** 括号**不可省略**！类型转换优先级高于大多数运算符，但语法要求目标类型必须用括号括起。

```cpp
// 正确：(float) 先作用于 sum，再执行除法
float avg1 = (float)sum / count;   // 结果：3.5（若 sum=7, count=2）

// 错误：(float)(a / b) - 先执行整数除法！
float avg2 = (float)(sum / count); // 结果：3.0（先截断整数除法）

// 错误：float a / b - 语法错误！
// float avg3 = float sum / count; // 编译错误！
```

**注意：** 类型转换运算符仅影响**计算中使用的值**；不会改变变量存储的值或变量的类型。

```cpp
int sum = 7;
float avg = (float)sum / 2;  // 仅本次计算使用 7.0

// sum 仍是值为 7 的 int
// (float)sum 是临时的，sum 本身保持不变！
```

> 另见：[幂运算](#1424-幂运算)

#### 1.4.3.3 C++ 风格类型转换（推荐）

C++ 提供四种类型转换运算符，实现更安全、更明确的转换：

| 转换 | 用途 | 示例 |
|------|-------|---------|
| `static_cast` | 安全的编译期转换 | `static_cast<double>(sum) / count` |
| `dynamic_cast` | 多态类转换 | （面向对象章节） |
| `const_cast` | 添加/移除 const 限定符 | `const_cast<char*>(str)` |
| `reinterpret_cast` | 底层位模式重新解释 | `reinterpret_cast<int*>(ptr)` |

##### 1.4.3.3.1 C++ 类型转换运算符比较

| 特性 | `static_cast` | `dynamic_cast` | `const_cast` | `reinterpret_cast` |
|----------------|---------------|----------------|--------------|---------------------|
| **安全性** | 安全 | 安全（运行时检查） | ⚠️ 谨慎使用 | ❌ 危险 |
| **编译期检查** | 是 | 部分 | 是 | 否 |
| **常见用途** | 基本类型转换 | 类继承转换 | 添加/移除 const | 底层操作 |

**建议：** C++ 中优先使用 `static_cast` 而非 C 风格类型转换。

```cpp
// C++ 风格（推荐）
double avg = static_cast<double>(sum) / count;
```

### 1.4.4 运算符优先级

| 优先级 | 运算符 | 结合性 |
|------------|----------|---------------|
| 1 | **括号：** `( )` | 由内向外 |
| 2 | **一元运算符：** `+` `-` `(类型)` | 从右向左 |
| 3 | **二元运算符：** `*` `/` `%` | 从左向右 |
| 4 | **二元运算符：** `+` `-` | 从左向右 |

**关键规则：**
- 一元运算符在二元运算符 `*`、`/`、`%` 之前计算
- 二元加法和减法最后计算
- 如果表达式中有多个相同优先级的运算符，变量或常量会按照特定顺序与运算符分组（或结合）
- 优先级越高（数字越小）越先计算
- 括号可覆盖优先级
- 相同优先级按结合性计算（从左向右或从右向左）

**示例：**
```cpp
int a = 5 + 3 * 2;       // 结果：11（不是 16），* 优先级比 + 高
int b = (5 + 3) * 2;     // 结果：16，括号覆盖优先级
int c = -5 + 3;          // 结果：-2，一元 - 优先级比二元 + 高
int d = -3 * 5;          // 结果：-15，一元 - 在 * 之前计算
int e = 8 / 4 / 2;       // 结果：1，左结合性：(8/4)/2
float f = (float)5 / 2;  // 结果：2.5，类型转换优先级更高

// 相同优先级表达式分组（从左到右结合性）
 a*b + b/c*d  // 计算为：(a*b) + ((b/c)*d)
// * 和 / 优先级相同，从左到右分组
```

### 1.4.5 表达式中的空格

运算符周围的空格是一个**风格问题**。选择一种风格并始终使用。

| 风格 | 描述 | 示例 |
|------|------|------|
| **所有运算符周围加空格** | 有些人喜欢每个运算符周围都有空格 | `a * b + b / c * d` |
| **只在 + 和 - 周围加空格** | 推荐风格：只在二元加减法周围加空格（最后计算） | `a*b + b/c*d` |

**关键点：**
- 空格是风格选择，不是语法要求
- **建议：** 只在二元 `+` 和 `-` 周围加空格，因为它们最后计算
- 这使表达式结构更清晰

```cpp
// 推荐风格 - 只在 + 和 - 周围加空格
int result = a*b + b/c*d;  // 结构更清晰

// 替代风格 - 所有运算符周围加空格
int result = a * b + b / c * d;  // 也有效，但不够清晰
```

> 另见：[1.2.2.3 表达式中的空格](#1223-表达式中的空格) 了解更多关于空白字符使用的细节。

数值数据类型分为两类：

| 类别 | 类型 | 说明 |
| ------------------ | -------------------------------- | ------------------------------ |
| **整数** | `short`、`int`、`long` | 无小数部分的整数 |
| **浮点数** | `float`、`double`、`long double` | 带小数点的数 |

#### 1.5.1.1 整数范围

| 类型 | 大小 | 范围 |
|------|------|-------|
| `short` | 2 字节 | -32,768 至 32,767 |
| `unsigned short` | 2 字节 | 0 至 65,535 |
| `int` | 4 字节 | -2,147,483,648 至 2,147,483,647 |
| `unsigned int` | 4 字节 | 0 至 4,294,967,295 |
| `long` | 4-8 字节 | 依赖于平台 |
| `long long` | 8 字节 | -9 百亿亿 至 9 百亿亿 |

#### 1.5.1.2 浮点精度

| 类型 | 大小 | 精度 | 典型范围 |
| ------------- | ---------- | ------------- | ------------------ |
| `float` | 4 字节 | 约 7 位 | ±3.4 × 10³⁸ |
| `double` | 8 字节 | 约 15 位 | ±1.7 × 10³⁰⁸ |

### 1.5.2 整数

**注意**：`signed` 和 `unsigned` 修饰符只能用于**整数类型**（`short`、`int`、`long`、`long long`）。**不能**用于浮点类型（`float`、`double`、`long double`）。

### 1.5.3 浮点数值

默认情况下，浮点常量如 `2.3` 被视为 **`double`** 常量。

要指定 **`float`** 或 **`long double`** 常量，需添加后缀 `F` 或 `L`：

| 常量 | 类型 | 示例 |
|----------|------|---------|
| `2.3` | `double`（默认） | `double x = 2.3;` |
| `2.3F` | `float` | `float y = 2.3F;` |
| `2.3L` | `long double` | `long double z = 2.3L;` |

```cpp
float a = 3.14;     // 警告：3.14 是 double，正在转换为 float
float b = 3.14F;    // 正确：3.14F 显式为 float
double c = 2.5;     // 正确：2.5 默认为 double
long double d = 2.5L;  // 正确：2.5L 为 long double
```

### 1.5.4 字符与字符串字面量

| 语法 | 类型 | 示例 | 存储 |
|--------|------|---------|---------|
| `' '` | **char**（单个字符） | `'A'`、`'\n'` | 1 字节 |
| `" "` | **string**（C 风格字符串） | `"Hello"` | 字符数组 + `\0` |

```cpp
char c = 'A';           // 正确，单个字符

char s1[] = "Hello";    // 正确，字符串字面量
string s2 = "Hello";    // 正确，C++ string
// char e = "A";        // 错误！不能将字符串赋值给 char
```

**关键点：**
- `'\n'` 是 `char`（换行字符）
- `"\n"` 是包含换行符 + 空终止符的字符串

#### 1.5.4.1 字符常量

- 用**单引号**括起：`'A'`、`'b'`、`'3'`
- 只能有**一个字符**
- `'3'` 是字符（ASCII 51），与整数 `3` 不同

```cpp
char c = 'A';       // 正确

### 1.5.5 符号常量

该指令可出现在 C++ 程序的任何位置。

#### 1.5.5.1 定义常量的方式

| 方式 | 语法 | 说明 |
| ------------------- | -------------------------------- | ------------------------------------------------ |
| **C 风格（宏）** | `#define PI 3.14159` | 预处理器指令，简单文本替换 |
| **C++ const** | `const double PI = 3.14159;` | 类型安全常量，C++ 中推荐使用 |
| **C++11 constexpr** | `constexpr double PI = 3.14159;` | 编译期常量，编译时求值 |

```cpp
// C 风格宏
#define PI 3.14159 // 注意：预处理器指令不以分号结尾
#define MAX_SIZE 100

// C++ const
const double PI = 3.14159;
const int MAX_SIZE = 100;
```

**注意：** `#define PI 3.14159` - 预处理器指令不以分号结尾

#### 1.5.5.2 重定义规则

**同一作用域**：不能用相同名称重定义
```cpp
const int MAX = 100;
const int MAX = 200;  // 错误！重定义

#define PI 3.14
#define PI 3.14159     // 错误！宏重定义
```

**不同作用域**：内层作用域隐藏外层作用域
```cpp
const int MAX = 100;  // 全局常量

int main() {
    const int MAX = 200;  // 局部常量，隐藏全局 MAX
    cout << MAX;          // 输出：200
    return 0;
}
```

**重定义宏**：必须先取消定义
```cpp
#define PI 3.14
#undef PI               // 取消定义
#define PI 3.14159      // 现在有效
```

#### 1.5.5.3 建议

- C++ 中**优先使用 `const` 或 `constexpr`** 而非 `#define`
- `const`/`constexpr` 具有类型检查和作用域规则
- `constexpr` 更严格：值必须在编译期可计算

### 1.5.6 类型转换

#### 1.5.6.1 类型提升层次

```
char → short → int → long → long long → float → double → long double
```

#### 1.5.6.2 运算类型转换

当操作数类型不同时，C++ 自动将"较窄"类型转换为"较宽"类型。

> 更多详情见 [混合运算](#混合运算) 和 [类型转换运算符](#类型转换运算符)。

| 场景 | 结果类型 | 说明 |
|----------|-------------|-------------|
| `int` + `double` | `double` | `int` 提升为 `double` |
| `float` + `double` | `double` | `float` 提升为 `double` |
| `short` + `int` | `int` | `short` 提升为 `int` |

```cpp
int a = 5;
double b = 2.5;
auto c = a + b;     // 结果：7.5（类型：double）
```

#### 1.5.6.3 常见陷阱

```cpp
double x = 5 / 2;       // 结果：2.0（先执行整数除法！）
double y = 5.0 / 2;     // 结果：2.5（正确）
```

**关键点**：整数间的运算产生整数结果。要得到小数结果，至少使用一个浮点数。

#### 1.5.6.4 赋值类型转换

| 方向 | 结果 | 示例 |
|-----------|--------|---------|
| 窄 → 宽 | 安全（隐式） | `double d = 3.14f;` |
| 宽 → 窄 | 警告，数据丢失 | `float f = 3.14;` // double→float |
| 浮点 → 整数 | 截断小数 | `int i = 3.9;` // i = 3 |

```cpp
// 安全：小 → 大
float f = 3.14f;
double d = f;           // 正确

// 不安全：大 → 小（避免）
double e = 3.141592653589793;
float g = e;            // 精度丢失！

// 浮点字面量陷阱
float x = 3.14;         // 警告：3.14 是 double
float y = 3.14f;        // 正确：3.14f 是 float

// 浮点 → 整数：截断，非四舍五入！
int a = 3.9f;           // a = 3（不是 4！）
int b = -2.7f;          // b = -2

## 1.7 数据输出

### 1.7.1 换行控制

两者都创建新行，但有关键区别：

| 语法 | 效果 | 用法 |
| ------ | --------------------------- | ------------------------------- |
| `endl` | 换行 + **刷新缓冲区** | 需要立即输出时 |
| `'\n'` | 仅换行 | 一般使用，性能更好 |

```cpp
cout << "Hello" << endl;    // 刷新缓冲区
cout << "Hello" << '\n';    // 仅换行，更快
```

**注意**：频繁使用 `endl` 会降低程序速度。除非需要强制立即输出，否则请使用 `'\n'`。

### 1.7.2 输出格式化

#### 1.7.2.1 头文件依赖

```cpp
#include <iomanip>
```

#### 1.7.2.2 核心机制

C++ 有三种浮点输出格式：

| 格式 | 说明 | 示例（12345.6789） |
|--------|-------------|----------------------|
| **默认** | 自动，可能使用科学记数法 | `12345.7` 或 `1.23457e+04` |
| `fixed` | 定点表示法 | `12345.678900` |
| `scientific` | 科学记数法 | `1.234568e+04` |

#### 1.7.2.3 setprecision(n) 含义

`setprecision(n)` 行为取决于当前格式：

- **默认**：`n` 位有效数字
- `fixed`/`scientific`：小数点后 `n` 位

#### 1.7.2.4 详细对比与示例

```cpp
double pi = 3.1415926535;
double big = 12345678.9;

// 1. 默认格式
cout << setprecision(4);
cout << pi << endl;    // 3.142（4 位有效数字）
cout << big << endl;   // 1.235e+07（科学记数法）

// 2. 定点格式
cout << fixed << setprecision(4);
cout << pi << endl;    // 3.1416（小数点后 4 位）
cout << big << endl;   // 12345678.9000

// 3. 科学记数法格式
cout << scientific << setprecision(4);
cout << 123.456 << endl;  // 1.2346e+02
```

#### 1.7.2.5 状态持久性（重要）

`fixed` 和 `setprecision` 持续到显式更改。

```cpp
cout << fixed << setprecision(2);
cout << 1.234 << endl;  // 1.23
cout << 5.678 << endl;  // 5.68（格式仍有效！）
```

#### 1.7.2.6 重置为默认

```cpp
// C++11 方式
cout << defaultfloat << setprecision(6);

// C++11 之前方式
cout.unsetf(ios::fixed | ios::scientific);
cout << setprecision(6);
```

#### 1.7.2.7 double 值的 cout 格式化

> 另见：[1.4.2.4 幂运算 - double 值的 cout 格式化](#1424-幂运算)

**问题：**
```cpp
cout << pow(5, 2);      // 输出：25（不是 25.0！）
cout << 25.0;           // 输出：25（不是 25.0！）
```

`pow()` **确实**返回 `double`（25.0），但 `cout` 对浮点数有**默认格式化规则**。

| 格式标志 | 效果 | 默认值 |
|----------|------|--------|
| `ios::fixed` | 固定小数位 | 未设置 |
| `ios::scientific` | 科学记数法 | 未设置 |
| `ios::showpoint` | **强制显示小数点** | **未设置** |

**默认行为：**
- 当 `showpoint` 未设置且小数部分为 `.0` 时，`cout` **省略小数点**
- 这是 C++ 中为了使输出更简洁的设计选择

**内存 vs 显示：**

```
内存 (IEEE 754):  [01000011 01100100 0000...] = 25.0 (double)
       ↓
   cout 格式化
       ↓
显示:            "25"（默认，省略 .0）
```

**如何强制显示小数：**

```cpp
#include <iomanip>  // 格式化需要

// 方法 1：强制显示小数点
cout << showpoint << 25.0;              // 输出：25.0000

// 方法 2：固定精度
cout << fixed << setprecision(2) << 25.0;  // 输出：25.00

// 方法 3：科学记数法
cout << scientific << 25.0;             // 输出：2.500000e+01
```

**验证：**

```cpp
auto result = pow(5, 2);  // auto 推导为：double
cout << typeid(result).name();  // 确认：它是 double！
```

> **关键要点：** 类型**确实是** `double`，但 `cout` 默认不显示 `.0`。这是格式化问题，不是类型错误！

#### 1.7.2.8 C 风格输出：`printf()` 格式说明符语法

##### 1.7.2.8.1 头文件依赖

```c
#include <stdio.h>
```

##### 1.7.2.8.2 基本语法结构

```
%[标志][宽度][.精度][长度修饰符]转换说明符

##### 1.7.2.8.3 转换说明符（必需）

| 说明符 | 数据类型 | 输出形式 |
|-----------|-----------|-------------|
| `%d` / `%i` | `int` | 有符号十进制整数 |

| `%e` / `%E` | `double` | 科学记数法（小写/大写 e） |

##### 1.7.2.8.4 标志（可选）

| 标志 | 名称 | 说明 |
|------|------|-------------|
| `-` | 左对齐 | 左对齐输出；右侧补空格（默认右对齐） |
| `+` | 符号 | 正数显示 `+`（负数始终显示 `-`） |
| ` `（空格） | 空格 | 正数前输出空格，负数输出 `-`（用于对齐） |

##### 1.7.2.8.5 宽度（可选）

- **数字**：最小字段宽度。若值较短，补空格（若标志 `0` 则补零）。若较长，不截断。
- **`*`**：宽度由额外的 `int` 参数指定。

##### 1.7.2.8.6 精度（可选）

以 `.` 为前缀：

| 类型 | 精度效果 |
|------|------------------|
| 整数（`d`、`i`、`o`、`u`、`x`、`X`） | 最少数字位数；不足时前导补零 |
| 浮点数（`f`、`e`、`E`） | 小数点后位数（默认 6） |
| 科学记数法（`g`、`G`） | 最大有效数字位数 |
| 字符串（`s`） | 最大输出字符数（截断） |

##### 1.7.2.8.7 长度修饰符（可选）

| 修饰符 | 用于 `d`/`i`/`o`/`u`/`x`/`X` | 用于 `n` |
|----------|----------------------------------|--------------|
| `hh` | `signed char` / `unsigned char` | `signed char*` |
| `h` | `short` / `unsigned short` | `short*` |
| `l` | `long` / `unsigned long` | `long*` |
| `ll` | `long long` / `unsigned long long` | `long long*` |
| `j` | `intmax_t` / `uintmax_t` | `intmax_t*` |
| `z` | `size_t` | `size_t*` |
| `t` | `ptrdiff_t` | `ptrdiff_t*` |
| `L` | 与 `f`/`e`/`g` 共用：`long double` | — |

##### 1.7.2.8.8 示例

```c
// 左对齐，宽度 10
printf("%-10d", 42);       // "42        "

// 显示正号，宽度 10
printf("%+10d", 42);       // "       +42"

// 零填充，宽度 10
printf("%010d", 42);       // "0000000042"

// 精度 5，最少 5 位数字
printf("%.5d", 42);        // "00042"

// 宽度 10，2 位小数
printf("%10.2f", 3.14159); // "      3.14"

// 字符串截断为 5 个字符
printf("%.5s", "HelloWorld"); // "Hello"

// 十六进制带 0x 前缀
printf("%#x", 255);        // "0xff"

// 动态宽度（宽度 = 10）
printf("%*d", 10, 42);     // "        42"

// 长长整数
printf("%lld", 9223372036854775807LL);
```
