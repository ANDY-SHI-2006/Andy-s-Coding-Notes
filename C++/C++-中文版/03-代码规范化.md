# 3 代码规范化

## 3.1 程序示例

这个完整示例展示了本章所有的代码规范化准则：

```cpp
/*------------------------------------------------------------*/
/* 程序：sinc_table.c                                         */
/*                                                            */
/* 该程序计算并输出用户指定区间 [a, b] 上的 sinc(x) 值表。     */
/*------------------------------------------------------------*/

#include <stdio.h>
#include <math.h>

#define PI 3.141593

/*------------------------------------------------------------*/
/* 该函数计算 sinc 函数的值。                                 */
/* Sinc(x) = sin(PI*x) / (PI*x) 当 x != 0，x = 0 时为 1      */
/*------------------------------------------------------------*/
double sinc(double x)
{
    if (fabs(x) < 0.0001)
        return 1.0;
    else
        return sin(PI * x) / (PI * x);
}

/*------------------------------------------------------------*/
int main(void)
{
    /* 声明变量。 */
    int k;
    double a, b, x_incr, new_x;

    /* 从用户获取区间端点。 */
    printf("Enter endpoints a and b (a<b): \n");
    scanf("%lf %lf", &a, &b);
    x_incr = (b - a) / 20;

    /* 计算并输出 sinc(x) 值表。 */
    printf("x and sinc(x) \n");
    for (k = 0; k <= 20; k++)
    {
        new_x = a + k * x_incr;
        printf("%f %f \n", new_x, sinc(new_x));
    }

    /* 退出程序。 */
    return 0;
}
/*------------------------------------------------------------*/
```

**展示的关键特性：**

| 准则 | 应用方式 |
|------|----------|
| **头部注释** (3.2.1) | 程序描述包含标题、用途和作者 |
| **函数注释** (3.2.3) | 每个函数前都有描述性注释块 |
| **步骤注释** (3.2.3) | `main()` 中的每一步都有注释：声明、输入、计算、输出 |
| **分隔线** (3.2.3) | `/*------------------*/` 块分隔各个组件 |
| **空行** (3.3.1) | 空行分隔声明、输入、计算等部分 |
| **缩进** (3.3.2) | 花括号内保持一致的 4 空格缩进 |
| **分行** (3.3.2) | 避免长表达式；每行一条清晰语句 |
| **命名** (3.4.2) | 描述性名称：`x_incr` 而非 `dx`，`new_x` 而非 `x1` |
| **不使用匈牙利命名法** (3.4.2) | 名称干净，无类型前缀 |
| **不使用关键字** (3.4.1) | 标识符中不使用保留字 |

## 3.2 注释

### 3.2.1 注释类型

| 类型 | 语法 | 用法 | 示例 |
|------|------|------|------|
| **单行注释** | `//` | 从 `//` 到行尾 | `// 这是单行注释` |
| **多行注释** | `/* */` | `/*` 和 `*/` 之间 | `/* 这是多行注释 */` |

```cpp
// 这是单行注释，// 之后的所有内容都会被忽略
int a = 10;  // 这是行尾注释

/*
 * 这是多行注释
 * 可以跨越多行
 * 常用于函数文档或详细说明
 */
int b = 20;

/* 多行注释也可以只占一行 */ int c = 30;
```

### 3.2.2 重要说明

| 注意点 | 描述 |
|--------|------|
| 嵌套 | 多行注释不能嵌套（`/* /* */ */` 会导致错误） |
| 调试 | 调试时可用 `//` 临时禁用一行代码 |

### 3.2.3 程序结构注释

注释常用于标记各节的开始和结束：

```cpp
/* 退出程序。 */
return 0;

/*---------------------------------------------*/

**函数注释：**
函数前应加上描述性注释，说明其用途：
```cpp
/* 该函数计算 sinc 函数的值。 */
```

在函数内部，使用注释记录每一步：
```cpp
/* 声明变量。 */
int k;
double a, b, x_incr, new_x;

/* 从用户获取区间端点。 */
printf("Enter endpoints a and b (a<b): \n");
scanf("%lf %lf", &a, &b);

/* 计算并输出 sinc(x) 值表。 */
printf("x and sinc(x) \n");

/* 退出程序。 */
return 0;
```

> 另请参阅：[8.3 程序员自定义函数](#83-programmer-defined-functions) 获取完整的函数组织示例。


## 3.3 空白字符

**空白字符**（空行和缩进）使程序更具**可读性**、更易于**修改**，并提供**一致的样式**。

| 类型 | 目的 | 示例 |
| ---- | ---- | ---- |
| **空行** | 分隔不同组件 | 函数各部分之间 |
| **缩进** | 展示程序结构 | 在 `{ }` 块内部 |

### 3.3.1 空行

空行分隔逻辑段以提高可读性：

```cpp
int main() {

    // 声明部分
    int x = 5;
    double y;

    // 语句部分
    y = x * 2;
    cout << y << endl;

    return 0;
}
```

### 3.3.2 缩进与分行

#### 3.3.2.1 基本缩进规则

花括号 `{}` 内的代码必须缩进以显示层次结构。

```cpp
int main() {          // 第 0 层：不缩进
    int a = 10;       // 第 1 层：1 级缩进（在 main 内）
    int b = 20;       // 第 1 层：1 级缩进
    if (a > b) {      // 第 1 层：1 级缩进
        cout << a;    // 第 2 层：2 级缩进（在 if 内）
    }                  // 第 1 层：回到 1 级缩进
    return 0;         // 第 1 层：1 级缩进
}                     // 第 0 层：不缩进
```

#### 3.3.2.2 行继续

第二行的缩进表明它是**前一行的继续**。

#### 3.3.2.3 长语句分行

如果语句太长，在保持可读性的位置将其拆分为两行。

##### 3.3.2.3.1 printf 语句的分行

**方法 1：在逗号后拆分**

在逗号后拆分，并将续行缩进至与左括号或第一个参数对齐。

```cpp
printf("The distance between the two points is %5.2f \n",
       distance);
```

**方法 2：将字符串拆分为两个带引号的部分**

将字符串字面量分成多个片段，每个片段放在各自的引号中。编译器会自动拼接它们。

```cpp
printf("The distance between the two points is"
       " %5.2f \n", distance);
```

**方法 3：在文本的自然断点处拆分**

在文本的逻辑断点处（例如单词之间）拆分以提高可读性。

```cpp
printf("The distance between the "
       "points is %5.2f \n", distance);
```

> 另请参阅：[7.2.6 拆分长 printf 语句](#726-splitting-long-printf-statements) 获取完整语法细节和更多示例。

**字符串拼接规则**

拆分字符串时，相邻的带引号字符串会被编译器自动拼接。这适用于任何字符串字面量，不仅限于 printf 语句。

```cpp
// cout 示例
cout << "This is a very long message that "
     << "needs to be split into multiple lines"
     << endl;
```

#### 3.3.2.4 拆分长表达式

将长表达式拆分为多条语句以提高可读性。

**示例：** 计算 f = (x³ - 2x² + x - 6.3) / (x² + 0.05005x - 3.14)

```cpp
// 糟糕：难以阅读
f = (x*x*x - 2*x*x + x - 6.3) / (x*x + 0.05005*x - 3.14);

// 较好：使用行继续
f = (x*x*x - 2*x*x + x - 6.3) /
    (x*x + 0.05005*x - 3.14);

// 最佳：分别计算分子和分母
numerator = x*x*x - 2*x*x + x - 6.3;
denominator = x*x + 0.05005*x - 3.14;
f = numerator / denominator;
```

**要点：**
- 长的一行表达式难以阅读和调试
- 对复杂的子表达式使用中间变量
- 确保变量为浮点类型以获得正确的除法结果

### 3.3.3 表达式中的间距

> 另请参阅：[4.12 间距与样式准则](#412-spacing-and-style-guidelines) 获取运算符专属的间距建议，以及 [4.6 运算符优先级](#46-operator-precedence) 了解表达式中的运算顺序。

#### 3.3.3.1 基本间距风格

运算符周围的间距有两种常见做法：

| 风格 | 描述 | 示例 |
|------|------|------|
| **所有运算符两侧都加空格** | 某些人喜欢每个运算符两侧都加空格 | `a * b + b / c * d` |
| **仅在 + 和 - 两侧加空格** | 推荐风格：仅在二元加减（最后求值）两侧加空格 | `a*b + b/c*d` |

**推荐：** 仅在二元 `+` 和 `-` 两侧加空格，因为它们最后求值，这使得表达式结构更清晰。

```cpp
// 推荐风格——突出表达式结构
int result = a*b + b/c*d;  // 更清晰：先算乘积，再相加

// 替代风格（也有效）——空格更多但一致
int result = a * b + b / c * d;
```

#### 3.3.3.2 多字符运算符的间距规则

**重要规则：** 运算符两侧可以使用空格，但多字符运算符**不能**用空格分隔。

| 有效 | 无效 | 原因 |
|------|------|------|
| `a == b` | `a = = b` | `==` 是单个运算符，不能拆分 |
| `x <= 10` | `x < = 10` | `<=` 是单个运算符，不能拆分 |
| `y >= 0` | `y > = 0` | `>=` 是单个运算符，不能拆分 |
| `a != b` | `a ! = b` | `!=` 是单个运算符，不能拆分 |

```cpp
// 正确间距——多字符运算符保持完整
if (a == b) { }      // 有效：== 在一起
if (x <= 10) { }     // 有效：<= 在一起

// 错误间距——拆分运算符会导致错误
if (a = = b) { }     // ✗错误：= = 是两个独立的运算符
if (x < = 10) { }    // ✗错误：< = 是两个独立的运算符
```

#### 3.3.3.3 关系运算符和逻辑运算符的间距

组合关系运算符和逻辑运算符时，使用以下间距约定以提高可读性：

**关系运算符（`<`、`>`、`<=`、`>=`）两侧不加空格：**
```cpp
// 推荐——关系运算符紧密绑定操作数
a<b && b<c

// 不推荐
a < b && b < c
```

**逻辑运算符（`&&`、`||`）两侧加空格：**
```cpp
// 推荐——逻辑运算符清晰连接条件
a<b && b<c

// 不推荐——更难阅读
a<b&&b<c
```

**原理：**
- 关系运算符（`<`、`>`、`<=`、`>=`）紧密绑定其操作数，反映比较优先发生
- 逻辑运算符（`&&`、`||`）连接独立条件，因此间距使逻辑结构更清晰
- 此约定使 `a<b && b<c` 这样的表达式自然读作"a 小于 b，且 b 小于 c"

**完整示例：**
```cpp
// 可读性好——关系比较分组，逻辑结构清晰
if (score>=60 && score<80) {
    cout << "Grade: B" << endl;
}
// 读作："score 大于等于 60，且 score 小于 80"

// 更多空格的替代写法（也可接受）
if (score >= 60 && score < 80) {
    cout << "Grade: B" << endl;
}
```

### 3.3.4 花括号风格

花括号 `{}` 的放置位置是编码风格问题。两种常见风格广泛使用：

| 风格 | 名称 | 描述 | 示例 |
|------|------|------|------|
| **风格 1** | Allman 风格 | 左花括号独占一行 | `if (condition) \n { \n ... \n }` |
| **风格 2** | K&R 风格 | 左花括号与语句同行 | `if (condition) { \n ... \n }` |

**风格 1（Allman 风格）——左花括号在新行：**
```cpp
if (condition)
{
    statement 1;
    statement 2;
    ...
    statement n;
}
```

**风格 2（K&R 风格）——左花括号与语句同行：**
```cpp
if (condition){
    statement 1;
    statement 2;
    ...
    statement n;
}
```

**对比：**

| 方面 | 风格 1（Allman） | 风格 2（K&R） |
| ---- | ---------------- | ------------- |
| **视觉清晰度** | 花括号垂直对齐，更容易发现块边界 | 更紧凑，在许多 C++ 项目中常见 |
| **行数** | 占用更多垂直空间 | 垂直方向上更紧凑 |
| **流行度** | Microsoft/C# 风格中常见 | C/C++ 社区常见（K&R 传统） |

> 另请参阅：[8.2.1 复合语句（块）](#721-compound-statements-blocks) 了解块的概念及其在控制语句中的用法。

## 3.4 标识符命名

### 3.4.1 强制规则（必须遵守）

这些规则由编译器强制执行。违反会导致编译错误。

#### 3.4.1.1 字符

**合法字符**

| 位置 | 允许 | 不允许 |
|------|------|--------|
| **首字符** | 字母（`a-z`、`A-Z`）、下划线（`_`） | 数字（`0-9`）、特殊字符（`@`、`$` 等） |
| **后续字符** | 字母、数字（`0-9`）、下划线（`_`） | 空格、连字符（`-`）、标点符号、特殊字符 |

**示例：**

| ✓有效 | ✗无效 | 无效原因 |
|------|------|----------|
| `distance` | `1x` | 以数字开头 |
| `x_1` | `my name` | 包含空格 |
| `_value` | `my-name` | 包含连字符 |
| `studentCount` | `$sum` | 包含 `$` |

**其他禁止字符（按类别）：**

| 类别 | 字符 | 用途 |
|------|------|------|
| **标点符号** | `,` `.` `;` `:` `'` `"` | 分隔符、成员访问、语句 |
| **括号** | `()` `[]` `{}` | 函数调用、数组、代码块 |
| **算术运算符** | `+` `-` `*` `/` | 数学运算 |
| **比较运算符** | `<` `>` `=` `!` | 布尔表达式 |
| **逻辑/位运算符** | `&` `|` `^` `~` | 逻辑和位运算 |
| **其他** | `?` `#` `\` `$` | 三目运算、预处理器、转义 |

> **核心原则**：仅允许 `[a-zA-Z0-9_]`。如有疑问，只使用字母、数字和下划线。

#### 3.4.1.2 大小写敏感

C++ 区分大小写字母。

| 标识符 | 它们相同吗？ |
|--------|-------------|
| `myVariable` 与 `myvariable` | ✗不同 |
| `myVariable` 与 `MYVARIABLE` | ✗不同 |
| `count` 与 `Count` | ✗不同 |

#### 3.4.1.3 保留字

**C++ 关键字（禁止用作标识符）：**

| 类别 | 关键字 |
|------|--------|
| **类型** | `int`, `char`, `double`, `float`, `bool`, `void`, `auto`, `short`, `long`, `signed`, `unsigned` |
| **字符类型** | `wchar_t`, `char8_t`, `char16_t`, `char32_t` |
| **类型修饰符** | `const`, `volatile`, `mutable`, `inline`, `explicit`, `extern`, `register` |
| **控制流** | `if`, `else`, `for`, `while`, `do`, `switch`, `case`, `default`, `break`, `continue`, `return`, `goto` |
| **类与面向对象** | `class`, `struct`, `union`, `enum`, `public`, `private`, `protected`, `virtual`, `template`, `this`, `friend`, `final`, `override` |
| **内存** | `static`, `sizeof`, `new`, `delete`, `alignas`, `alignof`, `thread_local` |
| **类型转换** | `const_cast`, `static_cast`, `dynamic_cast`, `reinterpret_cast` |
| **异常处理** | `try`, `catch`, `throw`, `noexcept` |
| **类型信息** | `typeid`, `typename`, `typedef`, `decltype` |
| **布尔与空值** | `true`, `false`, `nullptr` |
| **命名空间** | `namespace`, `using` |
| **运算符替代写法** | `and`, `and_eq`, `bitand`, `bitor`, `compl`, `not`, `not_eq`, `or`, `or_eq`, `xor`, `xor_eq` |
| **C++20 特性** | `concept`, `consteval`, `constexpr`, `constinit`, `co_await`, `co_return`, `co_yield`, `export`, `import`, `module`, `requires`, `static_assert` |
| **其他** | `asm`, `operator` |

> **提示：** 大多数 IDE 会用不同颜色（通常是蓝色或紫色）高亮关键字，使它们容易识别。

**保留标识符（避免使用）：**

| 模式 | 示例 | 位置 |
|------|------|------|
| 双下划线（`__*`） | `__internal` | 任何地方 |
| 下划线 + 大写字母（`_[A-Z]*`） | `_GlobalVar` | 全局命名空间 |

### 3.4.2 最佳实践与命名约定

#### 3.4.2.1 常见命名风格

| 风格 | 模式 | 用途 | 示例 |
| ---- | ---- | ---- | ---- |
| **小驼峰式** | `myVariableName` | 变量、函数 | `studentName`, `totalScore` |
| **大驼峰式 / PascalCase** | `MyClassName` | 类、结构体、枚举 | `StudentInfo`, `MyClass` |
| **蛇形命名法** | `my_variable_name` | 变量、常量（替代方案） | `student_name`, `total_score` |
| **全大写** | `MY_CONSTANT` | 常量、宏 | `MAX_SIZE`, `PI` |
| **匈牙利命名法** | `iCount`, `pData` | ✗现代 C++ 中**已弃用** | 不推荐 |

#### 3.4.2.2 按标识符类型的命名约定

| 标识符类型 | 约定 | 示例 |
|-----------|------|------|
| **类/结构体类型** | 大驼峰式 | `class StudentRecord`, `struct Point` |
| **变量与对象** | 小驼峰式或蛇形命名法 | `playerHealth` 或 `player_health` |
| **函数** | 小驼峰式或蛇形命名法 | `getValue()`, `calculate_average()` |
| **常量** | 全大写 + 蛇形命名法 | `MAX_CONNECTIONS`, `PI` |
| **私有成员变量** | 尾部下划线或 `m_` 前缀 | `int count_;` 或 `int m_count;` |
| **宏** | 全大写（谨慎使用） | `#define DEBUG_MODE` |

**建议：**
- 使用**描述性名称**：`studentCount` 比 `sc` 或 `n` 更好
- 避免单字母名称，除非用于循环计数器（`i`、`j`、`k`）
- 在整个项目中保持一种风格的一致性
- **遵循团队现有约定**——一致性比任何单一风格都重要
