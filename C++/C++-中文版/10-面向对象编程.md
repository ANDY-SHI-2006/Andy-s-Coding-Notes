# 10 面向对象编程

## 10.1 类与对象

**类与结构的区别：**
- 类是一种用户自定义的数据类型，封装了数据与函数
- 类似于 C 语言中的结构体声明与结构体变量
- 类的变量称为**对象**

**关键术语：**
- **属性/成员数据**：类的数据成员
- **方法/成员函数**：操作对象中数据的函数

**示例 —— 银行账户：**
```cpp
class BankAcct {
private:
    int acctNum;
    double balance;

public:
    BankAcct(int num, double amt);  // 构造函数
    void deposit(double amount);
    int withdraw(double amount);
};
```

**创建对象：**
```cpp
BankAcct ba1(1234, 500.50);  // 使用构造函数创建对象
BankAcct ba2(9999, 1001.40);
```

## 10.2 封装与访问说明符

**访问级别：**

| 说明符 | 访问权限 | 用途 |
|-----------|--------|-------|
| `public` | 任何人可访问 | 公共接口 |
| `private` | 仅在类内部可访问 | 内部数据成员 |
| `protected` | 类内部及派生类可访问 | 用于继承 |

**封装的优点：**
- 将数据及相关操作整合为单一包
- 向外部隐藏内部细节
- 保护数据免受未授权修改

**点运算符：**
```cpp
ba1.deposit(1000);      // 调用公共方法
// ba1.balance = 1000;  // 错误：私有成员
```

## 10.3 构造函数与析构函数

**构造函数：**
- 对象创建时自动调用的特殊方法
- 与类同名，无返回类型
- 可以有多个构造函数（重载）

**类型：**
1. **默认构造函数**：不接受任何参数
2. **带参数的构造函数**：接受参数

```cpp
class BankAcct {
public:
    BankAcct();                          // 默认构造函数
    BankAcct(int num);                   // 单参数
    BankAcct(int num, double amt);       // 双参数
};
```

**析构函数：**
- 对象超出作用域时自动调用
- 与类同名，前缀加 `~`，无返回类型
- 用于清理工作（例如释放资源）

```cpp
class Simple {
public:
    Simple() { cout << "Alive!" << endl; }
    ~Simple() { cout << "Dead!" << endl; }
};
```

## 10.4 `this` 指针

**作用：** 指向当前对象的指针

**用途：**
1. 当参数名与属性名相同时消除歧义
2. 返回当前对象

```cpp
class BankAcct {
private:
    int acctNum;
    double balance;

public:
    BankAcct(int acctNum, double balance) {
        this->acctNum = acctNum;      // 消除歧义
        this->balance = balance;
    }
};
```

## 10.5 继承

**概念：** 从已有类派生出新类
- **基类 / 父类 / 超类**：原始类
- **派生类 / 子类**：新类

**语法：**
```cpp
class SavingAcct : public BankAcct {
private:
    double rate;

public:
    SavingAcct(int num, double amt, double r)
        : BankAcct(num, amt), rate(r) {}

    void payInterest() {
        balance += balance * rate;  // 继承的成员
    }
};
```

**优点：**
- 代码可复用性
- 可扩展性
- 可维护性

**"is-a" 与 "has-a"：**
- 继承用于 "is-a" 关系（SavingAccount is-a BankAccount）
- 组合用于 "has-a" 关系

## 10.6 模板类

**目的：** 编写适用于任意数据类型的通用代码

**语法：**
```cpp
template <typename T>
class Pair {
private:
    T first;
    T second;

public:
    Pair(T a, T b) : first(a), second(b) {}
    T getFirst() const { return first; }
    T getSecond() const { return second; }
};
```

**用法：**
```cpp
Pair<int> intPair(1, 2);
Pair<double> doublePair(1.5, 2.5);
Pair<string> stringPair("Hello", "World");
```

**多类型参数：**
```cpp
template <typename T1, typename T2>
class Pair {
    T1 first;
    T2 second;
};
```

## 10.7 值传递与引用传递

**值传递（默认）：**
- 传递对象的副本
- 修改不影响原对象

**引用传递：**
```cpp
void transfer(BankAcct& from, BankAcct& to, double amt) {
    from.withdraw(amt);
    to.deposit(amt);
}
```

> **建议：** 对于大型对象，使用引用传递以避免复制开销



## 10.8 抽象数据类型（ADT）

抽象数据类型（ADT）是软件工程中的一个基本概念，它将**规格说明**（有哪些操作可用）与**实现**（操作如何执行）分离开来。


### 10.8.1 什么是抽象？

**抽象**是隐藏实现细节、仅暴露必要特性的过程。

**抽象的类型：**

| 类型 | 描述 |
|------|-------------|
| **数据抽象** | 隐藏数据表示，仅暴露必要操作 |
| **功能抽象** | 隐藏实现逻辑，暴露函数接口 |


### 10.8.2 ADT 的定义

**ADT = 数据 + 操作**

ADT 是一组数据以及定义在该数据上的一组操作的集合。

**关键特性：**

1. **规格说明** —— 接口：有哪些操作可用
2. **实现** —— 内部细节：数据结构与算法

**规格说明与实现相互独立：**
- 一份规格说明可以有多种实现
- 用户仅依赖于规格说明
- 实现的变更不会影响用户


### 10.8.3 抽象之墙

```
     用户程序
          │
          │ 使用
          ▼
    ┌─────────────┐
    │   ADT       │  ← 规格说明（接口）
    │  操作       │    - 方法签名
    │  （公共）   │    - 前置/后置条件
    └──────┬──────┘
           │
    ───────┼───────  ← 抽象之墙
           │
    ┌──────▼──────┐
    │  实现       │  ← 隐藏细节
    │  - 数据结构
    │  - 算法     │
    │  - 私有     │
    └─────────────┘
```

**规则：**
- 用户只能通过规定的操作进行交互
- 用户不应直接访问底层数据结构
- 实现可以变更而不会影响用户程序


### 10.8.4 ADT 的优点

| 优点 | 描述 |
|---------|-------------|
| **封装** | 数据与操作捆绑在一起 |
| **信息隐藏** | 内部细节对用户隐藏 |
| **模块化** | 接口与实现清晰分离 |
| **可维护性** | 变更仅局限于实现层 |
| **灵活性** | 允许多种实现并存 |
| **复杂度管理** | 将复杂系统拆分为可管理的单元 |


### 10.8.5 C++ 中的 ADT

在 C++ 中，**`class`** 结构是实现 ADT 的主要方式。

**组成：**

| C++ 特性 | ADT 概念 |
|-------------|-------------|
| `private` 成员 | 隐藏的数据/实现 |
| `public` 方法 | 规定的操作（接口） |
| 头文件（.hpp） | 规格说明 |
| 源文件（.cpp） | 实现 |

**示例：复数 ADT**

**规格说明（Complex.hpp）：**
```cpp
#pragma once
#include <string>

class Complex {
private:
    double real;      // 隐藏的实现细节
    double imag;

public:
    // 构造函数
    Complex(double r = 0, double i = 0);
    
    // 访问器（Getter）
    double getReal() const;
    double getImag() const;
    
    // 操作
    Complex add(const Complex& other) const;
    Complex subtract(const Complex& other) const;
    Complex multiply(const Complex& other) const;
    
    // 工具方法
    std::string toString() const;
};
```

**实现（Complex.cpp）：**
```cpp
#include "Complex.hpp"
#include <sstream>
#include <iomanip>

Complex::Complex(double r, double i) : real(r), imag(i) {}

double Complex::getReal() const { return real; }
double Complex::getImag() const { return imag; }

Complex Complex::add(const Complex& other) const {
    return Complex(real + other.real, imag + other.imag);
}

Complex Complex::subtract(const Complex& other) const {
    return Complex(real - other.real, imag - other.imag);
}

Complex Complex::multiply(const Complex& other) const {
    return Complex(
        real * other.real - imag * other.imag,
        real * other.imag + imag * other.real
    );
}

std::string Complex::toString() const {
    std::ostringstream oss;
    oss << std::fixed << std::setprecision(2);
    oss << real << (imag >= 0 ? " + " : " - ") << std::abs(imag) << "i";
    return oss.str();
}
```

**用户程序：**
```cpp
#include "Complex.hpp"
#include <iostream>

int main() {
    Complex c1(3, 4);   // 3 + 4i
    Complex c2(1, 2);   // 1 + 2i
    
    Complex c3 = c1.add(c2);
    
    std::cout << c3.toString() << std::endl;  // "4.00 + 6.00i"
    
    // 用户不能直接访问 real/imag：
    // c3.real = 5;  // ❌ 编译错误：私有成员
    
    return 0;
}
```


### 10.8.6 前置条件与后置条件

良好的 ADT 文档应包含：

| 条件 | 描述 |
|-----------|-------------|
| **前置条件** | 调用操作前必须为真的条件 |
| **后置条件** | 操作完成后为真的条件 |

**示例：**
```cpp
class List {
public:
    // 前置条件：0 <= index < size()
    // 后置条件：返回索引处的元素
    int get(int index) const;
    
    // 前置条件：列表未满
    // 后置条件：元素添加到末尾，大小增加 1
    void append(int value);
};
```


### 10.8.7 基本类型作为 ADT

即使是内置类型也是 ADT：

| 类型 | 隐藏表示 | 操作 |
|------|----------------------|------------|
| `int` | 平台相关（例如 32 位补码） | `+`, `-`, `*`, `/`, `%` |
| `float` | IEEE 754 标准 | 算术、比较 |
| `bool` | 由实现定义 | 逻辑运算 |

用户无需了解内部表示即可有效使用这些类型。


### 10.8.8 何时使用 ADT

**在以下情况下使用 ADT：**
1. 操作语言不直接支持的数据
2. 需要隐藏复杂的实现细节
3. 希望允许多种实现
4. 构建可复用组件
5. 管理软件复杂度

**示例：**
- 复数
- 银行账户
- 几何形体（球体、立方体）
- 数据结构（列表、栈、队列、树）


### 10.8.9 小结

**ADT 设计步骤：**

1. **确定要管理的数据**
2. **设计所需的操作**（接口）
3. **编写规格说明**（头文件）
4. **实现**（源文件）
5. **在程序中使用**（仅通过公共接口）

**牢记：**
- ADT = 数据 + 操作
- 规格说明 ≠ 实现
- 用户仅依赖于规格说明
- 实现可以变更而不会影响用户




### 10.8.10 列表 ADT 示例

列表 ADT 是一种基本的抽象数据类型，表示元素的有序集合。它展示了完整的 ADT 设计过程：先规格说明，再实现。


#### 10.8.10.1 列表 ADT 规格说明

列表是元素的有序集合，每个元素都有一个位置（索引）。列表在计算中无处不在：学生名单、事件列表、预约列表等。

**操作：**

| 操作 | 描述 | 前置条件 | 后置条件 |
|-----------|-------------|--------------|---------------|
| `isEmpty()` | 检查列表是否为空 | 无 | 若列表无元素则返回 true |
| `getLength()` | 获取元素数量 | 无 | 返回当前元素个数 |
| `insert(index, item)` | 在指定位置插入元素 | `0 <= index <= length` | 元素已插入，后续元素后移 |
| `remove(index)` | 移除指定位置的元素 | `0 <= index < length` | 元素已移除，后续元素前移 |
| `retrieve(index)` | 获取指定位置的元素 | `0 <= index < length` | 返回索引处的元素 |
| `replace(index, item)` | 替换指定位置的元素 | `0 <= index < length` | 元素已替换 |

**C++ 规格说明（版本 1 —— 使用布尔状态）：**

```cpp
#pragma once
#include <stdexcept>

const int MAXSIZE = 100;  // 列表最大容量

typedef int ItemType;     // 存储元素的类型

class List {
private:
    ItemType items[MAXSIZE];  // 存储元素的数组
    int size;                  // 当前元素数量

public:
    List() : size(0) {}       // 构造函数：空列表
    
    // 检查列表是否为空
    bool isEmpty() const { return size == 0; }
    
    // 获取当前长度
    int getLength() const { return size; }
    
    // 在指定位置插入元素（返回成功/失败）
    bool insert(int index, ItemType item);
    
    // 移除指定位置的元素（返回成功/失败）
    bool remove(int index);
    
    // 获取指定位置的元素
    ItemType retrieve(int index) const;
    
    // 替换指定位置的元素（返回成功/失败）
    bool replace(int index, ItemType item);
};
```


#### 10.8.10.2 基于异常的规格说明（版本 2）

**版本 1 的问题：** 布尔返回值可能被粗心的程序员忽略，导致难以调试的运行时错误。

**解决方案：** 使用异常强制错误处理。

```cpp
#pragma once
#include <stdexcept>
#include <string>

const int MAXSIZE = 100;
typedef int ItemType;

// 异常类
class ListException : public std::exception {
private:
    std::string msg;
public:
    ListException(const std::string& m) : msg(m) {}
    const char* what() const noexcept override { return msg.c_str(); }
};

class ListIndexException : public ListException {
public:
    ListIndexException(const std::string& m) : ListException(m) {}
};

class List {
private:
    ItemType items[MAXSIZE];
    int size;

public:
    List() : size(0) {}
    
    bool isEmpty() const { return size == 0; }
    int getLength() const { return size; }
    
    // 插入 —— 失败时抛出异常
    void insert(int index, ItemType item);
    
    // 移除 —— 失败时抛出异常
    void remove(int index);
    
    // 获取 —— 索引无效时抛出异常
    ItemType retrieve(int index) const;
    
    // 替换 —— 失败时抛出异常
    void replace(int index, ItemType item);
};
```


#### 10.8.10.3 基于数组的实现

数组是实现列表 ADT 的直接选择。

**优点：**
- 检索极快：O(1) 直接访问
- 实现简单
- 缓存友好（连续内存）

**缺点：**
- 最大容量固定
- 插入/删除需要移动元素

**实现：**

```cpp
#include "List.h"

void List::insert(int index, ItemType item) {
    // 检查前置条件
    if (size >= MAXSIZE) {
        throw ListException("List is full");
    }
    if (index < 0 || index > size) {
        throw ListIndexException("Invalid index");
    }
    
    // 移动元素以腾出空间
    for (int i = size; i > index; i--) {
        items[i] = items[i-1];
    }
    
    // 插入新元素
    items[index] = item;
    size++;
}

void List::remove(int index) {
    if (isEmpty()) {
        throw ListException("List is empty");
    }
    if (index < 0 || index >= size) {
        throw ListIndexException("Invalid index");
    }
    
    // 移动元素以填补空隙
    for (int i = index; i < size - 1; i++) {
        items[i] = items[i+1];
    }
    
    size--;
}

ItemType List::retrieve(int index) const {
    if (index < 0 || index >= size) {
        throw ListIndexException("Invalid index");
    }
    return items[index];
}

void List::replace(int index, ItemType item) {
    if (index < 0 || index >= size) {
        throw ListIndexException("Invalid index");
    }
    items[index] = item;
}
```


#### 10.8.10.4 数组实现效率分析

**时间复杂度：**

| 操作 | 最优情况 | 最坏情况 | 平均情况 |
|-----------|-----------|------------|---------|
| `retrieve` | O(1) | O(1) | O(1) |
| `insert` | O(1)（在末尾） | O(n)（在开头） | O(n) |
| `remove` | O(1)（在末尾） | O(n)（在开头） | O(n) |
| `isEmpty` | O(1) | O(1) | O(1) |
| `getLength` | O(1) | O(1) | O(1) |

**空间复杂度：**
- 固定分配：无论实际使用情况如何，均为 O(MAXSIZE)
- 若列表很小，则浪费空间
- 若列表超过 MAXSIZE，则空间不足


#### 10.8.10.5 示例用户程序

```cpp
#include <iostream>
#include "List.h"
using namespace std;

int main() {
    try {
        List myList;
        
        // 插入元素
        myList.insert(0, 10);  // 列表：[10]
        myList.insert(1, 20);  // 列表：[10, 20]
        myList.insert(0, 5);   // 列表：[5, 10, 20]
        
        cout << "List length: " << myList.getLength() << endl;  // 3
        cout << "Element at 1: " << myList.retrieve(1) << endl; // 10
        
        // 移除元素
        myList.remove(0);  // 列表：[10, 20]
        
        // 这将抛出异常
        cout << myList.retrieve(10) << endl;  // 无效索引！
        
    } catch (ListIndexException& e) {
        cerr << "Index error: " << e.what() << endl;
    } catch (ListException& e) {
        cerr << "List error: " << e.what() << endl;
    }
    
    return 0;
}
```


#### 10.8.10.6 何时使用数组实现

**适用于：**
- 固定大小的集合
- 插入/删除操作较少的场景
- 需要快速随机访问的场景

**不适用于：**
- 大小可变的集合
- 在任意位置频繁插入/删除的场景
- 最大容量未知的场景

> **注意：** 对于需要频繁修改的动态集合，链表实现更为合适。请参阅数据结构课程了解链表实现。


#### 10.8.10.7 小结

此列表 ADT 示例展示了：

1. **完整的 ADT 设计过程：**
   - 规格说明（操作、前置/后置条件）
   - 实现（基于数组）
   - 使用（客户端代码）

2. **异常处理：**
   - 强制程序员处理错误
   - 针对不同错误使用不同类型的异常

3. **实现权衡：**
   - 数组：访问快、修改慢、大小固定
   - 未来：链表以访问速度为代价换取修改速度


#### 10.8.10.8 基于链表的实现

基于指针的链表允许元素在内存中不连续存储。元素通过指针与其相邻元素关联，从而保持有序。

##### 节点结构

单向链表中的单个节点包含数据元素和指向下一个节点的指针。

```cpp
typedef int ListItemType;

struct ListNode {
    ListItemType item;    // 数据元素
    ListNode *next;       // 指向下一个节点的指针
};
```

> **要点：** C++ 允许直接使用结构体名称（`ListNode`），无需加 `struct` 关键字。

##### 链表示例

四个元素 `<a0, a1, a2, a3>` 的列表表示为：

```
head --> [a0|*] --> [a1|*] --> [a2|*] --> [a3|NULL]
```

**要求：**
- `head` 指针指向第一个节点
- 最后一个节点的 `next` 指针为 `NULL`

##### 构建链表

```cpp
ListNode* ptr4 = new ListNode;
ptr4->element = a3;
ptr4->next = NULL;

ListNode* ptr3 = new ListNode;
ptr3->element = a2;
ptr3->next = ptr4;

ListNode* ptr2 = new ListNode;
ptr2->element = a1;
ptr2->next = ptr3;

ListNode* head = new ListNode;
head->element = a0;
head->next = ptr2;
```

> **问题：** 链表构建完成后，是否还需要 `ptr2`、`ptr3`、`ptr4`？
> **答案：** 不需要。只需 `head` 指针即可访问整个链表。

##### 链表插入

**一般情况：** 在 `prev` 和 `cur` 之间插入新节点。

```cpp
// 步骤 1：将新节点链接到 cur
newPtr->next = cur;           // 或：newPtr->next = prev->next;

// 步骤 2：将 prev 链接到新节点
prev->next = newPtr;
```

> **问题：** 是否可以在没有 `cur` 指针的情况下完成插入？
> **答案：** 可以。`prev->next` 即可访问下一个节点。

**在表头插入：**

```cpp
// 步骤 1：将新节点链接到当前表头
newPtr->next = cur;           // 或：newPtr->next = head;

// 步骤 2：更新 head
head = newPtr;
```

##### 链表删除

**一般情况：** 删除 `cur` 指向的节点，其中 `prev` 指向其前一个节点。

```cpp
// 步骤 1：绕过 cur
prev->next = cur->next;

// 步骤 2：释放内存
delete cur;
cur = NULL;
```

**删除表头：**

```cpp
// 步骤 1：将 head 前移
head = cur->next;             // 或：head = head->next;

// 步骤 2：释放内存
delete cur;
cur = NULL;
```

##### 遍历链表

向前移动一个节点：

```cpp
// 步骤 1
prev = cur;

// 步骤 2
cur = cur->next;
```

**按索引遍历：**

```cpp
ListNode* find(int index) const
/* 目的：返回指向列表中第 Index 个节点的指针 */
{
    ListNode *prev, *cur;

    prev = NULL;
    cur = 列表头部;

    // 确保 1 <= index <= 列表长度

    for (int I = 1; I < index; I++) {
        prev = cur;
        cur = cur->next;
    }

    return cur;
}
```

> **问题：** 这种情况下是否需要 `prev`？
> **答案：** 检索时不需要，但插入和删除时需要。

**按值遍历：**

```cpp
ListNode* findValue(ListItemType value) const
/* 目的：返回指向列表中具有指定值的节点的指针 */
{
    ListNode *prev, *cur;

    prev = NULL;
    cur = 列表头部;

    while (未找到 && cur 不为 NULL) {
        if (cur->element == value) {
            found = true;
        } else {
            prev = cur;
            cur = cur->next;
        }
    }

    return cur;
}
```

##### C++ 规格说明（链表）

```cpp
// ListP.h：基于链表的列表 ADT
typedef int ListItemType;

class List {
public:
    List();
    ~List();                  // 声明析构函数

    // ...其他方法不变
    bool isEmpty() const;
    int getLength() const;

    void insert(int index, const ListItemType& newItem)
        throw(ListException, ListIndexOutOfRangeException);

    void remove(int index)
        throw(ListIndexOutOfRangeException);

    void retrieve(int index, ListItemType& dataItem) const
        throw(ListIndexOutOfRangeException);

private:
    struct ListNode {
        ListItemType item;
        ListNode *next;
    };

    int size;                 // 元素数量
    ListNode *head;           // 指向链表的指针
    ListNode* find(int index) const;  // 定位节点的私有方法
};
```

> **要点：**
> - 结构体声明可以是私有的
> - `find()` 是私有辅助方法

##### 实现

**构造函数与析构函数：**

```cpp
List::List() : size(0), head(NULL) {}

List::~List()
{
    while (!isEmpty())
        remove(1);
} // 析构函数结束

bool List::isEmpty() const
{
    return size == 0;
} // isEmpty 结束

int List::getLength() const
{
    return size;
} // getLength 结束
```

**私有 `find` 方法：**

```cpp
List::ListNode *List::find(int index) const
{
    if ((index < 1) || (index > getLength()))
        return NULL;
    else  // 从列表开头计数
    {
        ListNode *cur = head;
        for (int skip = 1; skip < index; ++skip)
            cur = cur->next;
        return cur;
    }
} // find 结束
```

**`retrieve` 方法：**

```cpp
void List::retrieve(int index, ListItemType& dataItem) const
    throw(ListIndexOutOfRangeException)
{
    if ((index < 1) || (index > getLength()))
        throw ListIndexOutOfRangeException("bad index in retrieve");
    else
    {
        // 获取节点指针，再获取节点中的数据
        ListNode *cur = find(index);
        dataItem = cur->item;
    } // if 结束
} // retrieve 结束
```

**`insert` 方法：**

```cpp
void List::insert(int index, const ListItemType& newItem)
    throw(ListIndexOutOfRangeException, ListException)
{
    int newLength = getLength() + 1;

    if ((index < 1) || (index > newLength))
        throw ListIndexOutOfRangeException("Bad index in insert");
    else
    {
        // 尝试创建新节点并将 newItem 放入其中
        ListNode *newPtr = new ListNode;
        size = newLength;
        newPtr->item = newItem;

        // 将新节点附加到列表
        if (index == 1)
        {
            // 在列表开头插入新节点
            newPtr->next = head;
            head = newPtr;
        }
        else
        {
            ListNode *prev = find(index - 1);
            // 在 prev 指向的节点之后插入新节点
            newPtr->next = prev->next;
            prev->next = newPtr;
        } // if 结束
    } // if 结束
} // insert 结束
```

**`remove` 方法：**

```cpp
void List::remove(int index)
    throw(ListIndexOutOfRangeException)
{
    ListNode *cur;

    if ((index < 1) || (index > getLength()))
        throw ListIndexOutOfRangeException("Bad index in remove");
    else
    {
        --size;
        if (index == 1)
        {
            // 从列表中删除第一个节点
            cur = head;        // 保存节点指针
            head = head->next;
        }
        else
        {
            ListNode *prev = find(index - 1);
            cur = prev->next;  // 保存节点指针
            prev->next = cur->next;
        } // if 结束

        cur->next = NULL;
        delete cur;            // 释放 cur 指针指向的内存空间
        cur = NULL;
    } // if 结束
} // remove 结束
```


#### 10.8.10.9 链表的变体

上述链表实现称为**单向链表**——每个节点只有一个指针（单链接）。

还有许多其他变体：

| 变体 | 描述 |
|-----------|-------------|
| **双向链表** | 每个节点有 `prev` 和 `next` 指针 |
| **循环链表** | 最后一个节点指回第一个节点 |
| **哑头节点** | 开头有一个额外的未使用节点 |
| **带尾指针的链表** | 维护指向尾部的指针 |

某些变体可以组合：
- 循环双向链表
- 带哑头节点的循环链表

##### 双向链表

**动机：**
- 单向链表仅支持单向移动（从头部到尾部）
- 无法回到前一个节点
- 到达最后一个节点耗时最长
- 双向链表支持双向移动
- 简化了大多数操作

**节点结构：**

```cpp
struct DListNode {
    ListItemType element;
    DListNode *prev;   // 指向前一个节点的指针
    DListNode *next;   // 指向下一个节点的指针
};
```

**示例：** `<a0, a1, a2, a3>`

```
NULL <-- [a0] <--> [a1] <--> [a2] <--> [a3] --> NULL
```

**要求：**
- `head` 指针指向第一个节点
- 第一个节点的 `prev` 指针为 `NULL`
- 最后一个节点的 `next` 指针为 `NULL`

**插入（在 `cur` 之前）：**

```cpp
// 步骤 1
newPtr->next = cur;
newPtr->prev = cur->prev;

// 步骤 2
cur->prev->next = newPtr;
cur->prev = newPtr;
```

**在表头插入：**

```cpp
// 步骤 1
newPtr->next = cur;
newPtr->prev = NULL;

// 步骤 2
cur->prev = newPtr;
head = newPtr;
```

> **问题：** 如何判断 `cur == head`？
> **答案：** 当 `cur->prev == NULL` 时。

**删除：**

```cpp
// 步骤 1：绕过 cur
cur->prev->next = cur->next;
cur->next->prev = cur->prev;

// 步骤 2：释放内存
delete cur;
cur = NULL;
```

**删除表头：**

```cpp
// 步骤 1
head = cur->next;
cur->next->prev = NULL;

// 步骤 2：释放内存
delete cur;
cur = NULL;
```

**C++ 规格说明：**

```cpp
// ListDLL.h：基于双向链表的列表 ADT
typedef int ListItemType;

class List {
public:
    /* 类似于基于链表的列表 ADT */
    // ... 其他方法未展示

private:
    struct DListNode {
        ListItemType item;
        DListNode *prev;
        DListNode *next;
    };

    int size;
    DListNode *head;
    DListNode* find(int index) const;
};
```

##### 循环链表

**概念：** 单向链表的最后一个节点指回第一个节点。

```
head --> [a0] --> [a1] --> [a2] --> [a3] -----+
            ^                                  |
            +----------------------------------+
```

**性质：**
- 无需修改 `ListNode` 结构
- 链表中没有任何 `NULL`

**动机：** 适用于需要反复遍历列表的场景。

**遍历：**

```cpp
cur = head;
do {
    访问 cur 指向的节点;
    cur = cur->next;
} while (cur != head);
```

> 只要列表不为空，方案就很简单。

**追踪尾部：**

若追踪**尾部**而非头部，则更为实用：

```
[a0] <-- [a1] <-- [a2] <-- [a3] <-- tail
  |                                      ^
  +--------------------------------------+
```

- 可以轻松访问尾部和头部
- 头部 = `tail->next`

##### 哑头节点

列表开头有一个额外节点：
- 它**不**用于存储真实元素，因此称为**哑节点**
- 简化 `insert()` 和 `remove()`，不再有特殊情况

```
head --> [dummy|*] --> [a0|*] --> [a1|*] --> [a2|*] --> [a3|NULL]
```

> **要点：** 有了哑头节点，每次插入/删除都可视为一般情况处理——无需单独的表头插入或表头删除逻辑。


#### 10.8.10.10 模板列表 ADT

**动机：** 当前列表 ADT 仅支持 `int`（通过 `typedef int ListItemType;`）。要用于其他数据类型，需要修改 typedef 并重新编译。

**解决方案：** 使用**模板**使列表 ADT 独立于数据类型。

**C++ 规格说明：**

```cpp
template <typename T>
class List {
public:
    /* 不受影响的方法未展示 */
    void insert(int index, const T& newItem)
        throw(ListException, ListIndexOutOfRangeException);
    void remove(int index)
        throw(ListIndexOutOfRangeException);
    void retrieve(int index, T& dataItem) const
        throw(ListIndexOutOfRangeException);

private:
    template <typename S>
    struct ListNode {
        S item;
        ListNode *next;
    };

    int size;
    ListNode<T> *head;
    ListNode<T>* find(int index) const;
};
```

**方法的改动：**

```cpp
template <typename T>
List<T>::List() : size(0), head(NULL) {}

template <typename T>
List<T>::ListNode<T> *List<T>::find(int index) const
{
    if ((index < 1) || (index > getLength()))
        return NULL;
    else
    {
        ListNode<T> *cur = head;
        for (int skip = 1; skip < index; ++skip)
            cur = cur->next;
        return cur;
    }
}

template <typename T>
void List<T>::insert(int index, const T& newItem)
    throw(ListIndexOutOfRangeException, ListException)
{
    int newLength = getLength() + 1;

    if ((index < 1) || (index > newLength))
        throw ListIndexOutOfRangeException("Bad index in insert");
    else
    {
        ListNode<T> *newPtr = new ListNode<T>;
        size = newLength;
        newPtr->item = newItem;

        if (index == 1)
        {
            newPtr->next = head;
            head = newPtr;
        }
        else
        {
            ListNode<T> *prev = find(index - 1);
            newPtr->next = prev->next;
            prev->next = newPtr;
        }
    }
}
```

**用户程序示例：**

```cpp
#include "TListP.cpp"

int main()
{
    List<double> dl;
    double d;

    try {
        dl.insert(1, 3.14);
        dl.insert(1, 1.23);
        dl.retrieve(1, d);
        cout << d << endl;    // 1.23
        dl.retrieve(2, d);
        cout << d << endl;    // 3.14
    } catch (ListException le) {
        cout << le.what() << endl;
    } catch (ListIndexOutOfRangeException lie) {
        cout << lie.what() << endl;
    }
}
```


#### 10.8.10.11 STL 中的列表

列表是一种常用的数据结构，在 C++ 标准模板库中有标准实现。

**头文件：**

```cpp
#include <list>
```

**支持的方法**（参见 Carrano 书籍附录 E）：
- `insert(iterator i, value)`
- `front()`

**用户程序示例：**

```cpp
#include <list>

int main()
{
    list<double> dl;

    dl.insert(dl.begin(), 3.14);
    dl.insert(dl.begin(), 1.23);

    list<double>::iterator li = dl.begin();

    cout << *li << endl;    // 1.23
    li++;
    cout << *li << endl;    // 3.14

    return 0;
}
```


#### 10.8.10.12 小结

本节关于列表 ADT 实现的内容涵盖：

1. **基于链表的实现：**
   - 带指针的节点结构
   - 插入和删除操作
   - 使用 `prev` 和 `cur` 指针遍历
   - 完整的 C++ 实现（含异常处理）

2. **链表的变体：**
   - **双向链表：** 双向遍历，简化操作
   - **循环链表：** 无 NULL 终止符，适用于反复遍历
   - **哑头节点：** 消除表头插入/删除的特殊情况

3. **模板列表 ADT：**
   - 使用 `template <typename T>` 支持通用数据类型
   - 结构与链表实现类似

4. **STL 中的列表：**
   - 通过 `#include <list>` 实现标准实现
   - 使用迭代器进行遍历和插入

**列表 ADT 完整对比：**

| 方面 | 基于数组 | 基于链表 |
|--------|-------------|-------------------|
| **检索** | O(1) —— 直接访问 | O(n) —— 必须遍历 |
| **插入** | O(n) —— 必须移动 | O(1) —— 若位置已知 |
| **删除** | O(n) —— 必须移动 | O(1) —— 若位置已知 |
| **大小** | 固定（MAXSIZE） | 动态 —— 按需增长 |
| **内存** | 连续，可能浪费空间 | 非连续，额外指针开销 |

> **关键要点：** 对于需要快速随机访问和固定大小的集合，选择基于数组的实现。对于需要频繁插入和删除的动态集合，选择链表。
