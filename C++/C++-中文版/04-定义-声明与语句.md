# 4 定义、声明与语句

## 4.1 核心概念：声明与定义

理解**声明**与**定义**的区别是掌握 C++ 的基础。

| 概念 | 声明 | 定义 |
|------|------|------|
| **目的** | 向编译器告知名称和类型 | 创建实际的变量/函数，分配内存 |
| **内存** | 不分配内存 | 分配内存 |
| **次数** | 允许多次声明 | 整个程序中**只能一次**（唯一定义规则） |
| **示例（变量）** | `extern int x;` | `int x = 5;` |
| **示例（函数）** | `void foo();` | `void foo() { ... }` |

**核心洞见：** 定义也是一种声明，但声明不一定是定义。

```cpp
extern int globalVar;      // 纯声明（不分配内存）
int globalVar = 10;        // 定义（分配内存，存储值）

void func();               // 函数声明（原型）
void func() { ... }        // 函数定义（实现）
```

> 另请参阅：[8.2.5.4 外部存储（`extern`）](#8254-external-storage-extern) 了解如何使用 `extern` 跨文件共享变量。

### 4.1.1 何时使用声明，何时使用定义

**使用定义的情况：**
- 创建需要存储空间的变量
- 首次实现函数
- 在函数内部（局部变量始终是定义）

**使用声明的情况：**
- 引用在另一个文件中定义的变量
- 创建函数原型（前置声明）
- 在头文件中避免循环依赖

### 4.1.2 唯一定义规则（ODR）

C++ 强制执行**唯一定义规则**：每个变量和函数在整个程序中只能**定义一次**。允许多次声明，但多次定义会导致链接器错误。

```cpp
// file1.cpp
int shared = 100;          // 定义

// file2.cpp
int shared = 100;          // ✗错误！重定义（违反 ODR）
extern int shared;         // ✓正确！仅为声明
```

## 4.2 变量定义

变量定义创建具有分配内存的实际变量。

### 4.2.1 基本定义语法

| 语法 | 描述 |
|------|------|
| `int a;` | 无初始化的定义（值不确定） |
| `int a = 5;` | 拷贝初始化定义 |
| `int a(5);` | 直接初始化定义 |
| `int a{5};` | 列表初始化定义（C++11） |

```cpp
double x1 = 1, y1 = 5;    // 多个定义
int a = 10, b;            // a 已初始化，b 未初始化（危险！）

double side_1, side_2, distance;  // 全部未初始化
```

> **注意：** 使用未初始化变量会导致未定义行为。使用前务必初始化。

### 4.2.2 多个定义

可在一条语句中定义多个变量：

```cpp
int a = 1, b = 2, c = 3;     // 全部已初始化
int x, y, z;                  // 全部未初始化（避免这样写！）
int m = 1, n;                 // 混合（m 已初始化，n 未初始化）
```

### 4.2.3 带初始化的定义

初始化在定义时赋予初始值。C++ 提供三种初始化语法：

#### 4.2.3.1 拷贝初始化

使用 `=` 运算符复制值。

```cpp
int a = 5;           // 拷贝初始化
double b = 3.14;     // 拷贝初始化
string s = "hello";  // 拷贝初始化
```

**特性：**
- 传统、直观的语法
- 可能涉及隐式类型转换
- 对于类类型，可能调用拷贝构造函数
- **不做窄化检查**——可能静默丢失数据

```cpp
int x = 7.5;         // ⚠️ 能编译，但 x = 7（截断小数）
short s = 100000;    // ⚠️ 能编译（可能溢出，未定义行为）
```

#### 4.2.3.2 直接初始化

使用圆括号 `()` 向构造函数传递参数。

```cpp
int a(5);            // 直接初始化
double b(3.14);      // 直接初始化
string s("hello");   // 调用 string 构造函数
vector<int> v(5, 0); // 5 个元素，全部初始化为 0
```

**"最令人困惑的解析"问题：**

```cpp
int a();        // ✗声明了一个返回 int 的函数 "a"！
int b{};        // ✓正确地将 b 初始化为 0
```

#### 4.2.3.3 列表初始化（花括号初始化）

使用花括号 `{}`。在 **C++11** 中引入，也称**统一初始化**。

```cpp
int a{5};            // 花括号初始化
double b{3.14};      // 花括号初始化
string s{"hello"};   // 花括号初始化
vector<int> v{1, 2, 3};  // 用列表初始化容器
```

**优势：**

| 优势 | 描述 |
|------|------|
| **防止窄化** | 若值会丢失精度，编译器报错 |
| **统一语法** | 适用于所有类型 |
| **避免最令人困惑的解析** | 不会被误解为函数声明 |

**窄化防护：**

```cpp
int x(7.5);      // ⚠️ 能编译，但 x = 7（截断）
int y{7.5};      // ✗错误！无法将 double 转为 int

short a(100000); // ⚠️ 能编译（可能溢出）
short b{100000}; // ✗错误！值太大
```

#### 4.2.3.4 零初始化（空花括号）

```cpp
int x{};         // x = 0
double y{};      // y = 0.0
string s{};      // s = ""（空字符串）
bool flag{};     // flag = false
```

### 4.2.4 对比与推荐

| 特性 | 拷贝初始化 `=` | 直接初始化 `()` | 花括号初始化 `{}` |
|------|---------------|----------------|-----------------|
| **语法** | `int a = 5;` | `int a(5);` | `int a{5};` |
| **窄化检查** | ✗无 | ✗无 | ✓有 |
| **最令人困惑的解析** | ✓不会 | ⚠️可能 | ✓永远不会 |
| **容器初始化** | 有限 | 有限 | ✓完整支持 |

**推荐：**

| 场景 | 推荐语法 |
|------|----------|
| **基本类型** | `int x{5};` |
| **类类型** | `string s{"hi"};` |
| **容器** | `vector<int> v{1, 2, 3};` |
| **零初始化** | `int x{};` |
| **配合 `auto`** | `auto x = 5;`（避免 `auto x{5}`） |

## 4.3 语句

**语句**是执行动作的命令。与定义不同，语句不创建新变量（尽管可能修改已有变量）。

### 4.3.1 语句类型

| 类型 | 目的 | 示例 |
|------|------|------|
| **表达式语句** | 求值表达式 | `x = 5;` `cout << x;` |
| **声明语句** | 声明/定义变量 | `int x = 5;` |
| **复合语句** | 将语句分组（块） | `{ x = 1; y = 2; }` |
| **控制语句** | 改变程序流程 | `if`, `for`, `while` |
| **返回语句** | 退出函数 | `return 0;` |

### 4.3.2 声明语句与定义语句

在函数体中，"声明语句"技术上其实是**定义语句**（它们分配内存）：

```cpp
int main() {
    int x = 5;       // 声明语句（技术上是一个定义）
    double y;        // 无初始化的定义
    
    y = x * 2;       // 表达式语句（赋值）
    cout << y;       // 表达式语句（函数调用）
    
    return 0;        // 返回语句
}
```

**关键规则：** 在 C++ 中，定义必须位于块内的语句之前。

```cpp
int main() {
    x = 5;           // ✗错误！x 尚未定义
    int x;
    
    int y;
    y = 10;          // ✓正确：定义在语句之前
}
```

## 4.4 使用 auto 进行类型推导

`auto` 让编译器根据初始值推导变量类型。

**基本用法：**
```cpp
auto i = 5;        // int（auto 推荐配合拷贝初始化）
auto d = 5.0;      // double
auto f = 5.0f;     // float
auto c = 'a';      // char
auto s = "hello";  // const char*
```

**关键规则：**

| 语法 | 结果 | 说明 |
|------|------|------|
| `auto x = expr` | 值类型（拷贝） | 剥离引用和顶层 const |
| `auto& x = expr` | 引用类型 | 保留引用和 const |
| `auto* x = expr` | 指针类型 | 显式指针 |
| `const auto x = expr` | const 值 | 显式 const |

**⚠️ 花括号初始化的重要陷阱：**

```cpp
auto x{5};         // ⚠️ std::initializer_list<int>，而非 int！
auto y = 5;        // ✓int（正确）
auto z(5);         // ✓int
```

**推荐：** `auto` 始终配合拷贝初始化（`=`）使用，不要配合花括号初始化。

**常见模式：**

```cpp
// 迭代器
auto it = v.begin();

// 基于范围的 for 循环
for (const auto& elem : container)  // const 引用，避免拷贝

// Lambda 表达式
auto func = [](int x) { return x * 2; };
```

> **重要：** `auto` 需要初始化。`auto x;` 是错误。
