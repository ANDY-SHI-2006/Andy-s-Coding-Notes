# 5 运算符

C++ 中的运算符是对操作数执行运算的符号。

## 5.1 运算符分类

**按操作数数量**

运算符可根据其所需的**操作数数量**进行分类：

| 分类 | 操作数数量 | 示例 |
| ---- | ---------- | ---- |
| **一元运算符** | 1 | `++`（自增）、`--`（自减）、`!`（逻辑非）、`~`（按位非）、`*`（解引用）、`&`（取地址） |
| **二元运算符** | 2 | `+`、`-`、`*`、`/`、`%`、`==`、`!=`、`<`、`>`、`&&`、`|\|`、`=` 等 |
| **三元运算符** | 3 | `?:`（条件运算符） |

**注意：** 某些符号根据上下文既可作为一元运算符也可作为二元运算符：
- `-`：一元负号（取反）与 二元减号（减法）
- `+`：一元正号与 二元加法
- `*`：解引用与 乘法
- `&`：取地址与 按位与

**按用途**

运算符也可根据其**功能或用途**进行分类：

| 类别 | 运算符 | 描述 |
|------|--------|------|
| **算术** | `+` `-` `*` `/` `%` | 数学计算 |
| **关系** | `==` `!=` `<` `>` `<=` `>=` | 比较值，返回布尔值 |
| **逻辑** | `&&` `\|\|` `!` | 布尔逻辑运算 |
| **位运算** | `&` `\|` `^` `~` `<<` `>>` | 对单个位进行操作 |
| **赋值** | `=` `+=` `-=` `*=` `/=` `%=` 等 | 给变量赋值 |
| **自增/自减** | `++` `--` | 加 1 或减 1 |
| **条件** | `?:` | 三元条件运算符 |
| **成员访问** | `.` `->` | 访问类/结构体成员 |
| **指针** | `*` `&` `->` `[]` | 解引用、取地址、索引 |
| **作用域** | `::` | 作用域解析 |
| **其他** | `,` `sizeof` `typeid` `new` `delete` | 逗号、大小查询、内存管理 |

## 5.2 算术运算符

### 5.2.1 乘法（`*`）

乘法运算符计算两个操作数的乘积。

```cpp
int product = 5 * 3;     // 15
double area = 4.5 * 2.0; // 9.0
```

#### 5.2.1.1 要点

- 适用于整数和浮点数
- 若两个操作数均为整数，结果为整数
- 若任一操作数为浮点数，结果为浮点数
- 大整数可能溢出（无符号数回绕，有符号数未定义行为）

### 5.2.2 除法（`/`）

除法运算符计算两个操作数的商。

```cpp
int a = 7 / 2;       // 结果：3（整数除法，截断小数）
double b = 7 / 2;    // 结果：3.0（仍然是整数除法！）
double c = 7.0 / 2;  // 结果：3.5（浮点数除法）
```

| 运算 | 结果 | 说明 |
|------|------|------|
| `int / int` | `int` | 小数部分丢弃 |
| `double / int` | `double` | 正常除法 |
| `int / double` | `double` | 正常除法 |
| `double / double` | `double` | 正常除法 |

#### 5.2.2.1 整数除法的陷阱

```cpp
// 常见错误：期望整数除法得到浮点结果
int x = 5, y = 2;
double result = x / y;  // 结果：2.0，而非 2.5！

// 正确做法：
double result1 = (double)x / y;  // 强制转换一个操作数：2.5
double result2 = x / 2.0;        // 使用浮点字面量：2.5
```

#### 5.2.2.2 除以零

- 整数除以零 → 运行时错误/崩溃
- 浮点数除以零 → 返回 `inf` 或 `nan`（IEEE 754 行为）

### 5.2.3 取模（`%`）—— 求余

取模运算符返回除法的**余数**。

```cpp
int r = 7 % 3;   // 结果：1（7 = 3*2 + 1）
int s = 10 % 4;  // 结果：2（10 = 4*2 + 2）
int t = 6 % 2;   // 结果：0（整除，无余数）
```

**重要**：`%` 仅适用于**整数**。

```cpp
// double x = 7.5 % 2;  // 错误！% 要求操作数为整数
```

#### 5.2.3.1 负数取模

若任一整数为负数，`a % b` 的结果根据 C++ 标准不同而**依赖于系统**：

**C++98/03：** 由实现定义

标准**未指定**其行为。不同编译器可能产生不同结果。

```cpp
// 结果可能因编译器而异！
-5 % 3;  // 可能是 -2 或 1，取决于编译器
```

**C++11+：** 向零截断

自 C++11 起，标准统一规定整数除法**向零截断**，由此确定取模结果。

```cpp
// 所有符合 C++11 的编译器产生相同结果
-5 % 3;   // 结果：-2（保证）
5 % -3;   // 结果：2（保证）
-5 % -3;  // 结果：-2（保证）
```

#### 5.2.3.2 关键特性

- 商 `a / b` 向零截断（小数部分丢弃）
- 余数的符号与被除数（分子）相同
- 恒等式 `(a/b)*b + a%b == a` 始终成立

### 5.2.4 幂运算

**重要：** C++ **没有**幂运算符。

> **警告：** `^` 符号是**按位异或运算符**，不是幂运算！

```cpp
int result = 2 ^ 3;  // 结果：1（XOR），而非 8！
```

| 表达式 | C++ | 说明 |
|--------|-----|------|
| x⁴ | 无 `^` 或 `**` | 不同于 Python（`**`）或数学符号 |
| a² | `a * a` | 使用重复乘法 |

**方法 1：重复乘法（适用于小指数）**

```cpp
int square = a * a;           // a²
int cube = a * a * a;         // a³
int fourth = a * a * a * a;   // a⁴
```

**方法 2：`pow()` 函数（适用于分数/变量指数）**

```cpp
#include <cmath>

double result1 = pow(x, 4);      // x⁴
double result2 = pow(2.0, 10);   // 2¹⁰
double result3 = pow(x, 0.5);    // √x
double result4 = pow(x, -1);     // 1/x
```

| 方法 | 速度 | 使用场景 |
|------|------|----------|
| `a * a` | 最快 | 小固定指数：a²、a³ |
| `pow(a, b)` | 较慢 | 变量/分数指数：aᵇ、√a |

## 5.3 自增和自减运算符

`++`（自增）和 `--`（自减）运算符将变量增加或减少 1。

### 5.3.1 限制

- 只能用于**变量**（不能用于常量或表达式）
- 示例：`++count` 有效，但 `++5` 或 `++(a+b)` 无效

### 5.3.2 两种形式

| 形式 | 语法 | 描述 |
| ---- | ---- | ---- |
| **前缀** | `++x` 或 `--x` | 先自增/自减，再使用新值 |
| **后缀** | `x++` 或 `x--` | 先使用当前值，再自增/自减 |

### 5.3.3 独立使用

单独使用时（不在表达式中），两种形式等价：
```cpp
x++;    // 等价于：x = x + 1;
++x;    // 等价于：x = x + 1;
y--;    // 等价于：y = y - 1;
```

### 5.3.4 在表达式中使用

| 表达式 | 等价于 | 结果（若 x=5, y=3） |
|--------|--------|-------------------|
| `w = ++x - y;` | `x = x + 1; w = x - y;` | x=6, w=3 |
| `w = x++ - y;` | `w = x - y; x = x + 1;` | w=2, x=6 |

> **关键规则：**
> - 前缀：先修改，再使用
> - 后缀：先使用，再修改

## 5.4 关系运算符

关系运算符比较两个值，返回布尔结果（`true` 或 `false`）。

| 运算符 | 名称 | 示例 | 结果 |
|--------|------|------|------|
| `==` | 等于 | `5 == 5` | `true` |
| `!=` | 不等于 | `5 != 3` | `true` |
| `<` | 小于 | `3 < 5` | `true` |
| `>` | 大于 | `5 > 3` | `true` |
| `<=` | 小于等于 | `5 <= 5` | `true` |
| `>=` | 大于等于 | `5 >= 3` | `true` |

### 5.4.1 浮点数比较

由于精度误差，直接用浮点数进行相等比较可能有问题。

```cpp
double a = 0.1 + 0.2;
// a == 0.3 可能为 false，因为浮点数精度问题

// 更好的做法：检查是否足够接近
const double EPSILON = 1e-9;
bool equal = fabs(a - 0.3) < EPSILON;
```

## 5.5 逻辑运算符

逻辑运算符执行布尔运算，返回 `true` 或 `false`。

### 5.5.1 逻辑非（`!`）

| 属性 | 描述 |
|------|------|
| 名称 | 逻辑非 |
| 描述 | 反转布尔值 |

**语法：**
```cpp
!expression
```

**示例：**
```cpp
bool a = true;
bool b = !a;      // b = false
bool c = !false;  // c = true
```

### 5.5.2 逻辑与（`&&`）

| 属性 | 描述 |
|------|------|
| 名称 | 逻辑与 |
| 描述 | 两个操作数均为真时才为真 |

**语法：**
```cpp
expr1 && expr2
```

**示例：**
```cpp
bool a = true, b = false;
bool c = a && b;  // c = false
bool d = a && true;  // d = true
```

### 5.5.3 逻辑或（`||`）

| 属性 | 描述 |
|------|------|
| 名称 | 逻辑或 |
| 描述 | 至少一个操作数为真时才为真 |

**语法：**
```cpp
expr1 || expr2
```

**示例：**
```cpp
bool a = true, b = false;
bool c = a || b;   // c = true
bool d = false || false;  // d = false
```

### 5.5.4 真值表

| A | B | !A | A && B | A \|\| B |
|---|---|----|--------|----------|
| true | true | false | true | true |
| true | false | false | false | true |
| false | true | true | false | true |
| false | false | true | false | false |

**示例：**
```cpp
// 短路求值
int a = 0, b = 5;
if (a != 0 && b / a > 2) {  // b/a 不会被求值，避免了除以零
    // ...
}

// 组合逻辑运算符
bool x = true, y = false, z = true;
bool result = !(x && y) || z;  // result = true
```

## 5.6 位运算符

位运算符对整数值的单个位执行操作。

### 5.6.1 按位与（`&`）

| 属性 | 描述 |
|------|------|
| 名称 | 按位与 |
| 描述 | 两位均为 1 时结果为 1 |

**语法：**
```cpp
expr1 & expr2
```

**示例：**
```cpp
int a = 5;   // 二进制 0101
int b = 3;   // 二进制 0011
int c = a & b;  // c = 1（二进制 0001）
```

### 5.6.2 按位或（`|`）

| 属性 | 描述 |
|------|------|
| 名称 | 按位或 |
| 描述 | 至少一位为 1 时结果为 1 |

**语法：**
```cpp
expr1 | expr2
```

**示例：**
```cpp
int a = 5;   // 二进制 0101
int b = 3;   // 二进制 0011
int c = a | b;  // c = 7（二进制 0111）
```

### 5.6.3 按位异或（`^`）

| 属性 | 描述 |
|------|------|
| 名称 | 按位异或 |
| 描述 | 两位不同时结果为 1 |

**语法：**
```cpp
expr1 ^ expr2
```

**示例：**
```cpp
int a = 5;   // 二进制 0101
int b = 3;   // 二进制 0011
int c = a ^ b;  // c = 6（二进制 0110）
```

### 5.6.4 按位非（`~`）

| 属性 | 描述 |
|------|------|
| 名称 | 按位非 |
| 描述 | 反转所有位 |

**语法：**
```cpp
~expr
```

**示例：**
```cpp
int a = 5;    // 32 位二进制：0000...00000101
int b = ~a;   // b = -6（补码表示：1111...11111010）
```

### 5.6.5 左移（`<<`）

| 属性 | 描述 |
|------|------|
| 名称 | 左移 |
| 描述 | 将位向左移动，等价于乘以 2ⁿ |

**语法：**
```cpp
expr << n
```

**示例：**
```cpp
int a = 5;       // 二进制 0101
int b = a << 1;  // b = 10（二进制 1010）
int c = a << 2;  // c = 20（0101 → 10100）
```

### 5.6.6 右移（`>>`）

| 属性 | 描述 |
|------|------|
| 名称 | 右移 |
| 描述 | 将位向右移动，等价于除以 2ⁿ |

**语法：**
```cpp
expr >> n
```

**示例：**
```cpp
int a = 20;      // 二进制 10100
int b = a >> 1;  // b = 10（二进制 01010）
int c = a >> 2;  // c = 5（二进制 00101）
```

### 5.6.7 实际示例

**检查数字是奇数还是偶数：**
```cpp
// 检查最低有效位
if (num & 1) {
    // 奇数（最低位为 1）
} else {
    // 偶数（最低位为 0）
}
```

**设置特定位：**
```cpp
int flags = 0;
flags = flags | (1 << 2);  // 设置第 2 位
// flags 现在是 4（0b0100）
```

**清除特定位：**
```cpp
int flags = 7;              // 0b0111
flags = flags & ~(1 << 1);  // 清除第 1 位
// flags 现在是 5（0b0101）
```

**翻转特定位：**
```cpp
int flags = 5;              // 0b0101
flags = flags ^ (1 << 0);   // 翻转第 0 位
// flags 现在是 4（0b0100）
```

**位标志模式：**
```cpp
const int READ = 1 << 0;     // 0b0001 = 1
const int WRITE = 1 << 1;    // 0b0010 = 2
const int EXECUTE = 1 << 2;  // 0b0100 = 4

int permissions = READ | WRITE;  // 0b0011 = 3

// 检查是否设置了读权限
if (permissions & READ) {
    // 有读权限
}

// 添加执行权限
permissions = permissions | EXECUTE;
```

## 5.7 赋值运算符

### 5.7.1 简单赋值

赋值运算符 `=` 将一个值赋给变量。

**多重赋值：**
```cpp
x = y = z = 0;  // 先将 0 赋给 z，再给 y，最后给 x
```

**求值顺序：** 从右到左（因为 `=` 是右结合的）
```cpp
// 等价于：
z = 0;
y = z;  // y = 0
x = y;  // x = 0
```

### 5.7.2 复合赋值

C/C++ 允许使用复合赋值运算符来简写简单赋值语句。

**一般形式：**
```cpp
identifier = identifier operator expression;  // 可简写为：
identifier operator= expression;
```

| 运算符 | 示例 | 等价于 |
|--------|------|--------|
| `+=` | `x += 3;` | `x = x + 3;` |
| `-=` | `x -= 3;` | `x = x - 3;` |
| `*=` | `x *= 3;` | `x = x * 3;` |
| `/=` | `x /= 3;` | `x = x / 3;` |
| `%=` | `x %= 3;` | `x = x % 3;` |
| `&=` | `x &= 3;` | `x = x & 3;` |
| `\|=` | `x \|= 3;` | `x = x \| 3;` |
| `^=` | `x ^= 3;` | `x = x ^ 3;` |
| `<<=` | `x <<= 3;` | `x = x << 3;` |
| `>>=` | `x >>= 3;` | `x = x >> 3;` |

**优势：**
- 更简短、更简洁
- 可能更高效（变量只被求值一次）
- 实践中常用

**位运算复合赋值示例：**

```cpp
// 设置位（使用或）
int flags = 0b0000;
flags |= (1 << 2);      // 设置第 2 位：flags = 0b0100（4）

// 清除位（使用与非）
flags = 0b0111;
flags &= ~(1 << 1);     // 清除第 1 位：flags = 0b0101（5）

// 翻转位（使用异或）
flags = 0b0101;
flags ^= (1 << 0);      // 翻转第 0 位：flags = 0b0100（4）

// 移位
int value = 8;          // 0b1000
value <<= 2;            // 左移：value = 32（0b100000）
value >>= 3;            // 右移：value = 4（0b100）
```

**实际应用场景——权限标志：**
```cpp
const int READ = 1 << 0;     // 0b0001 = 1
const int WRITE = 1 << 1;    // 0b0010 = 2
const int EXECUTE = 1 << 2;  // 0b0100 = 4

int permissions = 0;
permissions |= READ;         // 授予读权限
permissions |= WRITE;        // 授予写权限

// 检查权限
if (permissions & READ) {
    // 有读权限
}

// 撤销权限
permissions &= ~WRITE;       // 移除写权限
```

## 5.8 三元条件运算符

三元条件运算符 `?:` 是 C++ 中唯一的三元运算符。它为编写简单的 if-else 表达式提供了一种紧凑的方式。

> 另请参阅：[8.2.5 条件（三元）运算符](#825-the-conditional-ternary-operator) 了解在控制流上下文中的用法。

### 5.8.1 语法与用法

**语法：**
```cpp
condition ? expression_if_true : expression_if_false
```

**工作原理：**
1. 求值 `condition`
2. 若 `condition` 为真（非零），求值并返回 `expression_if_true`
3. 若 `condition` 为假（零），求值并返回 `expression_if_false`

```cpp
int max = (a > b) ? a : b;  // 若 a > b，则 max = a，否则 max = b

// 等价于：
int max;
if (a > b) {
    max = a;
} else {
    max = b;
}
```

### 5.8.2 嵌套三元运算符

三元运算符可以嵌套，但应谨慎使用，因为会降低可读性。

```cpp
// 成绩分级（避免深度嵌套三元运算）
char grade = (score >= 90) ? 'A' :
             (score >= 80) ? 'B' :
             (score >= 70) ? 'C' :
             (score >= 60) ? 'D' : 'F';
```

> **注意：** 深度嵌套的三元运算符使代码难以阅读。对于复杂条件，请改用 `if-else` 语句。

### 5.8.3 类型兼容性

两个表达式必须是兼容类型（或可隐式转换）。

```cpp
int a = 5;
double b = 3.14;

auto result = (condition) ? a : b;  // result 为 double（int 提升为 double）
```

**重要：** 三元运算符是一个**表达式**，而非语句。它返回一个值，可以在任何需要表达式的地方使用。

```cpp
// 可用于表达式中
int x = 10 + ((y > 0) ? y : 0);

// 可被赋值
((flag) ? a : b) = 100;  // 将 100 赋给 a 或 b（C++ 允许这样做）
```

### 5.8.4 结合性

若一个表达式中有多个条件运算符，它们按**从右到左**的顺序结合。

```cpp
// 此表达式：
a ? b : c ? d : e

// 等价于（右结合）：
a ? b : (c ? d : e)

// 而非：
(a ? b : c) ? d : e
```

**示例：**
```cpp
int result = 0 ? 1 : 0 ? 2 : 3;  // 求值为：0 ? 1 : (0 ? 2 : 3)
                                 // 结果：3
```

## 5.9 类型转换运算符

类型转换运算符用于显式地将值从一种数据类型转换为另一种。

### 5.9.1 使用场景

| 表达式 | 结果 | 说明 |
|--------|------|------|
| `sum / count`（均为 `int`） | `3`（若 sum=7, count=2） | 整数除法，小数丢弃 |
| `(float)sum / count` | `3.5` | `sum` 先被转换为 float |

### 5.9.2 C 风格类型转换

```cpp
float average = (float)sum / count;  // C 风格转换
int ascii = (int)'A';                // 将 char 转为 int
```

**语法：** `(target_type)expression`

**重要：** 括号**不能省略**！类型转换的优先级高于大多数运算符，但语法要求目标类型必须用括号括起来。

```cpp
// 正确：(float) 先作用于 sum，再做除法
float avg1 = (float)sum / count;   // 结果：3.5（若 sum=7, count=2）

// 错误：(float)(a / b) —— 整数除法先发生！
float avg2 = (float)(sum / count); // 结果：3.0（整数除法先截断）

// 错误：float a / b —— 语法错误！
// float avg3 = float sum / count; // 编译错误！
```

**注意：** 类型转换只影响**本次计算中使用的值**；它不会改变变量中存储的值或变量的类型。

```cpp
int sum = 7;
float avg = (float)sum / 2;  // 仅本次计算使用 7.0

// sum 仍然是值为 7 的 int
// (float)sum 是临时的，sum 本身保持不变！
```

> 另请参阅：[幂运算](#510-幂运算)

### 5.9.3 C++ 风格类型转换

C++ 提供四种类型转换运算符，以实现更安全、更明确的转换：

| 转换 | 用途 | 示例 |
| ---- | ---- | ---- |
| `static_cast` | 安全的编译期转换 | `static_cast<double>(sum) / count` |
| `dynamic_cast` | 多态类转换 | （面向对象章节） |
| `const_cast` | 添加/移除 const 限定符 | `const_cast<char*>(str)` |
| `reinterpret_cast` | 低级别位重解释 | `reinterpret_cast<int*>(ptr)` |

**对比表**

| 特性 | `static_cast` | `dynamic_cast` | `const_cast` | `reinterpret_cast` |
|------|---------------|----------------|--------------|---------------------|
| **安全性** | ✓安全 | ✓安全（运行时检查） | ⚠️谨慎使用 | ✗危险 |
| **编译期检查** | ✓是 | 部分 | ✓是 | ✗否 |
| **常见用途** | 基本类型转换 | 类继承转换 | 添加/移除 const | 低级别操作 |

**推荐：** 在 C++ 中优先使用 `static_cast` 而非 C 风格转换。

```cpp
// C++ 风格（推荐）
double avg = static_cast<double>(sum) / count;
```

## 5.10 幂运算

**重要：** C++ **没有内置运算符**用于幂运算。

> ⚠️ **警告：** `^` 符号是**按位异或运算符**，不是幂运算！

```cpp
int result = 2 ^ 3;  // 结果：1（XOR），而非 8！
```

| 表达式 | C++ | 说明 |
|--------|-----|------|
| x⁴ | ✗无 `^` 或 `**` | 不同于 Python（`**`）或数学符号 |
| a² | `a * a` | 小指数使用重复乘法 |
| aⁿ | `pow(a, n)` | 变量指数使用 <cmath> 库 |

**方法 1：重复乘法**
```cpp
int square = a * a;           // a²
int cube = a * a * a;         // a³
int fourth = a * a * a * a;   // a⁴
```

**方法 2：`pow()` 函数**
```cpp
#include <cmath>
double result = pow(x, 4);  // x⁴
```

| 方法 | 速度 | 使用场景 |
|------|------|----------|
| `a * a` | ⚡ 最快 | 小固定指数：a²、a³ |
| `pow(a, b)` | 🐢 较慢 | 变量/分数指数 |

## 5.11 运算符优先级与结合性

下表按从高到低的优先级列出本章涉及的所有运算符。

| 优先级 | 运算符 | 描述 | 结合性 |
|--------|--------|------|--------|
| 1 | `::` | 作用域解析 | 从左到右 |
| 2 | `()` `[]` `->` `.` | 括号、下标、成员访问 | 从左到右 |
| 3 | `++` `--`（后缀） | 后缀自增/自减 | 从左到右 |
| 4 | `++` `--`（前缀） `+` `-` `!` `~` `*` `&` `(type)` `sizeof` | 前缀一元运算符 | 从右到左 |
| 5 | `.*` `->*` | 成员指针 | 从左到右 |
| 6 | `*` `/` `%` | 乘法、除法、取模 | 从左到右 |
| 7 | `+` `-` | 加法、减法 | 从左到右 |
| 8 | `<<` `>>` | 位左移、位右移 | 从左到右 |
| 9 | `<` `<=` `>` `>=` | 关系运算符 | 从左到右 |
| 10 | `==` `!=` | 相等运算符 | 从左到右 |
| 11 | `&` | 按位与 | 从左到右 |
| 12 | `^` | 按位异或 | 从左到右 |
| 13 | `\|` | 按位或 | 从左到右 |
| 14 | `&&` | 逻辑与 | 从左到右 |
| 15 | `\|\|` | 逻辑或 | 从左到右 |
| 16 | `?:` | 三元条件 | 从右到左 |
| 17 | `=` `+=` `-=` `*=` `/=` `%=` `&=` `^=` `\|=` `<<=` `>>=` | 赋值运算符 | 从右到左 |
| 18 | `,` | 逗号运算符 | 从左到右 |

**关键规则：**

- **括号和作用域解析**优先级最高——用它们覆盖其他优先级规则。
- **一元运算符**（前缀 `++`、`--`、`+`、`-`、`!`、`~` 等）在算术运算符之前求值。
- **算术运算符**遵循标准顺序：`*`、`/`、`%` 在 `+`、`-` 之前。
- **位运算符**在算术运算符之后、逻辑运算符之前求值。
- **关系运算符**（`<`、`<=`、`>`、`>=`）在相等运算符（`==`、`!=`）之前求值。
- **逻辑与（`&&`）**优先级高于**逻辑或（`||`）**。
- **三元条件（`?:`）**在逻辑运算符之后求值。
- **赋值运算符**优先级最低（逗号除外），且为右结合。
- **逗号运算符**优先级最低。
- 相同优先级的运算符按结合性（从左到右或从右到左）求值。
- 括号始终可用于覆盖默认优先级并提高可读性。

**复合赋值示例：**

```cpp
// 含赋值运算符的复杂表达式
a = b += c + d;

// 求值顺序（赋值为右结合，+ 在 += 之前）：
// 第 1 步：c + d（算术 + 优先级更高）
// 第 2 步：b += (c + d)  （复合赋值）
// 第 3 步：a = b         （简单赋值）

// 等价于：
b = b + (c + d);
a = b;
```

**最佳实践：**
- 使用括号使复杂表达式更易读
- 避免在一个语句中混用多个赋值运算符
- 赋值运算符两侧加空格以提高清晰度（它们最后求值）


## 5.12 间距与样式准则

> 另请参阅：[3.3.3 表达式中的间距](#233-spacing-in-expressions) 获取一般间距指南。

运算符周围的间距是一个**风格问题**。选择一种风格并始终一致地使用。

| 风格 | 描述 | 示例 |
|------|------|------|
| **所有运算符两侧都加空格** | 有些人喜欢每个运算符两侧都加空格 | `a * b + b / c * d` |
| **仅在 + 和 - 两侧加空格** | 推荐风格：仅在二元加减（最后求值）两侧加空格 | `a*b + b/c*d` |

**要点：**
- 间距是风格选择，不是语法要求
- **推荐：** 仅在二元 `+` 和 `-` 两侧加空格，因为它们最后求值
- 这使表达式结构更清晰

```cpp
// 推荐风格——仅在 + 和 - 两侧加空格
int result = a*b + b/c*d;  // 结构更清晰

// 替代风格——所有运算符两侧都加空格
int result = a * b + b / c * d;  // 也有效，但不够清晰
```
