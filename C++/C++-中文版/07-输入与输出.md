# 7 输入与输出

C++ 提供了多种处理输入输出操作的方式。本章涵盖现代 C++ 基于流的 I/O 和传统 C 风格 I/O 函数。

## 7.1 C++ 流 I/O

现代 C++ 使用 `<iostream>` 库进行输入输出操作。流提供了类型安全、可扩展且面向对象的 I/O。


### 7.1.1 使用 `cin` 输入

使用 `cin`（字符输入）进行控制台输入。

### 7.1.1.1 基本语法

```cpp
cin >> variable;           // 读取单个变量
cin >> a >> b >> c;        // 链式输入，以空白分隔
```

### 7.1.1.2 输入分隔符

`cin` 将**空白字符**（空格、制表符、换行符）视为分隔符：

| 输入格式 | 有效？ | 变量接收值 |
|---------|--------|-----------|
| `1 2 3 4` | ✓ | a=1, b=2, c=3, d=4 |
| `1\n2 3\n4` | ✓ | 相同（换行符 = 空格） |
| `1234 56.78` 配合 `char c1,c2; int a; float b;` | ✓ | c1='1', c2='2', a=34, b=56.78 |

### 7.1.1.3 关键行为

1. **类型感知提取**：`cin` 根据变量类型提取字节
2. **多余输入被忽略**：超出变量的额外数据被丢弃
3. **自动跳过空白**：前导空格/换行符自动跳过

### 7.1.1.4 `cin` 的运算符优先级

> 另请参阅：[7.1.2.9 `cout` 的运算符优先级](#71129-operator-precedence-with-cout) 了解输出端的等价内容。

**常见错误：**

```cpp
int a, b;
cin >> a == b;  // ✗编译错误！
```

**失败原因：**
流提取运算符 `>>` 的**优先级高于**相等运算符 `==`。表达式被解析为：

```cpp
(cin >> a) == b;  // 将 istream& 与 int 比较——无效！
```

**解决方案：**
先读取，再在单独的语句中比较，或使用括号：

```cpp
// 正确做法 1：分开写
cin >> a;
if (a == b) { ... }

// 正确做法 2：先读取两个再比较
cin >> a >> b;
if (a == b) { ... }
```

**更多示例：**

```cpp
int x, y;

// 位运算符（低优先级）
cin >> x & y;       // ✗错误：(cin >> x) & y
cin >> x;
int result = x & y;  // ✓正确

// 逻辑运算符
cin >> x && cin >> y;     // ✗错误：((cin >> x) && cin) >> y
cin >> x >> y;            // ✓正确
bool result = x && y;

// 三元运算符
cin >> x > 0 ? a : b;     // ✗错误
cin >> x;                 // ✓正确
int val = (x > 0) ? a : b;
```

**运算符优先级速查：**

| 优先级 | 运算符 | 描述 |
|--------|--------|------|
| **较高** | `>>` `<<` | 流提取/插入 |
| **较低** | `==` `!=` `<` `>` `<=` `>=` | 比较运算符 |
| **较低** | `&` `^` `\|` | 位运算符 |
| **较低** | `&&` `\|\|` | 逻辑运算符 |
| **较低** | `?:` | 三元条件 |

> **最佳实践：** 当 `cin` 与任何比较或逻辑运算一起使用时，**先读取值，再在单独的语句中执行运算**。

### 7.1.2 使用 `cout` 输出

使用 `cout`（字符输出）进行控制台输出。

### 7.1.2.1 换行控制

两者都创建新行，但有关键区别：

| 语法 | 效果 | 用法 |
| ---- | ---- | ---- |
| `endl` | 换行 + **刷新缓冲区** | 需要立即输出时 |
| `'\n'` | 仅换行 | 一般使用，性能更好 |

```cpp
cout << "Hello" << endl;    // 刷新缓冲区
cout << "Hello" << '\n';    // 仅换行，更快
```

**注意**：频繁使用 `endl` 会拖慢程序。除非需要强制立即输出，否则使用 `'\n'`。

### 7.1.2.2 输出格式化

#### 7.1.2.2.1 头文件依赖

```cpp
#include <iomanip>
```

#### 7.1.2.2.2 核心机制

C++ 有三种浮点数输出格式：

| 格式 | 描述 | 示例（12345.6789） |
|------|------|-------------------|
| **默认** | 自动，可能使用科学计数法 | `12345.7` 或 `1.23457e+04` |
| `fixed` | 定点记数法 | `12345.678900` |
| `scientific` | 科学计数法 | `1.234568e+04` |

#### 7.1.2.2.3 `setprecision(n)` 的含义

`setprecision(n)` 的行为取决于当前格式：

- **默认**：`n` 位有效数字
- `fixed`/`scientific`：`n` 位小数

#### 7.1.2.2.4 详细对比与示例

```cpp
double pi = 3.1415926535;
double big = 12345678.9;

// 1. 默认格式
cout << setprecision(4);
cout << pi << endl;    // 3.142（4 位有效数字）
cout << big << endl;   // 1.235e+07（科学计数法）

// 2. 定点格式
cout << fixed << setprecision(4);
cout << pi << endl;    // 3.1416（4 位小数）
cout << big << endl;   // 12345678.9000

// 3. 科学计数法格式
cout << scientific << setprecision(4);
cout << 123.456 << endl;  // 1.2346e+02
```

#### 7.1.2.2.5 状态持久性

`fixed` 和 `setprecision` 会一直保持，直到显式更改。

```cpp
cout << fixed << setprecision(2);
cout << 1.234 << endl;  // 1.23
cout << 5.678 << endl;  // 5.68（格式仍然有效！）
```

#### 7.1.2.2.6 重置为默认

```cpp
// C++11 方法
cout << defaultfloat << setprecision(6);

// C++11 之前的方法
cout.unsetf(ios::fixed | ios::scientific);
cout << setprecision(6);
```

#### 7.1.2.2.7 `cout` 对 double 值的格式化

> 另请参阅：[1.4.3.4 幂运算 - cout 对 double 值的格式化](#1434-power-exponentiation)

**问题：**
```cpp
cout << pow(5, 2);      // 输出：25（不是 25.0！）
cout << 25.0;           // 输出：25（不是 25.0！）
```

`pow()` **确实**返回 `double`（25.0），但 `cout` 对浮点数有**默认格式化规则**。

| 格式标志 | 效果 | 默认值 |
|----------|------|-------- |
| `ios::fixed` | 固定小数位 | 未设置 |
| `ios::scientific` | 科学计数法 | 未设置 |
| `ios::showpoint` | **强制显示小数点** | **未设置** |

**默认行为：**
- 当 `showpoint` 未设置且小数部分为 `.0` 时，`cout` **省略小数点**
- 这是 C++ 的设计选择，使输出更简洁

**内存 vs 显示：**

```
内存（IEEE 754）： [01000011 01100100 0000...] = 25.0（double）
       ↓
   cout 格式化
       ↓
显示：            "25"（默认，省略 .0）
```

**如何强制显示小数：**

```cpp
#include <iomanip>  // 格式化所需

// 方法 1：强制 showpoint
cout << showpoint << 25.0;              // 输出：25.0000

// 方法 2：固定精度
cout << fixed << setprecision(2) << 25.0;  // 输出：25.00

// 方法 3：科学计数法
cout << scientific << 25.0;             // 输出：2.500000e+01
```

**验证：**

```cpp
auto result = pow(5, 2);  // auto 推导为：double
cout << typeid(result).name();  // 确认：它是 double！
```

> **关键要点：** 类型**确实**是 `double`，但 `cout` 默认不显示 `.0`。这是格式化问题，不是类型错误！

#### 7.1.2.2.8 截断浮点数输出（不四舍五入）

**问题：**
`setprecision` 和 `printf` 默认**四舍五入**：

```cpp
cout << fixed << setprecision(2) << 3.149;  // 输出：3.15（四舍五入！）
printf("%.2f", 3.149);                       // 输出：3.15（四舍五入！）
```

**解决方案：手动截断**

若想要**截断**（直接丢弃多余位数，不四舍五入），使用 `floor()` 或预先计算：

```cpp
#include <cmath>  // 使用 floor()

double value = 3.149;
int precision = 2;

// 方法 1：使用 floor()
double truncated = floor(value * 100) / 100;  // 3.14
cout << fixed << setprecision(2) << truncated; // 输出：3.14

// 方法 2：强制转换为 int（用于 2 位小数）
double truncated2 = (int)(value * 100) / 100.0;  // 3.14

// 方法 3：通用函数
double truncate(double val, int prec) {
    double multiplier = pow(10, prec);
    return floor(val * multiplier) / multiplier;
}
```

**对比：**

| 值 | 四舍五入（默认） | 截断 |
|----|----------------|------|
| 3.149 | 3.15 | 3.14 |
| 2.999 | 3.00 | 2.99 |
| -1.278 | -1.28 | -1.27 |

> **注意：** 对于负数，使用 `trunc()`（C++11）代替 `floor()` 以向零截断：

```cpp
 double truncated = trunc(value * 100) / 100;  // C++11
```

### 7.1.2.3 `cout` 的运算符优先级

> 另请参阅：[6.1.4 `cin` 的运算符优先级](#614-operator-precedence-with-cin) 了解输入端的等价内容。

**常见错误：**

```cpp
cout << 1 == 1;  // ✗编译错误！
```

**失败原因：**
流插入运算符 `<<` 的**优先级高于**相等运算符 `==`。表达式被解析为：

```cpp
(cout << 1) == 1;  // 将 ostream& 与 int 比较——无效！
```

**解决方案：**
使用**括号**确保比较先发生：

```cpp
cout << (1 == 1);  // ✓输出 "1"（true）
```

**更多示例：**

```cpp
// 位运算符（低优先级）
cout << a & b;       // ✗错误：(cout << a) & b
cout << (a & b);     // ✓正确

// 逻辑运算符
cout << x == 5 && y == 10;    // ✗错误：((cout << x) == 5) && ...
cout << (x == 5 && y == 10);  // ✓正确

// 三元运算符
cout << x > 0 ? "pos" : "neg";    // ✗错误
cout << (x > 0 ? "pos" : "neg");  // ✓正确
```

**运算符优先级速查：**

| 优先级 | 运算符 | 描述 |
|--------|--------|------|
| **较高** | `<<` `>>` | 流插入/提取 |
| **较低** | `==` `!=` `<` `>` `<=` `>=` | 比较运算符 |
| **较低** | `&` `^` `\|` | 位运算符 |
| **较低** | `&&` `\|\|` | 逻辑运算符 |
| **较低** | `?:` | 三元条件 |

> **最佳实践：** 当 `cout` 与任何比较或逻辑运算混用时，**始终使用括号**使意图明确。

### 7.1.3 字符串流

来自 `<sstream>` 的字符串流允许你将字符串当作流来处理，方便解析和格式化。

**头文件：** `#include <sstream>`

**输入字符串流：** 从字符串解析数据
```cpp
string data = "123 45.6 Hello";
istringstream iss(data);

int i;
double d;
string s;
iss >> i >> d >> s;  // i=123, d=45.6, s="Hello"
```

**输出字符串流：** 高效构建字符串
```cpp
ostringstream oss;
oss << "Name: " << name << ", Age: " << age;
string result = oss.str();
```

**使用场景：**
- 类型与字符串之间的转换
- 逐行解析复杂输入
- 构建格式化字符串

## 7.2 C 风格 I/O

来自 `<cstdio>` 的 C 风格 I/O 函数提供快速、基于格式的输入输出操作。虽然不如 C++ 流类型安全，但对于特定格式化需求和性能关键代码很有用。

### 7.2.1 使用 `scanf` 输入（C 风格输入）

C 的基于格式的输入函数。需要头文件 `<cstdio>` 或 `<stdio.h>`。

```cpp
#include <cstdio>
scanf("control string", &var1, &var2, ...);  // 注意 &（取地址运算符）
```

#### 7.2.1.1 头文件风格：C 与 C++

| 风格 | 头文件 | 用法 | 命名空间 |
|------|--------|------|----------|
| **C 风格** | `#include <stdio.h>` | C / C++ | 全局命名空间 |
| **C++ 风格** | `#include <cstdio>` | 仅 C++ | `std::` 命名空间（也可能在全局） |

> **注意**：在 C++ 代码中，优先使用 `<cstdio>` 而非 `<stdio.h>`。两者都能工作，但 `<cstdio>` 更地道。

**性能**：对于大数据量 I/O，通常比 `cin`/`cout` 更快，但类型安全性较低。

#### 7.2.1.2 基本用法

`scanf` 需要变量的**地址**来存储输入值。`&` 运算符返回变量的内存地址。

```cpp
int year;
scanf("%i", &year);  // &year = year 变量的地址
```

**常见错误：** 忘记 `&` 是 C++ 中最频繁的错误之一：

```cpp
scanf("%i", year);   // ✗错误——scanf 接收到的是值，不是地址
scanf("%i", &year);  // ✓正确——scanf 接收到的是地址
```

**为什么 `scanf` 需要 `&`：**

C/C++ 默认对函数参数使用**按值调用**。当你调用 `scanf("%i", year)` 而不带 `&` 时：
1. `year` 的**值**被复制到 `scanf` 的参数中
2. `scanf` 只能修改它的局部副本
3. 原始的 `year` 变量保持不变

通过使用 `&year`，你传递了 `year` 存储的**内存地址**。这允许 `scanf` 直接向该位置写入输入值，从而有效地修改原始变量。

```cpp
int year;
scanf("%i", &year);  // scanf 接收到地址 0x7fff...
                     // 向该地址写入输入值
                     // year 现在包含输入值
```

> **重要：** 对于数组（如 `char name[50]`），数组名本身就代表地址，因此不需要 `&`。

取地址运算符 `&` 与其他一元运算符处于同一优先级级别。若同一语句中出现多个一元运算符，它们按**从右到左**结合。

```cpp
&*ptr    // 先：*ptr（解引用），然后：&（取地址）
!~x      // 先：~x（按位非），然后：!（逻辑非）
```

#### 7.2.1.3 格式说明符语法

```
%[flags][width][length]specifier
     ↓     ↓      ↓      ↓
 可选   可选   可选   必需
```

**顺序**：`flags` → `width` → `length` → `specifier`（从左到右）

> **注意**：与 `printf` 不同，`scanf`**不支持**精度（例如 `%.2f` 无效）。

##### 1. 标志（可选）

| 标志 | 描述 | 示例 |
|------|------|------ |
| `*` | 抑制——读取但不赋值 | `%*d` 跳过一个整数 |

**抑制（`*`）**——读取但不赋值：

```cpp
int a, b;
scanf("%d%*d%d", &a, &b);  // 输入：10 20 30 → a=10, b=30（20 被跳过）
```

##### 2. 宽度（可选）

指定要读取的**最大**字符数：

```cpp
int a;
scanf("%3d", &a);      // 输入："12345" → 读取 "123"，a=123，"45" 留在缓冲区

char str[10];
scanf("%9s", str);     // 最多读取 9 个字符 + 空终止符（防止溢出）
```

| 宽度 | 描述 | 示例 |
|------|------|------ |
| `n` | 最大字段宽度 | `%3d` 最多读取 3 位数字 |

##### 3. 长度修饰符（可选）

指定接收变量的大小。**对于正确的内存访问至关重要。**

| 修饰符 | 用于 | C/C++ 类型 | 示例 |
|--------|------|-----------|------ |
| `hh` | `%d`, `%i`, `%o`, `%x`, **`%u`** | `signed/unsigned char` | `scanf("%hhd", &c);` `scanf("%hhu", &uc);` |
| `h` | `%d`, `%i`, `%o`, `%x`, `%u` | `short` / `unsigned short` | `scanf("%hd", &s);` `scanf("%hu", &us);` |
| `l` | `%d`, `%i`, `%o`, `%x`, `%u` | `long` / `unsigned long` | `scanf("%ld", &l);` `scanf("%lu", &ul);` |
| `ll` | `%d`, `%i`, `%o`, `%x`, **`%u`** | `long long` / `unsigned long long` | `scanf("%lld", &ll);` `scanf("%llu", &ull);` |
| `L` | `%f`, `%e`, `%g` | `long double` | `scanf("%Lf", &ld);` |
| `l` | `%c`, `%s` | `wchar_t` / `wchar_t*`（宽字符） | `scanf("%lc", &wc);` |

**关键区别——`printf` 与 `scanf` 中的 `double`：**

| 变量类型 | `printf` 说明符 | `scanf` 说明符 |
|----------|----------------|--------------- |
| `float` | `%f` | `%f` |
| `double` | `%f`（自动提升） | **`%lf`**（必需） |
| `long double` | `%Lf` | `%Lf` |

> **⚠️ 关键：** 对于 `scanf`，`double` **必须**使用 `%lf`，不能用 `%f`。对 `double` 使用 `%f` 会导致未定义行为。

##### 4. 转换说明符（必需）

| 说明符 | 类型 | 输入格式 | 示例输入 |
|--------|------|----------|---------- |
| **整数类型** ||||
| `%d` | `int` | 有符号十进制整数 | `123`, `-456` |
| `%i` | `int` | 自动检测：十进制/八进制/十六进制 | `123`（十进制）, `077`（八进制）, `0xff`（十六进制） |
| `%u` | `unsigned int` | 无符号十进制 | `789` |
| `%o` | `unsigned int` | 八进制（基数 8） | `377` |
| `%x` / `%X` | `unsigned int` | 十六进制（基数 16） | `ff`, `FF`, `0x1a` |
| **浮点类型** ||||
| `%f` | `float` | 十进制或科学计数法 | `3.14`, `1.5e-2` |
| `%lf` | `double` | 十进制或科学计数法 | `3.1415926535` |
| `%Lf` | `long double` | 十进制或科学计数法 | `3.1415926535L` |
| **字符/字符串类型** ||||
| `%c` | `char` | 单个字符（包括空白字符！） | `A`, ` `, `\n` |
| `%s` | `char[]` | 字符串（非空白字符序列） | `hello` |
| **扫描集（模式匹配）** ||||
| `%[chars]` | `char[]` | 只读取指定字符 | `%[abc]` 从 "abcba" 中读取 "abcb" |
| `%[^chars]` | `char[]` | 读取直到指定字符 | `%[^,]` 从 "hello,world" 中读取 "hello" |

#### 7.2.1.3.1 长度修饰符与类型匹配规则

长度修饰符、转换说明符和实际变量类型之间的关系必须**完全匹配**，以避免内存损坏。

**关键原则：** `长度修饰符` + `转换说明符` 必须匹配实际变量类型

**常见 Bug 示例：** 对 `short` 变量使用 `%d` 会导致 4 字节写入 2 字节内存空间，造成缓冲区溢出：

```cpp
short s;
scanf("%d", &s);   // ✗错误：%d 期望 int（4 字节），但 s 是 short（2 字节）
scanf("%hd", &s);  // ✓正确：%hd 匹配 short（2 字节）
```

**常见组合表：**

| 长度修饰符 | 转换说明符 | C++ 类型 | 大小 |
|-----------|-----------|----------|------ |
| （无） | `%d` | `int` | 4 字节 |
| `h` | `%hd` | `short` | 2 字节 |
| `hh` | `%hhd` | `signed char` | 1 字节 |
| `l` | `%ld` | `long` | 4/8 字节 |
| `ll` | `%lld` | `long long` | 8 字节 |
| `h` | `%hu` | `unsigned short` | 2 字节 |
| `hh` | `%hhu` | `unsigned char` | 1 字节 |
| `l` | `%lf` | `double` | 8 字节 |
| `L` | `%Lf` | `long double` | 8+ 字节 |

> **警告：** 不匹配的说明符会导致未定义行为。写入的数据可能超出变量内存边界，损坏相邻数据。

#### 7.2.1.3.2 `%c` 读取字符的特殊行为

`%c` 说明符在空白字符处理方面有独特行为：

**`scanf("%c", &c)`** —— 读取下一个字符，**包括空白字符**（空格、`\n`、`\t`）

```cpp
char c;
scanf("%c", &c);  // 若输入为 " a"，c 将是 ' '（空格）
```

**`scanf(" %c", &c)`** —— 前导空格告诉 scanf **先跳过所有前导空白字符**，然后读取字符

```cpp
char c;
scanf(" %c", &c);  // 若输入为 "  a"，c 将是 'a'（前导空格被跳过）
```

**对比：**

| 格式 | 空白字符处理 | 示例输入 | 结果 |
|------|-------------|----------|------ |
| `%c` | 读取任何字符，包括空白字符 | ` a` | `c = ' '` |
| ` %c` | 跳过前导空白字符，然后读取 | ` a` | `c = 'a'` |

> **提示：** 当你想忽略输入之间的空白字符时使用 ` %c`（带前导空格），当你需要精确捕获空白字符时使用 `%c`。


#### 7.2.1.3.3 格式字符串中的空白字符

**格式字符串中的任何空白字符匹配输入中的任意数量的空白字符。**

```cpp
scanf("%d %d", &a, &b);      // 空格匹配任何空白字符（空格、制表符、换行符）
```

**行为：**

| 格式模式 | 输入 | 结果 |
|----------|------|------ |
| `"%d %d"` | `10 20` | a=10, b=20 |
| `"%d %d"` | `10\t20` | a=10, b=20（制表符也匹配） |
| `"%d %d"` | `10\n20` | a=10, b=20（换行符也匹配） |
| `"%d %d"` | `10   20` | a=10, b=20（多个空格匹配） |

**要点：**
- 格式字符串中的单个空格 → 匹配输入中的**任意数量**的空白字符
- `%d`、`%f`、`%s` 在读取前自动跳过前导空白字符
- `%c` 是例外：它精确读取一个字符，包括空白字符

> **提示：** `%d %d` 之间的空格对大多数说明符是可选的（它们自动跳过空白字符），但包含它能提高可读性。

#### 7.2.1.4 数组与 `scanf`

> 另请参阅：[5.8.7 数组与输入](#587-arrays-and-input) 了解数组输入基础。

使用 `scanf` 读取数组数据时，理解数组如何与地址配合工作至关重要。

##### 7.2.1.4.1 核心原理

在 C/C++ 中，**数组名本身就代表第一个元素的地址**（`name` ≡ `&name[0]`）。当使用 `scanf`（它需要变量的**地址**）时，你直接传递数组名，**不加** `&` 运算符。

| 变量类型 | 地址语法 | 示例 |
|----------|----------|------ |
| 普通变量 | `&variable` | `scanf("%d", &age);` |
| 数组（整个数组） | 仅数组名 | `scanf("%s", name);` |
| 数组元素 | `&array[index]` | `scanf("%d", &arr[0]);` |

##### 7.2.1.4.2 字符数组（字符串）

```cpp
char name[50];
scanf("%s", name);  // 正确：name 本身就是地址
```

**常见错误：** 给数组名加 `&`

```cpp
char name[50];
scanf("%s", &name);     // ✗错误：&name 是"指向数组的指针"，类型不匹配
scanf("%s", name);      // ✓正确：name 是第一个元素的地址
```

**为什么错了：** `&name` 的类型是 `char (*)[50]`（指向 50 个 char 的数组的指针），而 `scanf` 期望 `char*`（指向 char 的指针）。尽管它们的数值地址相同，但类型不兼容。

##### 7.2.1.4.3 数值数组

对于数值数组，通常需要逐个读取元素：

```cpp
int numbers[5];

// 要填充整个数组，使用循环：
for (int i = 0; i < 5; i++) {
    scanf("%d", &numbers[i]);  // 单个元素需要 &
}
```

##### 7.2.1.4.4 安全考虑

**缓冲区溢出保护：**

`scanf` 不检查数组边界。字符串始终要指定最大宽度：

```cpp
char name[50];
scanf("%49s", name);  // 最多读取 49 个字符，为 '\0' 留位置
```

| 数组大小 | 安全的格式说明符 | 说明 |
|----------|----------------|------ |
| `char[50]` | `%49s` | 49 个字符 + 1 个空终止符 |
| `char[100]` | `%99s` | 99 个字符 + 1 个空终止符 |

##### 7.2.1.4.5 总结表

| 数据类型 | `scanf` 用法 | 需要 `&`？ | 示例 |
|----------|-------------|-----------|------ |
| `int` | `%d` | 是 | `scanf("%d", &x);` |
| `double` | `%lf` | 是 | `scanf("%lf", &d);` |
| `char`（单个） | `%c` | 是 | `scanf("%c", &c);` |
| `char[]`（字符串） | `%s` | **否** | `scanf("%s", str);` |
| 数组元素 | specifier | 是 | `scanf("%d", &arr[i]);` |
| 数组（第一个元素） | specifier | **否** | `scanf("%d", arr);` |

> **关键要点：** 数组名就是地址。对 `scanf` 直接使用它们，但始终要通过指定宽度限制来防止缓冲区溢出。

#### 7.2.1.5 格式示例

| 变量类型 | `scanf` 说明符 | 示例 |
|----------|---------------|------ |
| `int` | `%d` | `scanf("%d", &a);` |
| `short` | `%hd` | `scanf("%hd", &s);` |
| `long` | `%ld` | `scanf("%ld", &l);` |
| `long long` | `%lld` | `scanf("%lld", &ll);` |
| `unsigned int` | `%u` | `scanf("%u", &u);` |
| `float` | `%f` | `scanf("%f", &f);` |
| **`double`** | **`%lf`** | `scanf("%lf", &d);` |
| `long double` | `%Lf` | `scanf("%Lf", &ld);` |
| `char` | `%c` | `scanf("%c", &c);` |
| `char[]` | `%s` | `scanf("%s", str);` |

#### 7.2.1.6 关键区别与常见陷阱

**与 `cin` 对比：**

| 特性 | `scanf` | `cin` |
|------|---------|------- |
| 类型安全 | 否 | 是 |
| 性能 | 更快 | 更慢 |
| 地址运算符（`&`） | 必需 | 不需要 |
| 边界检查 | 否 | `std::string` 是安全的 |

**与 `printf` 对比：**

| 特性 | `scanf` | `printf` |
|------|---------|---------- |
| 非格式字符串 | **不显示**（仅匹配） | 按原样显示 |
| 提示信息 | 不能显示 | 可以显示 |
| 使用模式 | 先 `printf`，再 `scanf` | 直接输出 |

**⚠️ 重要：** `scanf` 不能显示提示。始终先使用 `printf`：

```cpp
printf("Enter a number: ");  // 显示提示
scanf("%d", &a);             // 读取输入
```

**常见陷阱：**

| 陷阱 | 问题 | 解决方案 |
| ---- | ---- | -------- |
| 缺少 `&` | `scanf("%d", a);` | 使用 `scanf("%d", &a);` |
| 错误的浮点说明符 | 对 `double` 使用 `scanf("%f", &d);` | 对 `double` 使用 `%lf` |
| 缓冲区溢出 | 对长输入使用 `scanf("%s", str);` | 使用带宽度的 `scanf("%99s", str);` |
| `%c` 读取空白字符 | `scanf("%c", &c);` 读取换行符 | 使用 `scanf(" %c", &c);`（空格跳过空白字符） |
| `%s` 停在空格处 | `"John Doe"` → 只读到 "John" | 使用 `scanf("%[^\n]", str);` 读取整行 |
| 格式字符串不匹配 | `scanf("a=%d", &a);` 配合输入 "10" | 输入必须精确匹配："a=10" |

**重要说明：**

1. **无精度控制**：`scanf` 不支持 `%.2f`（只有 `printf` 支持）
2. **空白作为分隔符**：`%d%d` 接受 "1 2"、"1\t2" 或 "1\n2"
3. **非格式字符必须匹配**：`scanf("(%d)", &x)` 要求输入 "(42)"，而非仅 "42"

   格式字符串中的非格式字符（如括号、逗号等）必须在输入中**精确匹配**。

   ```cpp
   scanf("(%d)", &x);           // 输入必须包含括号：(42)
   scanf("a=%d,b=%d", &a, &b);  // 输入必须精确为：a=10,b=20
   scanf("%d %d", &a, &b);      // 空格匹配任何空白字符（空格、制表符、换行符）
   ```

| 格式字符串 | 匹配输入 | 不匹配输入 |
|-----------|----------|----------- |
| `scanf("(%d)", &x)` | `(42)` | `42`, `(42`, `42)` |
| `scanf("%d %d", &a, &b)` | `10 20`, `10\t20` | - |

> **提示**：保持格式字符串简单；避免嵌入复杂的字面文本。

**输入顺序：**
```cpp
scanf("x=%d, y=%d", &x, &y);  // 输入必须为：x=2, y=3
// 输入时，先输 x，然后值，然后 y 和它的值
```

| 格式字符串 | 示例输入 |
|-----------|----------|
| `scanf("x=%d", &x)` | `x=2` |
| `scanf("y=%d", &y)` | `y=3` |
| `scanf("x=%d, y=%d", &x, &y)` | `x=2, y=3` |

4. **类型不匹配 = 未定义行为**：`scanf("%f", &int_var)` 导致垃圾值

#### 7.2.1.7 `scanf` 的返回值

##### 7.2.1.7.1 基本定义

`scanf` 返回一个 **`int`** 值，等于**成功转换的次数**（成功匹配并赋值的输入项数量）。

```cpp
int scanf(const char *format, ...);
```

##### 7.2.1.7.2 返回值含义

| 返回值 | 含义 |
|--------|------ |
| **正整数** | 成功转换的数据项数量 |
| **0** | 存在输入但没有转换成功（输入与格式不匹配） |
| **EOF** | 到达文件末尾（通常为 `-1`，在 `<cstdio>` 中定义），或发生读取错误 |

> **注意**：`EOF` 是一个宏，通常在 `<cstdio>` 中定义为 `-1`。

##### 7.2.1.7.3 实用代码示例

**示例 1：正常成功读取**
```c
int a, b;
int result = scanf("%d %d", &a, &b);
// 输入：10 20
// result = 2（成功读取 2 个整数）
```

**示例 2：部分匹配**
```c
int num;
char ch;
int result = scanf("%d %c", &num, &ch);
// 输入：42 X  → result = 2
// 输入：42（回车）→ result = 1（只读到整数，换行符留在缓冲区）
```

**示例 3：转换失败**
```c
int num;
int result = scanf("%d", &num);
// 输入：hello（非数字字符串）
// result = 0（无法转换为整数）
```

**示例 4：EOF（文件末尾）**
```c
int num;
int result = scanf("%d", &num);
// 按 Ctrl+Z（Windows）或 Ctrl+D（Unix/Linux/Mac）
// result = EOF (-1)
```

##### 7.2.1.7.4 实际应用场景

**1. 循环读取直到 EOF**
```c
int num;
while (scanf("%d", &num) == 1) {  // 只要成功读取一个整数就继续
    printf("Read: %d\n", num);
}
// 遇到非数字输入或 EOF 时退出
```

**2. 输入验证**
```c
int num;
printf("Enter a number: ");
while (scanf("%d", &num) != 1) {
    printf("Invalid input! Try again: ");
    while (getchar() != '\n');  // 从缓冲区清除错误输入
}
```

**3. 读取变长输入**
```c
int a, b, c;
int count = scanf("%d %d %d", &a, &b, &c);
// 根据 count 值判断用户输入了多少个数字
```

##### 7.2.1.7.5 与 `printf` 返回值对比

| 函数 | 返回值含义 |
|------|----------- |
| `scanf` | **成功读取的项数** |
| `printf` | **成功写入的字符数**（出错时为负数） |

```c
int n1 = printf("Hello\n");    // n1 = 6（5 个字符 + 换行符）
int n2 = scanf("%d", &num);    // n2 = 成功读取的数据项数量
```

##### 7.2.1.7.6 底层机制

`scanf` 维护一个内部**扫描指针**，从左到右解析输入流：

1. 跳过空白字符（空格、制表符、换行符）
2. 尝试匹配格式字符串
3. 每次成功转换，计数器加 1
4. 遇到不匹配或 EOF 时停止

返回值是这个**内部计数器的最终值**，反映"实际完成的工作量"。

### 7.2.2 使用 `printf` 输出（C 风格输出）

C 的基于格式的输出函数。需要头文件 `<cstdio>` 或 `<stdio.h>`。

```cpp
#include <cstdio>
printf("control string", arg1, arg2, ...);
```

#### 7.2.2.1 头文件风格：C 与 C++

| 风格 | 头文件 | 用法 | 命名空间 |
|------|--------|------|----------|
| **C 风格** | `#include <stdio.h>` | C / C++ | 全局命名空间 |
| **C++ 风格** | `#include <cstdio>` | 仅 C++ | `std::` 命名空间（也可能在全局） |

> **注意**：在 C++ 代码中，优先使用 `<cstdio>` 而非 `<stdio.h>`。两者都能工作，但 `<cstdio>` 更地道。

**性能**：对于大数据量输出，通常比 `cout` 更快，但类型安全性较低。

#### 7.2.2.2 基本用法

**控制字符串结构：**

```cpp
printf("control string", arg1, arg2, ...);
```

控制字符串包含：
- **转换说明符**（如 `%d`、`%f`）—— 被参数替换
- **字面文本** —— 按原样打印
- **转义序列**（如 `\n`、`\t`）

**基本示例：**

```cpp
int a = 88, b = 89;
printf("%d %d\n", a, b);           // 88 89
printf("%d,%d\n", a, b);           // 88,89
printf("a=%d,b=%d\n", a, b);       // a=88,b=89
```

**多个说明符：**

若控制字符串包含多个转换说明符，则必须提供相同数量的参数：

```cpp
printf("Results: x = %5.2f, y = %5.2f, z = %5.2f\n", x, y, z + 3);
// 输出：Results: x = 4.52, y = 0.15, z = -1.34
```

> **注意：** 参数可以是变量或表达式（例如 `z + 3`）。

> 另请参阅：[2.2.2.3.1 拆分 printf 语句](#22231-splitting-printf-statements) 了解行继续技巧。

#### 7.2.2.3 格式说明符语法

```
%[flags][width][.precision][length]specifier
     ↓     ↓       ↓         ↓      ↓
 可选   可选    可选     可选   必需
```

**顺序**：`flags` → `width` → `.precision` → `length` → `specifier`（从左到右）

##### 1. 标志（可选）

| 标志 | 描述 | 示例 |
|------|------|------ |
| `-` | 左对齐（默认：右对齐） | `%-10d` → `"42        "` |
| `+` | 正数显示符号 | `%+d` → `+42` |
| ` `（空格） | 正数前加空格 | `% d` → ` 42` |
| `#` | 替代形式（`0x`、`0` 前缀） | `%#x` → `0xff` |
| `0` | 零填充（配合宽度） | `%05d` → `00042` |

##### 2. 宽度（可选）

指定要打印的**最小**字符数。

| 行为 | 含义 | 示例 |
|------|------|------ |
| **默认右对齐** | 数字右对齐，左侧用空格填充 | `%8d` 配合 `42` → `"      42"`（6 个空格 + 42） |
| **需要时自动扩展** | 若数字太长，字段宽度会增加以容纳；从不截断 | `%4d` 配合 `-145` → `"-145"`（恰好 4 个字符） |

**要点：**
- `%8d` = **至少** 8 个字符；若更短，左侧用空格填充；若更长，自动扩展
- 宽度是**最小值**，不是固定值
- 可以把它想象成 Excel 单元格的最小宽度：内容短时填充空格，内容长时自动扩展而不截断。

| 宽度 | 描述 | 示例 |
|------|------|------ |
| `n` | 最小字段宽度 | `%10d` → `"        42"` |
| `*` | 宽度来自参数列表 | `printf("%*d", 10, 42);` |

##### 3. 精度（`.precision`）—— 可选

| 精度 | 用于类型 | 效果 | 示例 |
|------|----------|------|------ |
| `.n` | `%f`, `%e` | n 位小数（默认 6） | `%.2f` → `3.14` |
| `.n` | `%g` | n 位有效数字 | `%.3g` → `3.14` |
| `.n` | `%s` | 最多 n 个字符 | `%.3s` → `"Hel"` |
| `.n` | `%d` | 至少 n 位数字（用 0 填充） | `%.5d` → `00042` |
| `.*` | any | 精度来自参数 | `printf("%.*f", 2, 3.14159);` |

**精度行为：**
- 小数部分按指定精度**四舍五入**（`14.51678` 配合 `%.2f` → `14.52`）
- **宽度 + 精度**可以一起使用（例如 `%8.2f` 配合 `3.14159` → `"    3.14"`）

> **要截断而非四舍五入：** 预处理值（参见 [7.1.2.8](#71128-truncating-floating-point-output-no-rounding)）

##### 4. 长度修饰符（可选）

| 修饰符 | 用于 | C 类型 | 示例 |
|--------|------|--------|------ |
| `hh` | `%d`, `%u`, `%o`, `%x` | `signed/unsigned char` | `%hhd` |
| `h` | `%d`, `%i`, `%o`, `%x`, `%u` | `short` / `unsigned short` | `%hd`, `%hi`, `%hu` |
| `l` | `%d`, `%i`, `%o`, `%x`, `%u` | `long` / `unsigned long` | `%ld`, `%li`, `%lu` |
| `ll` | `%d`, `%u`, `%o`, `%x` | `long long` / `unsigned long long` | `%lld` |
| `j` | `%d`, `%u`, `%o`, `%x` | `intmax_t` / `uintmax_t` | `%jd` |
| `z` | `%d`, `%u`, `%o`, `%x` | `size_t` | `%zu` |
| `t` | `%d`, `%u`, `%o`, `%x` | `ptrdiff_t` | `%td` |
| `L` | `%f`, `%e`, `%g`, `%a` | `long double` | `%Lf`, `%Le`, `%Lg`（小写/大写：`%LF`, `%LE`, `%LG`） |
| `l` | `%c`, `%s` | 宽字符/宽字符串 | `%lc`, `%ls` |

#### 7.2.2.3.5 长度修饰符与类型匹配规则

长度修饰符、转换说明符和实际变量类型之间的关系必须**完全匹配**，以避免未定义行为。

**关键原则：** `长度修饰符` + `转换说明符` 必须匹配实际变量类型

**常见 Bug 示例：** 对 `long long` 变量使用 `%d` 只会从 8 字节变量中读取 4 字节，导致输出错误：

```cpp
long long ll = 8589934592LL;  // 2^33
printf("%d\n", ll);   // ✗错误：%d 期望 int（4 字节），但 ll 是 long long（8 字节）
printf("%lld\n", ll); // ✓正确：%lld 匹配 long long（8 字节）
```

**常见组合表：**

| 长度修饰符 | 转换说明符 | C++ 类型 | 大小 |
|-----------|-----------|----------|------ |
| （无） | `%d` / `%i` | `int` | 4 字节 |
| `h` | `%hd` / `%hi` | `short` | 2 字节 |
| `hh` | `%hhd` | `signed char` | 1 字节 |
| `l` | `%ld` / `%li` | `long` | 4/8 字节 |
| `ll` | `%lld` | `long long` | 8 字节 |
| `h` | `%hu` | `unsigned short` | 2 字节 |
| `hh` | `%hhu` | `unsigned char` | 1 字节 |
| `l` | `%lu` | `unsigned long` | 4/8 字节 |
| `ll` | `%llu` | `unsigned long long` | 8 字节 |
| `L` | `%Lf` / `%Le` / `%Lg` | `long double` | 8+ 字节 |

> **警告：** 不匹配的说明符会导致未定义行为。读取的数据可能不正确，或程序可能从参数栈读取错误数量的字节，可能导致崩溃或数据损坏。

##### 7. 转换说明符（必需）

| 说明符 | 类型 | 输出 | 示例 |
|--------|------|------|------ |
| **整数类型** | | | |
| `%d` / `%i` | `int` | 有符号十进制 | `123`, `-456` |
| `%u` | `unsigned int` | 无符号十进制 | `789` |
| `%o` | `unsigned int` | 八进制 | `377` |
| `%x` / `%X` | `unsigned int` | 十六进制 | `ff` / `FF` |
| **浮点类型** | | | |
| `%f` / `%F` | `double` | 定点（浮点形式） | `3.141500` |
| `%e` / `%E` | `double` | 指数形式（如 `2.3e+02` / `2.3E+02`） | `3.141500e+00` |
| `%g` / `%G` | `double` | 较短者：`%f` 或 `%e`/`%E` | `%g` 用 `%e`，`%G` 用 `%E` |
| `%a` / `%A` | `double` | 十六进制浮点 | `0x1.921fb5p+1` |
| **字符/字符串类型** | | | |
| `%c` | `int` | 单个字符 | `A` |
| `%s` | `char*` | 字符串 | `Hello` |
| **其他** | | | |
| `%p` | `void*` | 指针地址 | `0x7ffeeb2b3a5c` |
| `%n` | `int*` | 到目前为止打印的字符数 | （存储计数） |
| `%%` | - | 字面量 `%` | `%` |

> **注意：** `%d` 和 `%i` 在输出中等价（都打印有符号十进制整数）。

**选择合适的说明符：**
- `short` 或 `int` → 使用 `%i`（整数）或 `%d`（十进制）
- `long` → 使用 `%li` 或 `%ld`
- `float` 或 `double` → 使用 `%f`（定点）、`%e`/`%E`（指数）或 `%g`/`%G`（自动选择最短者）

#### 7.2.2.4 格式示例

**整数示例（值 = -145）：**

| 说明符 | 输出 | 说明 |
|--------|------|------ |

| `%4d` | `-145` | 宽度 4，值恰好适配 |

| `%-6i` | `-145  ` | 宽度 6，左对齐（右侧 2 个空格） |

**浮点示例（值 = 157.8926）：**

|--------|------|------ |

#### 7.2.2.5 转义序列

反斜杠（`\`）是**转义字符**，给后续字符赋予特殊含义。

| 序列 | 含义 | 描述 |
|------|------|------ |
| `\n` | 换行 | 跳到新行 |
| `\t` | 水平制表符 | 移到下一个制表位 |
| `\a` | 警报（蜂鸣） | 声音警报 |
| `\b` | 退格 | 回退一格 |
| `\f` | 换页 | 新页（很少使用） |
| `\r` | 回车 | 返回行首 |
| `\v` | 垂直制表符 | 垂直间距 |
| `\\` | 反斜杠 | 打印单个反斜杠 |
| `\'` | 单引号 | 打印单引号 |
| `\"` | 双引号 | 打印双引号 |

**示例：**

```cpp
printf("\"The End.\"\n");     // 输出："The End."（带换行）
printf("Path: C:\\Users\\John\n");  // 输出：Path: C:\Users\John
```

#### 7.2.2.6 拆分长 printf 语句

当 printf 语句太长而无法放在一行时，使用以下技巧拆分它们，同时保持可读性。

**方法 1：在逗号后拆分**

在逗号后拆分，并将续行缩进至与左括号或第一个参数对齐。

```cpp
printf("The distance between the two points is %5.2f \n",
       distance);
```

**方法 2：将字符串拆分为两个带引号的部分**

将字符串字面量分成多个片段，每个片段放在各自的引号中。编译器会自动拼接它们。

```cpp
printf("The distance between the two points is"
       " %5.2f \n", distance);
```

**方法 3：在文本的自然断点处拆分**

在文本的逻辑断点处（例如单词之间）拆分以提高可读性。

```cpp
printf("The distance between the "
       "points is %5.2f \n", distance);
```

**字符串拼接规则**

拆分字符串时，相邻的带引号字符串会被编译器自动拼接。这适用于任何字符串字面量，不仅限于 printf 语句。

```cpp
// cout 示例
cout << "This is a very long message that "
     << "needs to be split into multiple lines"
     << endl;
```

> 另请参阅：[2.2.2.3.1 拆分 printf 语句](#22231-splitting-printf-statements) 了解行继续技巧。

#### 7.2.2.7 与 `cout` 的关键区别

| 特性 | `printf` | `cout` |
|------|----------|-------- |
| 类型安全 | 否（错误时运行时出错） | 是（编译期检查） |
| 性能 | 更快 | 更慢 |
| 可扩展性 | 有限 | 易于扩展 |

> **注意：** 在性能关键代码中使用 `printf` 进行格式化输出，但在现代 C++ 中优先使用 `cout` 以获得类型安全。

#### 7.2.2.8 `printf` 的返回值

##### 7.2.2.8.1 基本定义

`printf` 返回一个 **`int`** 值，等于**成功写入输出流的字符数**。

```cpp
int printf(const char *format, ...);
```

##### 7.2.2.8.2 返回值含义

| 返回值 | 含义 |
|--------|------ |
| **正整数** | 成功写入的字符数（不包括终止空字节 `\0`） |
| **0** | 没有字符被写入 |
| **负数** | 输出期间发生错误（通常为 `-1`，检查 `errno` 获取详情） |

> **注意**：负返回值表示输出错误（例如流已关闭、磁盘已满、I/O 错误）。

##### 7.2.2.8.3 实用代码示例

**示例 1：正常成功输出**
```c
int result = printf("Hello, World!\n");
// result = 14（13 个字符 + 1 个换行符）
```

**示例 2：使用返回值检查输出**
```c
int result = printf("Value: %d\n", 42);
if (result < 0) {
    // 处理输出错误
    perror("printf failed");
}
// result = 10（"Value: " = 7，"42" = 2，"\n" = 1）
```

**示例 3：带宽度的格式说明符**
```c
int result = printf("%10d\n", 5);
// 打印 "         5"（10 个字符：9 个空格 + "5"）
// result = 11（10 个字符 + 1 个换行符）
```

##### 7.2.2.8.4 常见使用场景

| 使用场景 | 示例 | 目的 |
|----------|------|------ |
| **错误检查** | `if (printf(...) < 0)` | 检测输出失败 |
| **字符计数** | `int len = printf(...)` | 知道打印了多少字符 |
| **格式化字符串长度** | `snprintf` 配合 `NULL` | 计算所需缓冲区大小 |



## 7.3 文件 I/O

来自 `<fstream>` 的文件流提供了从文件读取和写入文件的设施。

### 7.3.1 文件输入（ifstream）

**从文件读取：**
```cpp
#include <fstream>

ifstream inFile("input.txt");
if (!inFile) {
    cerr << "Cannot open file!" << endl;
    return 1;
}

// 逐词读取
string word;
while (inFile >> word) {
    cout << word << endl;
}

// 逐行读取
string line;
while (getline(inFile, line)) {
    cout << line << endl;
}

inFile.close();
```

**重要：** 读取前始终检查文件是否成功打开。


### 7.3.2 文件输出（ofstream）

**写入文件：**
```cpp
#include <fstream>

ofstream outFile("output.txt");
if (!outFile) {
    cerr << "Cannot create file!" << endl;
    return 1;
}

outFile << "Hello World" << endl;
outFile << "Line 2: " << 42 << endl;

outFile.close();
```

**追加到文件：**
```cpp
ofstream outFile("output.txt", ios::app);  // 追加模式
outFile << "New line appended" << endl;
outFile.close();
```
