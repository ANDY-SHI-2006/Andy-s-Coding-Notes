# 9 函数

## 9.1 数学函数

> **头文件：** 本节大多数函数需要 `#include <cmath>`（C++ 风格）或 `#include <math.h>`（C 风格）
>
> **例外：** 整数 `abs()` 需要 `#include <cstdlib>` —— 参见 9.1.2 节
>
> **注意：** 在 C++ 中，`<cmath>` 将函数放入 `std` 命名空间。使用 `using namespace std;` 或加 `std::` 前缀。

### 9.1.1 `<cmath>` 函数

**参数类型：** 这些数学函数接受 `double` 参数。其他类型（`int`、`float`）自动转换为 `double`。返回类型始终为 `double`。

```cpp
int x = 9;
double a = sqrt(x);   // x 自动转换为 9.0，a = 3.0
int b = sqrt(x);      // 危险：3.0 转回 int，丢失精度
```

### 9.1.1.1 初等函数

| 函数 | 描述 | 说明 |
|------|------|------ |
| `fabs(x)` | 计算 x 的绝对值 | 返回 `double` |
| `sqrt(x)` | 计算 x 的平方根 | 要求 `x >= 0` |
| `pow(x, y)` | 计算 x 的 y 次幂（x^y） | **定义域错误：** <br>— `x = 0` 且 `y <= 0`（未定义：0^(-1)） <br>— `x < 0` 且 `y` 不是整数（例如 (-2)^1.5 不是实数） |
| `ceil(x)` | 将 x 向上取整到最近的整数（朝向 +∞） | `ceil(2.01) = 3`，`ceil(-2.01) = -2` |
| `floor(x)` | 将 x 向下取整到最近的整数（朝向 -∞） | `floor(2.01) = 2`，`floor(-2.01) = -3` |
| `exp(x)` | 计算 e^x | e ≈ 2.718282 |
| `log(x)` | 计算 x 的自然对数（ln x） | 要求 `x > 0` |
| `log10(x)` | 计算 x 的常用对数（log₁₀ x） | 要求 `x > 0` |

### 9.1.1.2 `fabs` 与 `abs`

| 函数 | 输入类型 | 返回类型 | 头文件 |
|------|----------|----------|-------- |
| `abs(x)` | `int` | `int` | `<cstdlib>` 或 `<stdlib.h>` |
| `fabs(x)` | `double` | `double` | `<cmath>` 或 `<math.h>` |

**关键区别：**
- `abs` 仅用于**整数**
- `fabs` 用于**浮点数**（**f** 代表 **floating-point**）

```cpp
#include <cstdlib>
int a = abs(-5);      // 返回 5（int）

#include <cmath>
double x = fabs(-5.5); // 返回 5.5（double）
```

> **提示：** 整数用 `abs`，浮点数用 `fabs`。混用可能导致意外的类型转换或精度丢失。

### 9.1.1.3 幂运算

> **重要：** C++ **没有**内置的幂运算符。

> **警告：** `^` 符号是**按位异或运算符**，不是幂运算！
```cpp
int result = 2 ^ 3;  // 结果：1（XOR），而非 8！
```

| 方法 | 速度 | 使用场景 |
|------|------|---------- |
| `a * a` | 最快 | 小固定指数：a²、a³ |
| `pow(a, b)` | 较慢 | 变量/分数指数 |

### 9.1.1.4 三角函数

> **关键：** 参数必须是**弧度**，而非角度！
>
> **参数类型：** 三角函数假设所有参数为 `double`，返回 `double` 值。

| 函数 | 描述 |
|------|------ |
| `sin(x)` | x 的正弦 |
| `cos(x)` | x 的余弦 |
| `tan(x)` | x 的正切 |
| `asin(x)` | 反正弦，返回 [-π/2, π/2] |
| `acos(x)` | 反余弦，返回 [0, π] |
| `atan(x)` | 反正切，返回 [-π/2, π/2] |
| `atan2(y, x)` | y/x 的反正切，返回 [-π, π] |

> **注意：** `atan` 始终返回第一或第四象限的角度（右半平面）。`atan2` 返回的角度可以位于**任何象限**，取决于 x 和 y 的符号。
>
> **推荐：** 在许多应用中，`atan2(y, x)` 优于 `atan(y/x)`，因为它正确处理所有象限，并避免 x = 0 时的除零错误。

**角度-弧度转换：**
```cpp
#define PI 3.141593

// 角度转弧度
rad = deg * PI / 180;

// 弧度转角度
deg = rad * 180 / PI;
```

### 9.1.1.5 双曲函数

| 函数 | 描述 | 公式 |
|------|------|------ |
| `sinh(x)` | 双曲正弦 | (e^x - e^(-x)) / 2 |
| `cosh(x)` | 双曲余弦 | (e^x + e^(-x)) / 2 |
| `tanh(x)` | 双曲正切 | sinh(x) / cosh(x) |

### 9.1.2 `<cstdlib>` 函数

> **头文件：** `#include <cstdlib>`（C++ 风格）或 `#include <stdlib.h>`（C 风格）

#### 9.1.2.1 `abs`

**整数绝对值：**

| 函数 | 描述 | 说明 |
|------|------|------ |
| `abs(x)` | 计算整数 x 的绝对值 | 返回 `int`，需要 `<cstdlib>` |

**与 `fabs` 对比：**

| 函数 | 输入类型 | 返回类型 | 头文件 |
|------|----------|----------|-------- |
| `abs(x)` | `int` | `int` | `<cstdlib>` |
| `fabs(x)` | `double` | `double` | `<cmath>` |

```cpp

int a = abs(-5);      // 返回 5（int）

#include <cmath>

```

> **提示：** 整数用 `abs`，浮点数用 `fabs`。混用可能导致意外的类型转换或精度丢失。

#### 9.1.2.2 随机数

##### 9.1.2.2.1 基本用法

> **头文件：** `#include <cstdlib>`（C++ 风格）或 `#include <stdlib.h>`（C 风格）

`rand()` 函数生成伪随机整数：

```cpp
int r = rand();  // 返回 0 到 RAND_MAX 之间的整数（通常为 32767）
```

**示例：**
```cpp
printf("random numbers: %i %i\n", rand(), rand());
// 输出：41 18467
```

##### 9.1.2.2.2 问题：伪随机序列

`rand()` 生成的序列是**确定性的**。它从默认的**种子**值 `1` 开始，每次程序运行产生相同的序列。

```cpp
// 第 1 次运行：41 18467 6334 ...
// 第 2 次运行：41 18467 6334 ...  （完全相同！）
```

这称为**伪随机**序列，因为它看起来随机，但数学上是可复现的。

##### 9.1.2.2.3 解决方案：用 `srand()` 播种

要在每次运行时获得不同的序列，必须用不同的起始值**播种**随机数生成器。

```cpp
srand(seed_value);  // 设置序列的起始点
```

**实用方法：** 使用当前时间作为种子：

```cpp
#include <ctime>    // 使用 time()

srand(time(NULL));  // 用当前时间戳播种
```

**完整示例：**
```cpp
#include <cstdio>
#include <cstdlib>
#include <ctime>

int main() {
    srand(time(NULL));  // 在程序启动时播种一次
    
    printf("%i %i %i\n", rand(), rand(), rand());
    // 现在每次运行产生不同的值！
    return 0;
}
```

##### 9.1.2.2.4 播种的重要注意事项

**1. `srand()` 只影响后续调用**
```cpp
rand();           // 使用默认种子 1
srand(100);       // 改变种子
rand();           // 使用种子 100
```

**2. 只调用一次 `srand()`**
```cpp
// 错误：
for (int i = 0; i < 10; i++) {
    srand(time(NULL));  // 每次迭代都重置种子！
    printf("%i\n", rand());  // 可能重复打印相同的数字
}

// 正确：
srand(time(NULL));       // 只在开始时播种一次
for (int i = 0; i < 10; i++) {
    printf("%i\n", rand());  // 产生正确的序列
}
```

**3. 固定种子用于可复现性**
```cpp
srand(42);  // 固定种子使调试更容易
// 程序每次运行产生相同的序列
// 发布时改为 time(NULL)
```

##### 9.1.2.2.5 生成特定范围内的数字

**整数范围 [0, n-1]：**
```cpp
int x = rand() % n;  // 0 到 n-1（例如 rand() % 10 产生 0-9）
```

> **注意：** 这种方法对大范围有轻微偏差。对于密码学或统计应用，使用 `<random>` 库（C++11）。

**整数范围 [a, b]：**
```cpp
int rand_int(int a, int b) {
    return rand() % (b - a + 1) + a;
}

// 用法：rand_int(10, 20) 产生 10 到 20（含）
```

**浮点范围 [a, b]：**
```cpp
double rand_float(double a, double b) {
    return ((double)rand() / RAND_MAX) * (b - a) + a;
}

// 用法：rand_float(0.0, 1.0) 产生 0.0 到 1.0
```

### 9.1.3 `<random>` 函数

> **头文件：** `#include <random>`（C++11 及以后）

`<random>` 库提供了现代、类型安全的随机数生成方式。与 `rand()` 不同，它将**随机数引擎**（随机性来源）与**分布**（值如何映射到范围）分离。

#### 9.1.3.1 基本概念

**1. 引擎 —— 生成原始随机位**

| 类 | 描述 |
|----|------ |
| `std::random_device` | 基于硬件（或实现提供）的非确定性随机源，通常用于播种 |
| `std::mt19937` | 32 位梅森旋转引擎；周期很长，统计性质优良 |
| `std::mt19937_64` | 64 位版梅森旋转引擎 |

```cpp
std::random_device rd;        // 非确定性源
std::mt19937 gen(rd());       // 用种子初始化引擎
```

> **注意：** 若 `std::random_device` 不可用（罕见，主要在嵌入式系统），改用 `std::chrono::steady_clock::now().time_since_epoch().count()` 作为种子。

**2. 分布 —— 将原始位映射到期望的范围和概率**

分布对象独立于引擎。每次想要一个数时，将引擎传给分布的 `operator()`。

#### 9.1.3.2 常见分布

| 分布 | 使用场景 | 示例 |
|------|----------|------ |
| `std::uniform_int_distribution<int>` | [a, b] 中每个整数等概率 | 掷骰子、彩票 |
| `std::uniform_real_distribution<double>` | [a, b) 中每个实数等概率 | 0.0–1.0 随机浮点数 |
| `std::normal_distribution<double>` | 钟形曲线（高斯分布） | 噪声、自然变化 |

**整数范围 [a, b]（含）：**
```cpp
std::uniform_int_distribution<> dis(1, 6);  // 闭区间 [1, 6]
int x = dis(gen);  // 如同掷骰子
```

**浮点范围 [a, b)：**
```cpp
std::uniform_real_distribution<> dis(0.0, 1.0);
double r = dis(gen);  // 0.0 <= r < 1.0
```

**正态分布：**
```cpp
std::normal_distribution<> dis(0.0, 1.0);  // 均值 = 0，标准差 = 1
double r = dis(gen);
```

#### 9.1.3.3 完整示例

**示例 1：掷骰子 10 次**
```cpp
#include <iostream>
#include <random>

int main() {
    std::random_device rd;
    std::mt19937 gen(rd());
    std::uniform_int_distribution<> dis(1, 6);

    for (int i = 0; i < 10; ++i) {
        std::cout << dis(gen) << ' ';
    }
    std::cout << std::endl;
    return 0;
}
```

**示例 2：用时间播种（`random_device` 不可用时的替代方案）**
```cpp
#include <random>
#include <chrono>

auto seed = std::chrono::steady_clock::now().time_since_epoch().count();
std::mt19937 gen(static_cast<unsigned>(seed));
```

#### 9.1.3.4 `rand()` 与 `<random>` 对比

| 特性 | `rand()` / `srand()` | `<random>`（现代） |
|------|----------------------|--------------------- |
| **均匀性** | `rand() % n` 引入偏差 | 数学上保证均匀 |
| **最大值** | 限于 `RAND_MAX`（常为 32767） | 没有实际限制 |
| **周期** | 短 | 非常长（例如 `mt19937` 周期为 2^19937−1） |
| **分布类型** | 仅有粗略的均匀整数 | 均匀、正态、指数、泊松等 |
| **类型安全** | 单一函数返回 `int` | 模板类区分 `int` 与 `double` |

> **推荐：** 新 C++ 代码始终使用 `<random>`。仅在遗留 API 要求或特定课程约束时使用 `rand()`。

#### 9.1.3.5 常见陷阱

**不要在循环内重新创建引擎：**
```cpp
// 错误：每次迭代都重新播种引擎
for (int i = 0; i < 10; ++i) {
    std::mt19937 gen(std::random_device{}());
    std::cout << std::uniform_int_distribution<>(1, 6)(gen) << ' ';
}
```

**正确：创建一次引擎，重复使用：**
```cpp
std::random_device rd;
std::mt19937 gen(rd());
std::uniform_int_distribution<> dis(1, 6);

for (int i = 0; i < 10; ++i) {
    std::cout << dis(gen) << ' ';
}
```

### 9.1.4 字符函数

> **头文件：** 本节涵盖来自两个不同头文件的函数：
> - `<cstdio>` —— 字符 I/O 函数（9.1.4.1 节）
> - `<cctype>` —— 字符分类与转换（9.1.4.2 节）

### 9.1.4.1 `<cstdio>` 字符 I/O

> **头文件：** `#include <cstdio>`（C++ 风格）或 `#include <stdio.h>`（C 风格）

C 提供两种字符 I/O 方法：
1. 使用 `printf`/`scanf` 配合 `%c` 格式说明符
2. 使用专用字符函数 `getchar()` 和 `putchar()`

### 9.1.4.1.1 使用 `printf` 和 `scanf` 配合 `%c`

`%c` 格式说明符处理单个字符：

```cpp
char ch;
scanf("%c", &ch);   // 读取一个字符
printf("%c", ch);   // 打印一个字符
```

**重要说明：**

- `scanf("%c", &ch)` 读取**任何**字符，包括空白字符（空格、制表符、换行符）
- 要跳过空白字符再读取字符，加一个空格：`scanf(" %c", &ch)`

### 9.1.4.1.2 使用 `getchar()` 和 `putchar()`

这些是专用字符 I/O 函数：

| 函数 | 描述 | 示例 |
|------|------|------ |
| `getchar()` | 从键盘读取下一个字符，返回其 ASCII 值 | `c = getchar();` |
| `putchar(c)` | 将字符打印到屏幕 | `putchar('a');` |

**基本示例：**

```cpp
// 打印字符
putchar('a');     // 输出：a
putchar('\n');    // 换行

// 使用 ASCII 值
putchar(97);      // 输出：a（ASCII 97）
putchar(65);      // 输出：A（ASCII 65）
```

### 9.1.4.1.3 常见问题与解决方案

**问题 1：输入缓冲区残留**

混合使用 `scanf`（读取数字）与 `getchar` 时，先前输入的换行符（`\n`）留在缓冲区中：

```cpp
int num;
char ch;
scanf("%d", &num);    // 输入：42\n
// \n 仍在缓冲区中！
ch = getchar();       // 读取 '\n' 而非等待新输入
```

**解决方案：**
- **Windows：** 调用 `fflush(stdin)` 清除缓冲区（注意：`fflush(stdin)` 仅在 Windows 上有效，不是标准 C）
- **跨平台：** 使用 `getchar()` 一次消耗换行符，或使用带前导空格的 `scanf(" %c", &ch)`

```cpp
// 方案 1：fflush（Windows）
scanf("%d", &num);
fflush(stdin);        // 清除残留的 \n
ch = getchar();       // 现在读取新输入

// 方案 2：scanf 中的前导空格
scanf(" %c", &ch);    // 自动跳过所有空白字符
```

**问题 2：`putchar()` 的连续输出**

多个 `putchar` 调用在同一行上顺序打印字符，直到遇到换行符：

```cpp
putchar('H');         // 输出：H
putchar('i');         // 输出：Hi（同一行）
putchar('\n');        // 输出：Hi\n（现在移到新行）
putchar('!');         // 下一行输出
```

> **注意：** 这是预期行为——字符累积，直到 `\n` 刷新输出或移到新行。

### 9.1.4.2 `<cctype>` 字符分类与转换

> **头文件：** `#include <cctype>`（C++ 风格）或 `#include <ctype.h>`（C 风格）

### 9.1.4.2.1 分类函数

| 函数 | 若非零（真）则返回…… |
| ---- | ------------------- |
| `isalpha(ch)` | ch 是字母（a-z、A-Z） |
| `isdigit(ch)` | ch 是十进制数字（0-9） |
| `isalnum(ch)` | ch 是字母数字（字母或数字） |
| `islower(ch)` | ch 是小写字母 |
| `isupper(ch)` | ch 是大写字母 |
| `isblank(ch)` | ch 是空格或制表符（' ' 或 '\t'） |
| `isspace(ch)` | ch 是空白字符（空格、制表符、换行符等） |
| `isprint(ch)` | ch 是可打印字符（包括空格） |
| `isgraph(ch)` | ch 是可打印字符（不包括空格） |
| `iscntrl(ch)` | ch 是控制字符（0-21、127） |
| `ispunct(ch)` | ch 是标点符号（不是空格、字母或数字） |
| `isxdigit(ch)` | ch 是十六进制数字（0-9、A-F、a-f） |

> **注意：** `isblank()` 只检查空格 `' '` 和制表符 `'\t'`，而 `isspace()` 检查所有空白字符，包括换行符 `'\n'`、回车符 `'\r'`、换页符 `'\f'` 和垂直制表符 `'\v'`。

### 9.1.4.2.2 转换函数

| 函数 | 描述 |
|------|------ |
| `tolower(ch)` | 返回 ch 的小写版本（若为大写） |
| `toupper(ch)` | 返回 ch 的大写版本（若为小写） |

**函数行为：**

两个函数转换字母大小写而不修改原始变量：

| 输入 `ch` | `toupper(ch)` 返回 | `tolower(ch)` 返回 |
|-----------|-------------------|------------------- |
| 小写 `'a'`–`'z'` | 大写 `'A'`–`'Z'` | 原始值（已经是小写） |
| 大写 `'A'`–`'Z'` | 原始值（已经是大写） | 小写 `'a'`–`'z'` |
| 非字母（数字、符号等） | 原始值不变 | 原始值不变 |

**重要说明：**

- **原始变量不会被修改** —— 函数只返回转换后的值
- 要改变变量本身，必须重新赋值：`ch = toupper(ch);`

**示例：**

```cpp
char ch = 'a';
char upper = toupper(ch);     // upper = 'A'，ch 仍然是 'a'

ch = toupper(ch);             // 现在 ch 变为 'A'

// 非字母不变
char sym = '5';
char result = toupper(sym);   // result = '5'（不变）
```



## 9.2 程序员自定义函数

**函数组织：**
使用虚线注释块分隔函数，使其在视觉上清晰：
```cpp
/*------------------------------------------------------------*/
/* 该函数计算 sinc 函数的值。                                 */
double sinc(double x)
{
    if (fabs(x) < 0.0001)
        return 1.0;
    else
        return sin(PI*x)/(PI*x);
}
/*------------------------------------------------------------*/
```

在 `main()` 中，用行内注释记录每一步：
```cpp
int main(void)
{
    /* 声明变量。 */
    int k;
    double a, b;

    /* 从用户获取区间端点。 */
    printf("Enter endpoints: \n");

    /* 计算并输出值表。 */
    /* 退出程序。 */
    return 0;
}
```

### 9.2.1 函数定义

**一般形式：**
```cpp
return_type function_name(parameter_declarations)
{
    declarations;
    statements;
}
```

**组成部分：**
1. **返回类型**：返回值的类型（无则用 `void`）
2. **函数名**：函数的标识符
3. **参数**：输入值（可以为空）
   - 若无参数：可以显式写 `void` 或留空：`return_type func(void)` 或 `return_type func()`
4. **函数体**：花括号中的语句

**重要规则：**
- 函数不能嵌套（一个函数不能定义在另一个函数内部）
  > **注意：** 与 Python 和某些其他语言不同，C++ 不支持嵌套函数（在另一个函数内部定义函数）。嵌套函数是 Python 中闭包和装饰器的基础。
- 函数必须在另一个函数开始之前完全定义

### 9.2.2 函数原型

#### 9.2.2.1 问题：声明顺序

**没有原型时**，函数必须在调用*之前*定义：

```cpp
// 这样可行：在 main 之前定义
void foo() { ... }

int main() {
    foo();  // OK
}

// 这样失败：在 main 之后定义
int main() {
    bar();  // 错误：bar() 未声明！
}
void bar() { ... }
```

**有原型时**，函数可以定义在*任何位置*：

```cpp
void bar();  // 原型声明 bar() 存在

int main() {
    bar();  // OK：编译器从原型知道 bar()
}

void bar() { ... }  // 定义可以在 main 之后
```

#### 9.2.2.2 语法

**基本形式：**
```cpp
return_type function_name(parameter_list);
```

**带参数名**（推荐用于文档）：
```cpp
void printTable(double start, double end, int rows);
double sinc(double x);
```

**不带参数名**（也有效）：
```cpp
void printTable(double, double, int);
double sinc(double);
```

#### 9.2.2.3 原型放置位置

| 位置 | 用法 |
|------|------ |
| **文件顶部** | 在 `main()` 之前，以便所有函数都能使用 |
| **头文件中** | 用于在多个源文件之间共享 |

**标准库头文件**（如 `<cmath>`、`<cstdio>`）包含库函数的原型。没有 `#include`，你就需要手动编写它们。

#### 9.2.2.4 最佳实践

| 场景 | 推荐 |
|------|------ |
| **单文件** | 将所有原型放在 `#include` 之后、`main()` 之前 |
| **多文件** | 将原型放在 `.h` 头文件中，在需要处 `#include` |
| **始终？** | 是——即使函数在使用前定义，原型也能提高清晰度 |

**典型布局：**
```cpp
#include <iostream>
using namespace std;

// 函数原型
void functionA(int x, double y);
int functionB(const string& str);

int main() {
    // 主逻辑
    functionA(10, 3.14);
    return 0;
}

// 函数定义
void functionA(int x, double y) { /* ... */ }
int functionB(const string& str) { /* ... */ }
```

**总结：** 原型将"声明"与"定义"解耦，让你可以自由地按逻辑组织代码。

### 9.2.3 参数传递

#### 9.2.3.1 按值传递

默认情况下，C++ 按**值**传递参数。这意味着形参接收实参的*副本*。函数内部对形参的任何修改都**不会**影响调用者中的原始变量。

**示例：**
```cpp
void swap(int a, int b) {  // a、b 是形参
    int temp = a;
    a = b;
    b = temp;  // 更改只影响局部副本
}

int main() {
    int x = 5, y = 7;
    swap(x, y);  // x、y 是实参
    // x 仍然是 5，y 仍然是 7（没有变化！）
}
```

#### 9.2.3.2 按指针（地址）传递

##### 9.2.3.2.1 问题：为什么需要指针

函数面临两个根本限制，而指针可以解决它们：

- **单一返回值**：函数只能 `return` 一个值，但有时我们需要修改多个外部变量
- **按值调用的限制**：按值传递参数只发送副本；函数内部的修改不影响原始变量

通过传递**内存地址**而非值，函数获得对调用者原始变量的直接访问。

##### 9.2.3.2.2 核心概念：传递地址

不传递变量值的副本，而是传递变量存储的**地址**。函数接收该地址，导航到该内存位置，并直接修改值。

##### 9.2.3.2.3 两个基本运算符

| 运算符 | 名称 | 用途 | 示例 |
|--------|------|------|------ |
| `&` | 取地址 | 获取变量的内存地址 | `&x` 返回 x 的地址（例如 `0x7fff5e...`） |
| `*` | 解引用/间接 | 访问或修改地址处的值 | `*ptr` 读取/写入 ptr 地址处的值 |

##### 9.2.3.2.4 工作原理：分步示例

```cpp
void swap(int *a, int *b) {  // 指向 int 的指针
    int hold = *a;   // *a 解引用指针以获取该地址处的值
    *a = *b;         // 将 b 的值存入 a 的地址
    *b = hold;       // 将 hold 存入 b 的地址
}

int main() {
    int x = 5, y = 7;
    swap(&x, &y);  // 使用 & 运算符传递地址
    // 现在 x = 7，y = 5（成功交换！）
}
```

**执行分解：**
1. `swap(&x, &y)` 传递 x 和 y 的**地址**
2. 参数 `a` 和 `b` 是**指针**，存储这些地址
3. `*a` 和 `*b`**解引用**以访问实际内存位置
4. 像 `*a = *b` 这样的赋值修改原始地址处的值

##### 9.2.3.2.5 关键澄清：`*` 的双重含义

`*` 符号根据上下文有**两种截然不同的含义**：

| 上下文 | 含义 | 示例 |
|--------|------|------ |
| 在声明中 | "这是一个指针变量" | `int *a` 声明 `a` 为指向 int 的指针 |
| 在表达式中 | "解引用"（访问地址处的值） | `*a = 10` 向 `a` 存储的地址写入 10 |

**重要：** `&` 和 `*` **并非简单的逆运算**：
- `&` 始终表示"取地址"（获取变量的地址）
- `*` 仅在表达式中用作运算符时才是 `&` 的"逆运算"（例如 `*(&x)` 等于 `x`）

##### 9.2.3.2.6 对比：按值传递与按指针传递

| 方式 | 函数签名 | 函数调用 | 能否修改原始值？ |
|------|----------|----------|---------------- |
| 按值传递 | `void swap(int a, int b)` | `swap(x, y)` | 否（操作副本） |
| 按指针传递 | `void swap(int *a, int *b)` | `swap(&x, &y)` | 是（在地址处修改） |

##### 9.2.3.2.7 重要规则与最佳实践

**指针类型必须匹配数据**

指针声明中的类型表示指针指向的数据类型：

| 指针类型 | 数据大小 | `*p` 读/写 | `p+1` 跳过 |
|----------|----------|------------|------------ |
| `char *p` | 1 字节 | 1 字节 | 1 字节 |
| `int *p` | 4 字节 | 4 字节 | 4 字节 |
| `long long *p` | 8 字节 | 8 字节 | 8 字节 |
| `double *p` | 8 字节 | 8 字节 | 8 字节 |

**为什么类型重要：**
- `int *a` 表示 "`*a` 将按 int 读取/写入 4 字节"
- `long long *a` 表示 "`*a` 将按 long long 读取/写入 8 字节"

**类型不匹配很危险：**
```cpp
int x = 10;
long long *p = &x;  // 错误！指针类型与变量类型不匹配
*p = 100;            // 危险：向 4 字节变量写入 8 字节
                     // 这会覆盖相邻内存！
```

**指针声明风格**

`int *p` 和 `int* p` 都有效且等价：

```cpp
int *p;   // C 风格：强调 *p 是一个 int
int* p;   // C++ 风格：强调 p 是一个指向 int 的指针
```

**多个声明时需谨慎：**

```cpp
int* p, q;   // 只有 p 是指针！q 是普通 int
int *p, *q;  // 两者都是指针
```

**推荐：** 每行声明一个指针以提高清晰度：

```cpp
int* p;
int* q;
```

> **注意：** `*` 只绑定到紧跟其后的变量，而非整个类型声明。

#### 9.2.3.3 参数匹配规则

当函数有多个参数（例如 `printTable`）时，形参和实参必须在以下方面匹配：
- **数量**：参数个数必须相等
- **类型**：对应参数必须具有兼容类型
- **顺序**：参数按从左到右的位置匹配

#### 9.2.3.4 隐式类型转换（强制转换）

若实参类型与形参类型不同，C++ 尝试自动类型转换（强制转换）。

- **窄化转换**可能导致数据丢失：
  - `float` → `int`：截断小数部分（例如 3.7 变为 3）
  - `double` → `float`：可能丢失精度
  - `int` → `char`：值可能被截断以适配
- **宽化转换**通常安全：
  - `int` → `double`
  - `char` → `int`
- **警告**：某些转换可能产生意外结果或编译器警告

**示例——隐式转换（不推荐）：**
```cpp
void printInt(int x) {
    printf("%d\n", x);
}

int main() {
    printInt(3.7);      // 传递 double，转换为 int：打印 3（截断！）
    printInt('A');      // 传递 char，转换为 int：打印 65
}
```

**示例——显式转换（推荐）：**
```cpp
void printInt(int x) {
    printf("%d\n", x);
}

int main() {
    printInt(static_cast<int>(3.7));   // 显式将 double 转为 int：打印 3
    printInt(static_cast<int>('A'));   // 显式将 char 转为 int：打印 65

    // C 风格转换也能工作，但 static_cast 在 C++ 中更推荐
    printInt((int)3.7);                // 结果相同，语法安全性较低
}
```

> **最佳实践：** 避免依赖隐式转换。使用显式转换来记录预期的转换，并在编译期捕获潜在的数据丢失。
>
> 另请参阅：[3.2.3 列表初始化（花括号初始化）](#323-list-initialization-brace-initialization) 了解如何使用 `{}` 在编译期防止窄化转换。
>
> 另请参阅：[4.9.3 C++ 风格类型转换](#493-c-style-casts) 了解 `static_cast` 和显式转换运算符的语法。

### 9.2.4 return 语句

**语法：**
```cpp
return expression;  // 用于非 void 函数
return;             // 用于 void 函数（可选）
```

**要点：**
- 表达式类型应与返回类型匹配（或可转换）
- `return` 立即终止函数执行
- 一个函数可以有多个 `return` 语句

**void 函数：**
```cpp
void printMessage() {
    printf("Hello\n");
    return;  // void 函数中可选
}
```

> **注意：** 在 `void` 函数中，`return` 语句不包含表达式。可以写成 `return;` 或完全省略。省略时，函数在执行到达闭花括号 `}` 时自动返回。

### 9.2.5 存储类与作用域

存储类决定变量的**生命周期**（变量存在多久）和**作用域**（变量可以在哪里访问）。

#### 9.2.5.1 概述

C++ 提供四种存储类来控制变量的生命周期和可见性：

| 存储类 | 关键字 | 作用域 | 生命周期 | 使用场景 |
|--------|--------|--------|----------|---------- |
| **自动** | `auto`（默认） | 局部（块/函数） | 块期间 | 局部变量 |
| **静态** | `static` | 局部（函数）或文件 | 程序期间 | 持久状态、文件私有 |
| **外部** | `extern` | 全局（整个程序） | 程序期间 | 跨文件共享 |
| **线程** | `thread_local` | 线程局部 | 线程期间 | 线程特定数据 |

#### 9.2.5.2 自动存储（`auto`）

**特性：**
- **局部变量**（函数或块内部）的默认存储类
- 变量在进入块时创建，退出时销毁
- 在现代 C++ 中很少显式写出

```cpp
void func() {
    int x = 10;        // 默认为 auto
    auto int y = 20;   // 旧风格，冗余
    // x 和 y 只在 func() 内部存在
}
```

> **注意：** 自 **C++11** 起，`auto` 主要用于**自动类型推导**（`auto x = 10;`）。旧的存储类用法（`auto int x = 10;`）有效但**已过时**。

#### 9.2.5.3 `static` 存储类

`static` 关键字根据上下文改变**链接性**（跨文件的可见性）或**生命周期**（变量存在多久）：

| 上下文 | `static` 的效果 |
|--------|---------------- |
| **局部变量** | 将生命周期延长到程序期间（值在调用之间持久存在） |
| **全局变量** | 将链接性限制为仅当前文件（内部链接） |

##### 9.2.5.3.1 静态局部变量

在函数调用之间保留值，只初始化一次。

**静态局部变量 vs 自动局部变量：**

| 特性 | `auto`（普通局部） | `static` 局部 |
| ---- | ------------------ | -------------- |
| **初始化** | 每次函数调用 | 只在第一次调用 |
| **销毁** | 函数退出 | 程序终止 |
| **值保留** | 否（每次调用重新创建） | 是（在调用之间持久存在） |
| **作用域** | 仅函数/块 | 仅函数/块 |
| **生命周期** | 仅函数/块 | 程序期间 |

**示例——对比：**

```cpp
// 普通局部变量
void auto_counter() {
    int count = 0;        // 每次调用都重新创建
    count++;
    printf("Auto: %d\n", count);  // 始终打印：1
}

// 静态局部变量  
void static_counter() {
    static int count = 0; // 只初始化一次，持久存在
    count++;
    printf("Static: %d\n", count); // 打印：1、2、3...
}

int main() {
    auto_counter();    // Auto: 1
    auto_counter();    // Auto: 1
    auto_counter();    // Auto: 1
    
    static_counter();  // Static: 1
    static_counter();  // Static: 2
    static_counter();  // Static: 3
}
```

**关键概念：** 静态局部变量具有**局部作用域**（只在函数内可见）但**全局生命周期**（存在于整个程序期间）。

**使用场景：** 函数调用计数、状态持久化、记忆化。

##### 9.2.5.3.2 静态全局变量

将全局变量限制为**仅当前文件**（内部链接）。

**静态全局变量 vs 普通全局变量：**

| 特性 | 普通全局 | 静态全局 |
|------|----------|---------- |
| **链接性** | 外部（可从其他文件访问） | 内部（仅文件内） |
| **生命周期** | 程序期间 | 程序期间 |
| **作用域** | 整个程序 | 仅文件内 |
| **声明** | `int value = 100;` | `static int value = 100;` |
| **从其他文件访问** | 是（通过 `extern`） | 否（链接器错误） |

**示例：**

```cpp
// 文件：helper.cpp
static int internalCounter = 0;  // 只在此文件内可见

void incrementInternal() {
    internalCounter++;
}

// 文件：main.cpp
extern int internalCounter;  // 错误：无法访问静态全局变量
```

**使用场景：** 封装、隐藏实现细节、防止大型项目中的名称冲突。

##### 9.2.5.3.3 总结：`static` 的两种含义

`static` 关键字根据使用位置有**相反的效果**：

| 用法 | 效果 | 目的 |
|------|------|------ |
| 函数内的 `static` | **延长**生命周期（更长） | 在调用之间保留值 |
| 函数外的 `static` | **限制**可见性（更小） | 对其他文件隐藏 |

**记忆口诀：**
- 在局部上："static = **stay**"（值保持存活）
- 在全局上："static = **stay here**"（留在本文件）

#### 9.2.5.4 外部存储（`extern`）

`extern` 实现全局变量的**跨文件共享**。

> 另请参阅：[3.1 核心概念：声明与定义](#31-the-core-concepts-declaration-vs-definition) 了解这些概念之间的根本区别。

**声明与定义：**

| 方面 | 定义 | 声明 |
|------|------|------ |
| **目的** | 创建变量，分配存储 | 告诉编译器变量存在于别处 |
| **内存** | 分配内存 | 不分配内存 |
| **次数** | **只能一次**（ODR） | 允许多次 |
| **语法** | `int value = 100;` | `extern int value;` |

**多文件使用：**

```cpp
// 文件：globals.cpp
int sharedValue = 100;         // 定义（只有一个）

// 文件：main.cpp
extern int sharedValue;        // 声明
int main() {
    printf("%d\n", sharedValue);  // 使用 globals.cpp 中的全局变量
}
```

**头文件模式（推荐）：**

```cpp
// globals.h
extern int sharedValue;        // 仅声明

// globals.cpp
int sharedValue = 100;         // 定义

// main.cpp
#include "globals.h"           // 获取声明
```

**函数也具有外部链接性：**

与全局变量一样，函数名默认具有外部存储类。这允许调用在其他文件中定义的函数。

**跨文件示例：**

*文件：math_utils.cpp*
```cpp
int add(int a, int b) {        // 定义
    return a + b;
}

int multiply(int a, int b) {   // 定义
    return a * b;
}
```

*文件：main.cpp*
```cpp
// 声明（原型）—— 告诉编译器函数存在于别处
int add(int a, int b);
int multiply(int a, int b);

int main() {
    int sum = add(5, 3);           // 调用 math_utils.cpp 中的函数
    int product = multiply(4, 7);  // 调用 math_utils.cpp 中的函数
    return 0;
}
```

**要点：** 函数默认具有外部链接性，因此可以像带 `extern` 的全局变量一样跨文件共享。

**注意：** 函数原型本质上是外部的。不需要在函数声明前写 `extern`：

```cpp
// 两者等价（第一个是标准写法，第二个冗余）
void foo();           // 标准函数原型
extern void foo();    // 冗余——函数默认就是外部的
```

**用 `static` 限制函数可见性：**

若要防止函数被其他文件调用，使用 `static`：

```cpp
// 文件：helper.cpp
static void internalFunc() { }  // 只在此文件内可见

// 文件：main.cpp
void internalFunc();  // 错误：找不到定义
```

声明为 `static` 的函数具有**内部链接性**——只能在同一翻译单元（源文件）内调用。

> **最佳实践：** 避免全局变量。优先传递参数。仅在必要时使用 `extern`。

**为什么参数比全局变量更好：**

函数原型清晰地展示依赖关系：
```cpp
void process(int data, double factor);  // 依赖关系显式可见
```

全局变量创建隐藏依赖：
```cpp
void process();  // 依赖关系隐藏——必须阅读函数体才能找到
// process() 内部使用了 globalVar1、globalVar2 等
```

参数使数据流显式且自文档化。全局变量使代码更难理解、测试和维护。

#### 9.2.5.5 线程存储（`thread_local`）

**C++11 特性：** 每个线程获得自己的独立副本。

```cpp
thread_local int threadCounter = 0;

void threadFunc() {
    threadCounter++;
    printf("Thread %d: counter = %d\n",
           std::this_thread::get_id(), threadCounter);
}

// 每个线程看到它自己的 threadCounter
```

**使用场景：** 线程特定缓存、每线程随机数生成器。

#### 9.2.5.6 可变存储（`mutable`）

允许在 `const` 成员函数中修改（用于逻辑常量性）：

```cpp
class Logger {
    mutable int logCount = 0;  // 即使在 const 方法中也能修改

public:
    void log(const string& msg) const {
        logCount++;  // OK：mutable 成员
        // msg 不变，但内部计数器更新
    }
};
```

#### 9.2.5.7 已过时：`register`

**C++11 中已弃用，C++17 中移除。**

```cpp
void process() {
    register int i;  // 已过时：编译器忽略此提示
    for (i = 0; i < 1000000; i++) { }
}
```

现代编译器自动优化。不要使用。

#### 9.2.5.8 作用域规则总结

| 变量类型 | 定义位置 | 可访问位置 | 生命周期 |
|----------|----------|------------|---------- |
| **局部** | 函数内部 | 仅在块内 | 块期间 |
| **全局** | 函数外部 | 任何函数 | 程序期间 |
| **静态局部** | 函数内带 `static` | 仅在函数内 | 程序期间 |
| **静态全局** | 外部带 `static` | 仅同一文件内 | 程序期间 |
| **线程局部** | 带 `thread_local` | 线程特定 | 线程期间 |

**作用域解析：**

```cpp
int x = 10;  // 全局

void func() {
    int x = 20;      // 局部隐藏全局
    printf("%d\n", x);  // 打印 20（局部）
    printf("%d\n", ::x); // 打印 10（全局）
}
```

### 9.2.6 内联函数

**什么是 `inline`？**

`inline` 是给编译器的提示："这个函数很小，所以直接将其主体复制到调用点，而不是执行真正的函数调用。" 这消除了压入参数、跳转到函数和返回的开销。

**为什么优先使用 `inline` 而非宏？**

| 特性 | 宏 `#define MAX(a,b)` | `inline` 函数 |
|------|------------------------|--------------- |
| 参数求值 | 文本替换，可能多次求值 | ✓ 只求值一次 |
| 类型安全 | ✗ 无 | ✓ 完整类型检查 |
| 可调试性 | ✗ 调试器看到的是展开后的代码 | ✓ 调试器看到函数名 |
| 作用域 | 全局污染 | ✓ 尊重命名空间 |
| 性能 | 无函数调用开销 | ✓ 无函数调用开销（由编译器内联） |

**使用示例：**

```cpp
inline int square(int x) { return x * x; }

int main() {
    int a = 5;
    int result = square(a++);  // a 自增一次，result = 25
    
    // 宏版本会导致 bug：
    // #define SQUARE(x) ((x) * (x))
    // SQUARE(a++) → ((a++) * (a++))  // a 自增两次！
}
```

**何时使用 `inline`：**

- 用于小型、频繁调用的函数（getter、简单数学运算）
- 在头文件中需要包含函数定义时（避免多重定义链接器错误）

**现代 C++ 说明：**

现代编译器即使没有 `inline` 关键字也会自动内联小型函数。显式 `inline` 在头文件中仍有用以避免多重定义错误，但出于优化目的，编译器通常最清楚该怎么做。
